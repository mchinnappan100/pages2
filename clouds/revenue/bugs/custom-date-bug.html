Root Cause Analysis (RCA)

Issue Summary

Error Message:

UNKNOWN_EXCEPTION: An unexpected error occurred.
Please include this ErrorId if you contact support:
519600426-3503 (1700169536)

User Impact: Users are unable to add a new step in the Change Fulfillment Workspace. The operation fails and displays an unexpected error message.

Steps to Reproduce

  1. Open App Launcher.
  2. Search for and open Fulfillment Workspace.
  3. Select Change Fulfillment Workspace.
  4. Navigate to the Change SOLM swim lane.
  5. Click Add Step.
  6. The following error is displayed:
UNKNOWN_EXCEPTION: An unexpected error occurred.
Please include this ErrorId if you contact support.

Additional logs indicate a Java Index Out Of Bounds Exception related to processing the Custom Date field.

Root Cause

When a user clicks Add Step, the application performs validation and processing for all configured fields, including the Custom Date field.

During this process, the system expects an active SalesTransactionContext record. If the associated SalesTransactionContext has a status other than Active, the Custom Date processing logic attempts to access data that is not available, resulting in a Java Index Out Of Bounds Exception.

The exception is not properly handled and surfaces to the user as a generic:

UNKNOWN_EXCEPTION

Root Cause Details

Condition Triggering the Error:

  • SalesTransactionContext Status ≠ Active

Failure Flow:

  1. User clicks Add Step.
  2. System validates step configuration fields.
  3. Custom Date processing is executed.
  4. Required context data is not available because the SalesTransactionContext is inactive.
  5. Java code attempts to access an invalid array/list index.
  6. A Java Index Out Of Bounds Exception is thrown.
  7. Salesforce surfaces the exception as an UNKNOWN_EXCEPTION.

Resolution

Update the associated SalesTransactionContext record so that its Status is set to:

Active

After activating the SalesTransactionContext:

  1. Refresh the Fulfillment Workspace.
  2. Navigate back to the Change SOLM swim lane.
  3. Click Add Step again.
  4. The step should be created successfully without errors.

Preventive Actions

Short-Term

  • Verify that the SalesTransactionContext status is Active before attempting to add fulfillment steps.

Long-Term

  • Add validation to check SalesTransactionContext status before processing Custom Date fields.
  • Display a user-friendly error message such as:

"Unable to add step because the associated Sales Transaction Context is not Active."

  • Add defensive exception handling to prevent Java Index Out Of Bounds exceptions from propagating to the UI.

Fix Verification

Expected Result After Fix:

  • SalesTransactionContext Status = Active
  • User can successfully add a step in the Change SOLM swim lane.
  • No UNKNOWN_EXCEPTION is displayed.
  • Custom Date field processing completes successfully.
