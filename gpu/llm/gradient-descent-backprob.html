01

The Mountain Intuition

Finding the lowest point in a foggy landscape

Imagine you are standing on a foggy mountain and want to find the lowest valley. You can't see the whole landscape — but you can feel the slope of the ground beneath your feet. Gradient descent is simply the strategy of always stepping in the downhill direction.

  • Height = Loss (how wrong the model is)
  • Your position = Current values of all model weights
  • Gradient = The slope — which direction is uphill, and how steep
  • Step = Updating the weights by a small amount in the downhill direction

Repeat this process thousands or millions of times and you gradually descend to a low point where the model's predictions are good.

Loss Landscape — Finding the Valley Loss ↑ │ ● ← starting position (random weights, high loss) │ ╲ │ ╲ ← gradient points uphill; we step opposite │ ╲ │ ● ← after one update │ ╲ │ ╲ │ ● ← after several updates (descending) │ ╲ │ ╲___●___ ← near the minimum │ ↑ │ local minimum └──────────────────────────────→ Weight value (θ) At every step: move opposite to the gradient → loss decreases

The Gradient Descent Formula

The update rule that drives all neural network training

Parameter update rule: θ_new = θ_old − α · ∇J(θ) Where: θ = model parameters (weights and biases) α = learning rate (how big each step is) ∇J(θ) = gradient of the loss J with respect to θ (how much the loss changes when θ changes) − = we move OPPOSITE to the gradient (downhill) For a single weight w: w_new = w_old − α · (∂L/∂w) Example: w = 2.0, gradient ∂L/∂w = −12, α = 0.01 w_new = 2.0 − 0.01 × (−12) = 2.0 + 0.12 = 2.12 ← weight increased because gradient was negative

Why the Minus Sign?

The gradient points uphill — in the direction of increasing loss. To decrease the loss, we move in the opposite direction. Hence the minus sign: θ = θ − α · ∇J.

What Is ∇J(θ)?

The gradient ∇J(θ) is a vector containing the partial derivative of the loss with respect to every single weight. It has exactly the same shape as θ. For a model with 1 million weights, the gradient is a vector of 1 million numbers — one for each weight, telling it which direction to move and by how much.

03

The Learning Rate

Too large diverges, too small stalls — finding the sweet spot

The learning rate α controls how large each step is. It is arguably the most important hyperparameter in training a neural network.

Effect of Learning Rate on Training α too small (e.g. 1e-7): Loss ↑ ● │ ● │ ● ← progress is real but extremely slow │ ● │ ●●●●●●●●● ... (many thousands of steps still high) α just right (e.g. 3e-4): Loss ↑ ● │ ● │ ● │ ● │ ●●● ← converges smoothly α too large (e.g. 0.1): Loss ↑ ● ● ● ← oscillates, overshoots the minimum │ ● ● ● ● │ ●● ●●● └────────────────────────────→ Steps The sweet spot: loss descends steadily without oscillation

📈 Too Large

Overshoots the minimum. Loss oscillates or diverges. Model never converges.

  • Loss goes up and down erratically
  • Weights become very large (NaN in extreme cases)
  • Typical sign: loss spikes randomly during training

📉 Too Small

Takes forever to converge. Gets trapped in local minima or saddle points. Extremely inefficient.

  • Training makes progress but painfully slowly
  • May never reach a good minimum in practice
  • Wastes compute time

✅ Just Right

Steady, reliable descent. Loss curves are smooth. Reaches a good minimum efficiently.

  • Loss decreases consistently each epoch
  • Start larger, decay over time (cosine schedule)
  • For LLMs: ~3e-4 with cosine decay is standard

Batch, Mini-batch & Stochastic GD

How many examples to use per gradient update

The original gradient descent computes the gradient over the entire dataset before each update. In practice, modern deep learning always uses mini-batch gradient descent.

VariantExamples per updateGradient qualitySpeedUsed in
Batch GDAll N examplesExact — no noiseVery slow for large datasetsTiny datasets only
Stochastic GD (SGD)1 exampleVery noisyFast per step, many stepsOnline learning
Mini-batch GD32–512 examplesGood estimate, some noiseEfficient on GPUAll modern DL
Why Mini-batch?

Mini-batches provide a good balance: the gradient estimate is accurate enough to make reliable progress, and processing multiple examples simultaneously uses the GPU's parallel compute efficiently. The noise from mini-batches also acts as a regulariser — it helps escape sharp local minima.

Mini-batch Gradient Descent — one epoch Dataset: 10,000 examples | Batch size: 32 | Steps per epoch: 10,000/32 ≈ 313 Step 1: Sample batch [ex_1, ex_7, ..., ex_32] Compute loss on these 32 examples Compute gradient ∂L/∂θ Update: θ ← θ − α·∂L/∂θ Step 2: Sample next batch [ex_33, ex_40, ..., ex_64] ...repeat... Step 313: All examples seen once → end of epoch Shuffle dataset, start epoch 2 Each step is much faster than processing all 10,000 examples. GPU processes all 32 examples in the batch in parallel.
05

Momentum, Adam & AdamW

Better optimisers built on top of gradient descent

Plain gradient descent can be slow and noisy. Modern optimisers add momentum (smooth out gradient noise) and adaptive learning rates (give each weight its own effective step size).

Momentum SGD: v ← β·v + (1−β)·g ← velocity: exponential moving average of gradients θ ← θ − α·v ← step using smoothed gradient Adam (Adaptive Moment Estimation): m ← β₁·m + (1−β₁)·g ← first moment (mean of gradients) v ← β₂·v + (1−β₂)·g² ← second moment (mean of squared gradients) m̂ = m / (1 − β₁ᵗ) ← bias correction v̂ = v / (1 − β₂ᵗ) ← bias correction θ ← θ − α · m̂ / (√v̂ + ε) ← adaptive step per parameter AdamW (Adam + decoupled weight decay): θ ← θ · (1 − α·λ) ← weight decay FIRST (decoupled) θ ← θ − α · m̂ / (√v̂ + ε) ← then gradient update Typical hyperparameters for LLMs: β₁ = 0.9, β₂ = 0.95, ε = 1e-8, α = 3e-4, λ = 0.1
Why AdamW for LLMs?

AdamW is the standard for training transformer-based LLMs (GPT, LLaMA, BERT). The adaptive learning rates mean sparse features (rare words) get larger steps — correcting the "gradient starvation" problem. The decoupled weight decay properly regularises the model without interfering with the adaptive gradient scaling.

The Loss Landscape

Local minima, saddle points, and why they matter

The loss function defines a surface over the space of all possible weight values. Training is navigation on this surface. Understanding its features — saddle points, flat regions, sharp vs. flat minima — explains many training phenomena.

Features of the Loss Landscape Loss ↑ │ saddle local │ point min │ /╲ /╲ │ / ╲ / ╲ ← global minimum │ / ╲──────────/ ╲___● │ / gradient=0 ╲ │/ at saddle, but not min ╲___ └──────────────────────────────────→ Weight space (high-dim in practice) Types of critical points (gradient = 0): Local minimum: all directions go uphill → trapped here Global minimum: lowest point overall → ideal Saddle point: some directions up, some down → gradient descent escapes For deep networks: saddle points are far more common than local minima. The good news: in high dimensions, most local minima are good enough!
07

What Is Backpropagation?

The algorithm that computes gradients for millions of weights — efficiently

Backpropagation is the algorithm that computes the gradient of the loss with respect to every weight in the network — in one efficient backward pass. Without backprop, we would need a separate forward pass for each weight to estimate its gradient, making training of deep networks computationally impossible.

Backprop uses the chain rule of calculus to propagate gradient information backward through the network, from the loss down to the very first layer's weights.

Forward vs Backward Pass

  • Forward pass: compute the output and loss. Data flows left to right. All intermediate activations are stored.
  • Backward pass: compute gradients. Gradient information flows right to left — from loss back to inputs. Uses stored activations from the forward pass.
Forward Pass → Loss → Backward Pass FORWARD PASS: x ──→ [Layer 1] ──→ [Layer 2] ──→ [Layer 3] ──→ ŷ ──→ Loss a₁ stored a₂ stored a₃ stored BACKWARD PASS: ∂L/∂W₁ ←── ∂L/∂a₁ ←── ∂L/∂a₂ ←── ∂L/∂a₃ ←── ∂L/∂ŷ ←── Loss Each layer receives ∂L/∂output from the layer above it (right) and computes ∂L/∂W (for its own weights) and ∂L/∂input (to pass further left). This is the chain rule in action.
08

The Chain Rule

The calculus principle that makes backpropagation work

The chain rule says: if z depends on y, and y depends on x, then the rate of change of z with respect to x equals the product of the individual rates:

Chain Rule (single variable): dz/dx = dz/dy · dy/dx Example: y = x², z = sin(y) dz/dx = dz/dy · dy/dx = cos(y) · 2x = cos(x²) · 2x Multi-variable chain rule (neural network): If L = f(a), a = g(z), z = w·x + b then: ∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w = ∂L/∂a · g'(z) · x ∂L/∂b = ∂L/∂a · ∂a/∂z · ∂z/∂b = ∂L/∂a · g'(z) · 1 Key insight: each layer multiplies the incoming gradient by its local derivative and passes the result to the previous layer.
Why This Is Efficient

The chain rule lets us reuse computations. When computing ∂L/∂W₁, we need ∂L/∂a₁. When computing ∂L/∂W₂ (next layer), we also needed ∂L/∂a₁ as an intermediate. By computing the backward pass layer by layer, each intermediate gradient is computed exactly once and reused — this is what makes backpropagation O(n) in the number of parameters, not O(n²).

Computation Graph

Representing calculations as a directed acyclic graph (DAG)

Every neural network computation can be represented as a computation graph — a directed acyclic graph (DAG) where nodes are operations and edges carry data. Backpropagation traverses this graph in reverse topological order.

PyTorch and JAX build this graph dynamically during the forward pass and then traverse it backward when you call .backward().

Computation Graph for L = (w·x − y)² Forward (build graph, store values): w ──────────────┐ × ──→ z=w·x ──┐ x ──────────────┘ − ──→ e=z−y ──→ (·)² ──→ L=e² y ─────────────────────────────┘ Backward (traverse in reverse, apply chain rule): ∂L/∂e = 2e ∂L/∂z = ∂L/∂e · ∂e/∂z = 2e · 1 = 2e ∂L/∂w = ∂L/∂z · ∂z/∂w = 2e · x ∂L/∂x = ∂L/∂z · ∂z/∂x = 2e · w ∂L/∂y = ∂L/∂e · ∂e/∂y = 2e · (−1) = −2e Numerical example: w=2, x=3, y=8 z = 6, e = 6−8 = −2, L = 4 ∂L/∂w = 2×(−2)×3 = −12 (increase w to reduce loss) ∂L/∂b = 2×(−2)×1 = −4 (if there's a bias b)

Worked Example — Complete Forward & Backward Pass

One neuron, one example, all steps shown

1

Set up the network

Single neuron: y_pred = ReLU(w·x + b). Loss: MSE = (y_pred − y_true)².
Initial values: w=0.5, b=0, x=2.0, y_true=3.0.

2

Forward pass

z = w·x + b = 0.5×2 + 0 = 1.0
a = ReLU(z) = max(0, 1.0) = 1.0
L = (a − y_true)² = (1.0 − 3.0)² = 4.0

3

Backward pass — step 1: gradient of L w.r.t. a

∂L/∂a = 2·(a − y_true) = 2·(1.0 − 3.0) = −4.0

4

Backward pass — step 2: gradient through ReLU

∂a/∂z = 1 if z > 0, else 0 → since z=1.0 > 0: ∂a/∂z = 1.0
∂L/∂z = ∂L/∂a · ∂a/∂z = −4.0 × 1.0 = −4.0

5

Backward pass — step 3: gradient w.r.t. w and b

∂z/∂w = x = 2.0
∂L/∂w = ∂L/∂z · ∂z/∂w = −4.0 × 2.0 = −8.0
∂z/∂b = 1
∂L/∂b = ∂L/∂z · ∂z/∂b = −4.0 × 1 = −4.0

6

Weight update (learning rate α=0.1)

w_new = 0.5 − 0.1×(−8.0) = 0.5 + 0.8 = 1.3
b_new = 0.0 − 0.1×(−4.0) = 0.0 + 0.4 = 0.4

7

Verify improvement

New prediction: z = 1.3×2 + 0.4 = 3.0. Loss = (3.0−3.0)² = 0.0 ✓
One update solved it perfectly for this linear example!

PyTorch Autograd

How automatic differentiation works in practice

Python · Manual example matches worked example above
import torch
import torch.nn.functional as F

# Set up parameters with requires_grad=True so PyTorch tracks operations
w = torch.tensor(0.5, requires_grad=True)   # weight — to be learned
b = torch.tensor(0.0, requires_grad=True)   # bias   — to be learned
x = torch.tensor(2.0)                         # input  — fixed, no gradient needed
y_true = torch.tensor(3.0)                    # target — fixed

# ── Forward pass: compute prediction and loss ──────────────────────────
z      = w * x + b          # pre-activation = 1.0
a      = F.relu(z)           # activation = 1.0 (ReLU, z>0)
loss   = (a - y_true) ** 2   # MSE loss = (1-3)^2 = 4.0

print(ff"Forward: z={z.item():.2f}  a={a.item():.2f}  loss={loss.item():.2f}")

# ── Backward pass: compute all gradients in one call ──────────────────
loss.backward()   # traverses the computation graph backward

print(ff"∂L/∂w = {w.grad.item():.2f}  (expected: -8.0)")
print(ff"∂L/∂b = {b.grad.item():.2f}  (expected: -4.0)")

# ── Weight update ──────────────────────────────────────────────────────
lr = 0.1
with torch.no_grad():   # disable gradient tracking during update
    w -= lr * w.grad     # w = 0.5 - 0.1×(-8) = 1.3
    b -= lr * b.grad     # b = 0.0 - 0.1×(-4) = 0.4

print(ff"Updated: w={w.item():.2f}  b={b.item():.2f}")

# Verify: new prediction should be 3.0 (zero loss)
with torch.no_grad():
    new_pred = F.relu(w * x + b)
    new_loss = (new_pred - y_true) ** 2
    print(ff"New pred={new_pred.item():.2f}  new loss={new_loss.item():.2f}")
Python · Full training loop with nn.Module
import torch, torch.nn as nn, torch.optim as optim

# ── Model ─────────────────────────────────────────────────────────────
class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(4, 8)   # 4 inputs → 8 hidden
        self.fc2 = nn.Linear(8, 1)   # 8 hidden → 1 output

    def forward(self, x):
        x = torch.relu(self.fc1(x))  # hidden layer with ReLU
        return self.fc2(x)            # output (no activation for regression)

model     = SimpleNet()
criterion = nn.MSELoss()              # mean squared error loss
optimizer = optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1e-4)

# ── Data ──────────────────────────────────────────────────────────────
torch.manual_seed(42)
X = torch.randn(100, 4)              # 100 examples, 4 features
y = X[:, 0] * 2 + X[:, 1] - 1       # ground truth: simple linear rule

# ── Training loop ─────────────────────────────────────────────────────
for epoch in range(100):
    # 1. Forward pass
    pred = model(X).squeeze()           # shape: (100,)
    loss = criterion(pred, y)           # scalar loss

    # 2. Zero gradients (must do every step — they accumulate!)
    optimizer.zero_grad()

    # 3. Backward pass — compute ∂L/∂W for every parameter
    loss.backward()

    # 4. Gradient clipping (prevents exploding gradients)
    nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

    # 5. Update weights
    optimizer.step()

    if epoch % 20 == 0:
        print(ff"epoch {epoch:3d}  loss={loss.item():.4f}")

Vanishing & Exploding Gradients

Why deep networks are hard to train — and how to fix it

During backpropagation through many layers, gradients are multiplied together via the chain rule. If those values are consistently less than 1, the gradients shrink exponentially as they travel back — reaching near-zero by the time they arrive at the first layers. This is the vanishing gradient problem.

Conversely, if they are consistently greater than 1, they grow exponentially — the exploding gradient problem.

Vanishing Gradient — 10-layer network Layer 10 receives gradient: 1.000 Layer 9 receives gradient: 0.250 ← multiplied by sigmoid'(x) ≤ 0.25 Layer 8 receives gradient: 0.063 Layer 7 receives gradient: 0.016 Layer 6 receives gradient: 0.004 Layer 5 receives gradient: 0.001 Layer 4 receives gradient: 0.0002 Layer 3 receives gradient: ~0.00006 Layer 2 receives gradient: ~0.00001 Layer 1 receives gradient: ~0.000003 ← effectively zero! Early layers learn nothing — the gradients are too small to matter. Solutions: 1. Use ReLU/GELU ← gradient = 1 for positive inputs (no shrinkage) 2. Residual connections ← skip-connections create gradient highways 3. Layer/RMS Normalisation ← keeps activations in good range 4. Careful initialisation ← Xavier, Kaiming init

🛤️ Residual Connections

Add the input directly to the output of each layer: x = x + f(x). The gradient flows through the "highway" of direct connections unchanged, regardless of depth.

📊 Normalisation Layers

LayerNorm and RMSNorm keep activations at unit scale. This prevents the per-layer derivatives from collapsing toward zero.

⚡ GELU / ReLU Activations

Sigmoid and tanh have maximum derivatives of 0.25 and 1.0 respectively. ReLU has derivative exactly 1 for positive inputs — no gradient shrinkage in the positive regime.

✂️ Gradient Clipping

Cap the gradient norm to a threshold (e.g. 1.0) before each update. Prevents individual large batches from causing catastrophic weight updates.

The Full Training Loop

Putting it all together — gradient descent + backprop in practice

Complete Training Cycle Initialise: weights ← random (Xavier/Kaiming) optimizer ← AdamW(lr=3e-4, weight_decay=0.1) scheduler ← CosineAnnealingLR(T_max=total_steps) For each epoch: For each mini-batch of B examples: Step 1: Forward Pass x_batch, y_batch = sample B examples ŷ = model(x_batch) ← run all layers, store activations Step 2: Compute Loss loss = criterion(ŷ, y_batch) ← scalar value Step 3: Zero Gradients optimizer.zero_grad() ← clear accumulated gradients from last step Step 4: Backward Pass loss.backward() ← chain rule, fills .grad for all parameters Step 5: Gradient Clipping clip_grad_norm_(params, 1.0) ← prevent exploding gradients Step 6: Update Weights optimizer.step() ← θ ← θ − α·∇J (with Adam momentum) Step 7: Update LR Schedule scheduler.step() ← decay learning rate over time Validate on held-out set Log metrics, save checkpoint

The One-Sentence Summary

Backpropagation uses the chain rule to efficiently compute ∂L/∂w for every weight in one backward pass. Gradient descent then uses those gradients to update weights via θ ← θ − α·∇J — moving the model toward better predictions. Together, repeated over millions of steps, these two algorithms are what "training a neural network" means.
