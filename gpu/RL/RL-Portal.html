How RL Fits into LLM Training
RL is not where models learn knowledge — it's where they learn behaviour
Stage 1: Pretraining (self-supervised — ~90% of capability) Internet + Books + Code → Predict Next Token → Pretrained LLM Stage 2: Supervised Fine-Tuning (SFT) (no RL yet) Human demonstrations → Imitate ideal responses → SFT Model Stage 3a: RLHF (human feedback → reward model → PPO) Human preference pairs → Train Reward Model → PPO optimisation → Aligned Model Stage 3b: RLAIF / CAI (AI feedback at scale) AI judge scores responses → Preference data → RL training → Aligned Model at scale Stage 4: RL with Verifiable Rewards (math, code, logic) Run unit tests / check answers → Reward = pass rate → Improved reasoning

Key Concepts at a Glance
One-line definitions for every important term
RL · Book 1
Policy π(a|s)
The agent's strategy — a mapping from states to action probabilities. For LLMs: P(next token | context).
RL · Book 1
Reward Signal r
Scalar feedback after each action. The agent's objective is to maximise cumulative discounted reward Σ γᵗrₜ.
RL · Book 1
PPO
Proximal Policy Optimisation — clips the policy update ratio to stay within ε of the old policy, preventing destabilising large updates.
RL · Book 1
GRPO
Group Relative Policy Optimisation — computes advantages relative to the group mean, eliminating the need for a separate value/critic network.
RLHF · Book 2
SFT Model
Supervised Fine-Tuned model — the starting point for RLHF, trained to imitate human demonstrations before RL begins.
RLHF · Book 2
Reward Model
Neural network trained to score (prompt, response) pairs using human preference data. Output is used as reward signal in PPO.
RLHF · Book 2
KL Penalty
Penalises the policy for diverging from the SFT reference model, preventing reward hacking and maintaining coherent language.
RLHF · Book 2
DPO
Direct Preference Optimisation — optimises human preferences directly without a separate reward model or RL training loop.
RLAIF · Book 3
AI Judge
A powerful model (GPT-4, Claude) that replaces human annotators, generating preference labels at ~100× lower cost and higher scale.
RLAIF · Book 3
Constitutional AI
Anthropic's technique where a model critiques and revises its own responses against a written set of principles, then uses AI-generated preferences for RL.
RLAIF · Book 3
Self-Play
Model generates and evaluates its own responses iteratively, improving each round by selecting the best candidates as training data.
RLAIF · Book 3
Scalable Oversight
Research agenda for supervising AI on tasks beyond human ability, using debate, amplification, or recursive reward modelling.
