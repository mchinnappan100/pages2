What Is Reinforcement Learning?

Learning by interacting with an environment and receiving rewards

Reinforcement learning (RL) is a paradigm where an agent learns to make decisions by interacting with an environment. Unlike supervised learning (which learns from labelled examples) and unsupervised learning (which finds patterns in data), RL learns from reward signals — feedback that says whether an action was good or bad.

Think of it like training a dog: you don't tell the dog exactly what to do — you reward good behaviour and ignore or correct bad behaviour. Over many interactions, the dog learns the policy that maximises rewards.

The RL Loop Agent │ observes State (s) │ ▼ Agent selects Action (a) ← based on its Policy π(a|s) │ ▼ Environment │ transitions to new state s' │ emits Reward (r) │ ▼ Agent updates its policy to maximise cumulative reward Goal: find π* = policy that maximises E[Σ γᵗ rₜ] where γ (gamma) is the discount factor (0 < γ ≤ 1)
02

Core Components

Agent, environment, state, action, reward, policy, value function

🤖

Agent & Policy

The agent is the decision-maker. Its policy π(a|s) is the mapping from states to actions — the agent's strategy. For LLMs, the policy is the language model itself: state = conversation context, action = next token.

  • Deterministic policy: π(s) = a (always same action)
  • Stochastic policy: π(a|s) = probability of action a
  • LLM policy: π(token|context) = softmax over vocabulary
🌍

Environment & Reward

The environment is everything the agent interacts with. It receives actions and returns the next state and a scalar reward. In RLHF, the "environment" is the reward model — a neural network that scores responses.

  • Dense reward: signal after every action (every token)
  • Sparse reward: only at the end of an episode (full response)
  • RLHF: reward from a human preference model
  • RL+verifiable: reward from code tests or math checker
💎

Value Function

V(s) estimates the expected total future reward from state s. Q(s,a) estimates the value of taking action a in state s. These functions help the agent plan ahead — deciding whether to take a low-immediate-reward action for a higher long-term payoff.

  • V(s) = E[Σ γᵗrₜ | start at s, follow π]
  • Q(s,a) = E[Σ γᵗrₜ | take a at s, then follow π]
  • Advantage A(s,a) = Q(s,a) − V(s) (is this action better than average?)
03

Markov Decision Process (MDP)

The formal mathematical framework for RL problems

Every RL problem is formally defined as a Markov Decision Process (MDP): a 5-tuple (S, A, T, R, γ) where S is the state space, A is the action space, T is the transition function, R is the reward function, and γ is the discount factor.

The Markov property states that the future depends only on the current state, not on history. In language modelling, this is approximated — the full conversation context encodes the relevant history.

MDP = (S, A, T, R, γ) S: state space (e.g. all possible conversation contexts) A: action space (e.g. vocabulary of 50,257 tokens) T: T(s'|s,a) = probability of transitioning to s' given (s,a) R: R(s,a,s') = immediate reward γ: discount factor ∈ [0,1] (γ=1: future matters equally; γ=0: only immediate reward) Return: Gₜ = rₜ + γrₜ₊₁ + γ²rₜ₊₂ + ... = Σ γᵏ rₜ₊ₖ Bellman Equation (policy evaluation): V^π(s) = Σₐ π(a|s) Σₛ' T(s'|s,a) [R(s,a,s') + γ V^π(s')]

PPO — Proximal Policy Optimisation

The algorithm that trained GPT-4 and Claude — a deep dive

PPO (Schulman et al., 2017) is the workhorse of RLHF. The key insight: vanilla policy gradient updates can be too large, causing the policy to collapse into degenerate behaviour. PPO clips the ratio of new-to-old policy probabilities to stay within a trust region, keeping updates stable.

PPO Objective (clipped surrogate): L_CLIP(θ) = E[min(rₜ(θ)·Aₜ, clip(rₜ(θ), 1−ε, 1+ε)·Aₜ)] Where: rₜ(θ) = π_θ(aₜ|sₜ) / π_θ_old(aₜ|sₜ) ← probability ratio Aₜ = advantage estimate (is this action better than average?) ε = clip range (typically 0.2) Intuition: rₜ > 1+ε: action more likely under new policy → clip, don't over-update rₜ < 1−ε: action less likely under new policy → clip, don't over-punish Full RLHF PPO objective: L = L_CLIP − β · KL(π_θ || π_ref) + λ · L_value ↑ RL reward ↑ don't drift far ↑ value function accuracy
Python · PPO advantage and clipped loss
import torch

def ppo_clip_loss(log_probs_new, log_probs_old, advantages, eps=0.2):
    """
    log_probs_new: log π_θ(a|s)      shape (B,)
    log_probs_old: log π_θ_old(a|s)  shape (B,)
    advantages: Aₜ                   shape (B,)
    """
    # Probability ratio  rₜ = π_new / π_old  (in log space → exp of diff)
    ratio = torch.exp(log_probs_new - log_probs_old)

    # Unclipped objective:  rₜ · Aₜ
    unclipped = ratio * advantages

    # Clipped objective:  clip(rₜ, 1-ε, 1+ε) · Aₜ
    clipped = torch.clamp(ratio, 1 - eps, 1 + eps) * advantages

    # Take minimum → conservative update
    loss = -torch.min(unclipped, clipped).mean()   # negate: we minimise
    return loss

# Example
log_p_new = torch.tensor([-0.5, -0.8, -0.3])
log_p_old = torch.tensor([-0.6, -0.7, -0.4])
advantages = torch.tensor([0.8, -0.3, 1.2])

loss = ppo_clip_loss(log_p_new, log_p_old, advantages)
print(ff"PPO loss: {loss.item():.4f}")

RL with Verifiable Rewards

When the correct answer can be checked automatically — no humans needed

For tasks with objectively correct answers, the reward signal can be computed automatically — no human raters and no reward model required. This is called outcome-based RL or RL with verifiable rewards, and it is central to how modern reasoning models are trained.

Math verification Problem: 125 × 17 = ? Model answer: 2125 Checker: 125 × 17 == 2125 ✓ Reward: +1 Model answer: 2100 Checker: 125 × 17 == 2100 ✗ Reward: 0
Code verification Task: Write sort function Model writes: def sort(arr): ... Run unit tests: test_empty() ✓ test_sorted() ✓ test_reverse() ✓ Reward: +1 (all 3 pass) Reward: +0.33 (1/3 pass)
🧮

Best for Verifiable Rewards

Tasks where correctness can be checked without human judgment — the gold standard for scalable RL training.

  • Mathematics (symbolic verification)
  • Programming (unit test pass rate)
  • Logic & formal reasoning
  • Structured data extraction (schema validation)
  • Tool use (API call correctness)
🚀

GRPO — Group Relative Policy Optimisation

Used by DeepSeek-R1. Samples multiple responses per prompt, computes relative rewards within the group — no separate value network required. Simpler and more stable than PPO for reasoning tasks.

  • Sample G responses per prompt
  • Reward each: r₁, r₂, ..., rG
  • Advantage = (rᵢ − mean(r)) / std(r)
  • No critic network — reduces memory & complexity
  • Enables large-batch verifiable reward training

Key Insight

RL does not teach the model new knowledge — that comes from pretraining on trillions of tokens. RL teaches the model how to use its knowledge: following instructions faithfully, producing safe responses, reasoning step-by-step, and writing correct code. Think of pretraining as filling a library, and RL as teaching how to navigate it helpfully.
