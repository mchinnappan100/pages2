What Is a Gradient?

The direction and rate of change of the loss with respect to every weight

A gradient in a neural network is a vector of numbers that answers one fundamental question for every single weight in the model:

"How much does the loss change if I increase this weight by a tiny amount?"

A positive gradient for weight w means increasing w would increase the loss — so you should decrease it. A negative gradient means increasing w would decrease the loss — so you should increase it. The magnitude tells you how big an adjustment to make.

Without gradients, a neural network has no way to know which weights are causing errors or how to fix them. The gradient is the signal that drives all learning.

➕

Positive Gradient

∂L/∂w > 0 means the loss increases when w increases. To reduce the loss, we should decrease w. Gradient descent subtracts the gradient, so this weight will get smaller.

  • Loss goes up if weight goes up
  • Gradient descent: w ← w − η·(+grad) → w decreases
  • Analogy: turning a knob clockwise makes things worse
➖

Negative Gradient

∂L/∂w < 0 means the loss decreases when w increases. To reduce the loss, we should increase w. Gradient descent subtracts a negative number, so this weight gets larger.

  • Loss goes down if weight goes up
  • Gradient descent: w ← w − η·(−grad) → w increases
  • Analogy: turning a knob clockwise makes things better
≈0

Near-Zero Gradient

|∂L/∂w| ≈ 0 means this weight barely affects the loss. It is already near-optimal, or it's in a flat region of the loss landscape. The update will be tiny and convergence will be slow.

  • Weight already close to optimal
  • Or: saturated activation (sigmoid/tanh problem)
  • Or: vanishing gradient in deep networks
  • Update: nearly zero → learning stalls
02

Mountain Intuition

Gradient descent as finding the lowest point in a foggy landscape

Imagine you are standing on a foggy mountain and want to reach the lowest point (the global minimum of the loss). You can't see the whole landscape — only the slope under your feet right now. The gradient tells you which direction is uphill. To go downhill and reduce the loss, you walk in the opposite direction of the gradient.

  • Height = Loss (how wrong the model's predictions are)
  • Your position = Current values of all weights
  • Gradient = Arrow pointing in the direction of steepest ascent
  • Step size = Learning rate (how far you walk per step)
Gradient Descent — finding the valley Loss │ ● │ ← high loss (bad model) / ↑ │ ← Gradient (uphill) / │ │ / │ │ / │ │ / ↓ gradient ← walk opposite direction to gradient / │ ●──────────────────────────────── weights ↑ current position Each training step: 1. Compute gradient (which direction is uphill here?) 2. Walk opposite to gradient (downhill) 3. Repeat until you reach (or get close to) the valley Loss landscape for 2 weights w₁, w₂ (2D slice): w₂ ↑ │ ∇ ∇ ← gradient arrows point away from valley │ ∇ ● ∇ ← ● = current position │ ∇ ∇ │ valley (minimum loss) └──────────────────→ w₁
Why "Opposite Direction"?

The gradient points uphill — toward increasing loss. To decrease the loss, we move against the gradient: w_new = w − η · ∇L. The minus sign is what makes gradient descent. If we added the gradient instead, we'd do gradient ascent — making the model worse (used in some adversarial training scenarios).

Single Weight — Step-by-Step

Computing the gradient by hand for y = wx + b (from gradient.md)

Let's trace through the complete gradient computation for the simplest possible neural network: a single neuron with one weight and one bias, using Mean Squared Error loss.

Model: y = w·x + b Loss: L = (y − y_true)² Given: w = 2, x = 3, b = 0, y_true = 8 Forward pass: y = 2 × 3 = 6 L = (6 − 8)² = (−2)² = 4 Gradient of L w.r.t. w (chain rule): ∂L/∂w = ∂L/∂y · ∂y/∂w = 2(y − y_true) · x = 2(6 − 8) · 3 = 2(−2) · 3 = −12 Interpretation: Gradient = −12 (negative) → Increasing w reduces the loss → We should increase w to better predict y_true = 8
Why is the gradient −12? — Visual intuition Current state: w=2, prediction=6, truth=8, error=−2 (under-predicted) If we increase w by 0.01: y_new = (2.01) × 3 = 6.03 ← prediction went up L_new = (6.03 − 8)² = 3.8809 ← loss went DOWN (from 4.0) ΔL/Δw ≈ (3.8809 − 4.0) / 0.01 = −11.91 ≈ −12 ← matches gradient! Gradient = −12 means: "For every unit increase in w, the loss decreases by about 12" (More precisely: at this exact point, the derivative is −12) Large magnitude (12): weight needs significant adjustment Negative sign: we should INCREASE w to reduce loss
Python · Manual gradient computation + PyTorch verification
import torch

# ── Setup (matches gradient.md example) ───────────────────────────
w      = torch.tensor(2.0, requires_grad=True)   # weight
x      = torch.tensor(3.0)                          # input
b      = torch.tensor(0.0)                          # bias
y_true = torch.tensor(8.0)                          # target

# ── Forward pass ──────────────────────────────────────────────────
y    = w * x + b        # prediction: 2*3 = 6
loss = (y - y_true)**2  # MSE loss: (6-8)² = 4

print(ff"Prediction: {y.item()}")    # 6.0
print(ff"Loss:       {loss.item()}")  # 4.0

# ── Backward pass (compute gradient) ─────────────────────────────
loss.backward()

print(ff"∂L/∂w  = {w.grad.item()}")   # -12.0 ✓ matches gradient.md

# ── Manual verification via finite differences ────────────────────
h = 1e-5
with torch.no_grad():
    L1 = ((w + h)*x + b - y_true)**2
    L2 = ((w - h)*x + b - y_true)**2
    numerical_grad = (L1 - L2) / (2*h)
print(ff"Numerical ∂L/∂w ≈ {numerical_grad.item():.6f}")  # ≈ -12.0 ✓

# ── Step by step: chain rule manually ─────────────────────────────
y_val    = 2.0 * 3.0              # = 6.0
dL_dy    = 2 * (y_val - 8.0)     # = 2(-2) = -4     ← ∂L/∂y
dy_dw    = 3.0                    # = x              ← ∂y/∂w
dL_dw    = dL_dy * dy_dw           # = -4 * 3 = -12   ← ∂L/∂w (chain rule)
print(ff"Chain rule: ∂L/∂y={dL_dy}, ∂y/∂w={dy_dw}, ∂L/∂w={dL_dw}")

Gradient Descent — The Update Rule

How gradients are used to improve the model's weights

Gradient descent update rule: w_new = w_old − η · ∂L/∂w Where: η (eta) = learning rate (e.g. 0.01) ∂L/∂w = gradient of loss w.r.t. weight w From gradient.md example: w_old = 2.0 gradient = −12.0 η = 0.01 w_new = 2.0 − 0.01 × (−12.0) = 2.0 + 0.12 = 2.12 After update: y_new = 2.12 × 3 = 6.36 (closer to 8!) L_new = (6.36 − 8)² = 2.69 (down from 4.0!)
Python · Full training loop — gradient descent
import torch

w      = torch.tensor(2.0, requires_grad=True)
x      = torch.tensor(3.0)
y_true = torch.tensor(8.0)
eta    = 0.01   # learning rate

print(f"{'Step':>4}  {'w':>8}  {'pred':>8}  {'loss':>8}  {'grad':>8}")
print("-" * 50)

for step in range(20):
    # Forward pass
    pred = w * x
    loss = (pred - y_true)**2

    print(ff"{step:4d}  {w.item():8.4f}  {pred.item():8.4f}  {loss.item():8.4f}  {w.grad.item() if w.grad is not None else 0.0:8.4f}")

    # Backward pass
    loss.backward()

    # Weight update (gradient descent)
    with torch.no_grad():
        w -= eta * w.grad   # w = w - η · ∂L/∂w

    w.grad.zero_()          # must clear gradient before next backward()

print(ff"\nFinal weight: {w.item():.4f}  (target: {y_true.item()/x.item():.4f})")
# Should converge toward w = 8/3 ≈ 2.6667
Weight converging toward optimal value Step 0: w=2.00 pred=6.00 loss=4.00 grad=-12.00 (large, far from optimal) Step 1: w=2.12 pred=6.36 loss=2.69 grad=-9.84 Step 2: w=2.22 pred=6.65 loss=1.80 grad=-8.07 ... Step 10: w=2.52 pred=7.55 loss=0.20 grad=-2.70 ... Step 19: w=2.64 pred=7.93 loss=0.006 grad=-0.44 (near zero → near optimal) Loss decreasing → model improving each step

Multiple Weights — The Gradient Vector

Real networks have billions of weights — every one gets its own gradient

A real neural network has millions or billions of weights. Each weight has its own gradient, computed simultaneously during backpropagation. The collection of all these gradients forms the gradient vector (or gradient tensor, matching the shape of the weight tensor).

Each element of the gradient vector answers the question: "How much does the loss change if I increase this particular weight by a tiny amount, holding all other weights constant?"

Gradient vector — same shape as weight tensor Weight matrix W (4 weights): W = [ 0.42, -0.17, 1.38, 0.05] Gradient ∇L_W (also 4 values, same shape): ∇L = [-0.03, 0.91, -1.25, 0.02] Interpretation of each gradient: W[0] = 0.42: grad=-0.03 → tiny adjustment needed, slightly right direction W[1] = -0.17: grad=+0.91 → large positive → decrease this weight significantly W[2] = 1.38: grad=-1.25 → large negative → increase this weight significantly W[3] = 0.05: grad=+0.02 → nearly zero → already near optimal Update step (η=0.01): W_new = W - η · ∇L = [0.42+0.0003, -0.17-0.0091, 1.38+0.0125, 0.05-0.0002] = [0.4203, -0.1791, 1.3925, 0.0498]
Python · Gradient vectors in a real linear layer
import torch
import torch.nn as nn

# A single linear layer: maps 4 inputs to 2 outputs
layer = nn.Linear(in_features=4, out_features=2)

# Random input and target
x = torch.tensor([0.8, 0.2, 0.5, 0.9])
target = torch.tensor([1.0, 0.0])

# Forward pass
pred = layer(x)
loss = ((pred - target)**2).mean()

print(f"Weight matrix shape: {layer.weight.shape}")  # (2, 4)
print(f"Bias shape:          {layer.bias.shape}")     # (2,)
print(ff"Loss: {loss.item():.4f}")

# Backward pass — computes gradient for EVERY weight and bias
loss.backward()

print("\nGradient of weight matrix ∂L/∂W:")
print(layer.weight.grad.numpy().round(4))

print("\nGradient of bias ∂L/∂b:")
print(layer.bias.grad.numpy().round(4))

# Every weight in the 2×4 matrix has its own gradient value
# The gradient matrix has EXACTLY the same shape as the weight matrix
# This is the key property of backpropagation

total_params = sum(p.numel() for p in layer.parameters())
print(ff"\nTotal parameters: {total_params}  →  {total_params} gradient values computed")

Backpropagation

How gradients flow backward through the network to reach every weight

Backpropagation (backprop) is the algorithm that efficiently computes the gradient of the loss with respect to every weight in the network simultaneously. It applies the chain rule of calculus layer by layer, starting from the loss and working backward to the input.

Without backpropagation, computing gradients for a million-weight network would require a million separate forward passes. Backpropagation does it in just one backward pass, making deep learning computationally feasible.

Forward pass → Compute loss → Backward pass FORWARD PASS (left to right): x ──→ [Layer 1: z₁=W₁x+b₁, a₁=ReLU(z₁)] ──→ [Layer 2: z₂=W₂a₁+b₂, a₂=ReLU(z₂)] ──→ Loss ↑ Store activations a₁, a₂ at each step (needed for backward pass) BACKWARD PASS (right to left): ∂L/∂a₂ ← Layer 2 receives gradient from loss │ ├→ ∂L/∂W₂ = ∂L/∂z₂ · a₁ᵀ ← gradient for Layer 2 weights ├→ ∂L/∂b₂ = ∂L/∂z₂ ← gradient for Layer 2 bias └→ ∂L/∂a₁ = W₂ᵀ · ∂L/∂z₂ ← pass gradient to Layer 1 ∂L/∂a₁ ← Layer 1 receives gradient from Layer 2 │ ├→ ∂L/∂W₁ = ∂L/∂z₁ · xᵀ ← gradient for Layer 1 weights └→ ∂L/∂b₁ = ∂L/∂z₁ ← gradient for Layer 1 bias One backward pass computes ALL gradients simultaneously!
Python · Backprop through a 2-layer network
import torch
import torch.nn as nn
import torch.nn.functional as F

# 2-layer network: 4 → 8 → 2
model = nn.Sequential(
    nn.Linear(4, 8),
    nn.ReLU(),
    nn.Linear(8, 2)
)

x      = torch.randn(3, 4)   # batch of 3, 4 features each
target = torch.randint(0, 2, (3,))   # random class labels

# Forward pass
logits = model(x)
loss   = F.cross_entropy(logits, target)
print(ff"Loss: {loss.item():.4f}")

# Before backward: no gradients exist
print(f"Grad before backward: {model[0].weight.grad}")  # None

# Backward pass — computes ALL gradients in one call
loss.backward()

# After backward: every parameter has a gradient
print(f"\nLayer 0 (Linear 4→8):")
print(ff"  weight grad shape: {model[0].weight.grad.shape}")   # (8, 4)
print(ff"  bias grad shape:   {model[0].bias.grad.shape}")     # (8,)

print(f"\nLayer 2 (Linear 8→2):")
print(ff"  weight grad shape: {model[2].weight.grad.shape}")   # (2, 8)
print(ff"  bias grad shape:   {model[2].bias.grad.shape}")     # (2,)

total_grad_elements = sum(p.grad.numel() for p in model.parameters())
print(ff"\nTotal gradient elements: {total_grad_elements}")
# = 4×8 + 8 + 8×2 + 2 = 32 + 8 + 16 + 2 = 58

The Chain Rule

The mathematical foundation of backpropagation

The chain rule from calculus is the engine of backpropagation. It says: if z = f(y) and y = g(x), then the derivative of z with respect to x is the product of the individual derivatives:

Chain Rule: dz/dx = dz/dy · dy/dx For a neural network with multiple layers: L = loss(ŷ) ŷ = output(aₙ) aₙ = activation(zₙ) zₙ = W·aₙ₋₁ + b ∂L/∂W = ∂L/∂ŷ · ∂ŷ/∂aₙ · ∂aₙ/∂zₙ · ∂zₙ/∂W Expand each term: ∂L/∂ŷ = derivative of loss w.r.t. prediction ∂ŷ/∂aₙ = derivative of output activation ∂aₙ/∂zₙ = derivative of activation function (ReLU', GELU', etc.) ∂zₙ/∂W = aₙ₋₁ᵀ (previous layer activations)
Python · Chain rule step by step — manual vs autograd
import torch

# Computation: L = (ReLU(w*x + b) - y_true)^2
# Chain: L → relu_out → z → w

w      = torch.tensor(1.5, requires_grad=True)
x      = torch.tensor(2.0)
b      = torch.tensor(-1.0, requires_grad=True)
y_true = torch.tensor(3.0)

# ── Forward pass (store intermediates for manual chain rule) ──────
z         = w * x + b               # = 1.5*2 + (-1) = 2.0
relu_out  = torch.relu(z)           # = max(0, 2.0) = 2.0
loss      = (relu_out - y_true)**2  # = (2-3)^2 = 1.0

# ── Manual chain rule ─────────────────────────────────────────────
# ∂L/∂relu_out
dL_drelu  = 2 * (relu_out.item() - y_true.item())   # = 2*(2-3) = -2
# ∂relu_out/∂z  (ReLU derivative = 1 if z>0, else 0)
drelu_dz  = 1.0 if z.item() > 0 else 0.0           # = 1.0 (z=2>0)
# ∂z/∂w
dz_dw     = x.item()                                   # = 2.0
# ∂z/∂b
dz_db     = 1.0                                       # always 1

# Chain rule: multiply all pieces
manual_dL_dw = dL_drelu * drelu_dz * dz_dw   # = -2 * 1 * 2 = -4
manual_dL_db = dL_drelu * drelu_dz * dz_db   # = -2 * 1 * 1 = -2

# ── PyTorch autograd ──────────────────────────────────────────────
loss.backward()

print(ff"Manual  ∂L/∂w = {manual_dL_dw:.4f},  Autograd ∂L/∂w = {w.grad.item():.4f}")
print(ff"Manual  ∂L/∂b = {manual_dL_db:.4f},  Autograd ∂L/∂b = {b.grad.item():.4f}")
# Both should match: ∂L/∂w = -4.0,  ∂L/∂b = -2.0

Partial Derivatives

Computing how the loss changes with respect to one weight at a time

A partial derivative ∂L/∂wᵢ measures how the loss changes with respect to weight wᵢ, treating all other weights as constants. When we compute the gradient vector for a neural network, we are computing the partial derivative of the loss with respect to every single weight simultaneously.

The gradient vector ∇L = [∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ] points in the direction of steepest increase of the loss in the weight space. Subtracting it (scaled by η) moves weights toward lower loss.

Python · Partial derivatives — finite differences vs autograd
import torch

# Network with 3 weights: L = f(w0, w1, w2)
w = torch.tensor([0.5, -0.3, 1.2], requires_grad=True)
x = torch.tensor([1.0,  2.0, 3.0])
y = torch.tensor(5.0)

# Loss: L = (w·x − y)²  (dot product then MSE)
pred = (w * x).sum()
loss = (pred - y)**2
loss.backward()

print("Autograd partial derivatives:")
for i in range(3):
    print(ff"  ∂L/∂w{i} = {w.grad[i].item():.4f}")

# Verify with finite differences (numerical gradient)
print("\nNumerical partial derivatives:")
h = 1e-5
w_data = w.detach().clone()
for i in range(3):
    w_plus = w_data.clone(); w_plus[i] += h
    w_minus = w_data.clone(); w_minus[i] -= h
    L_plus  = ((w_plus  * x).sum() - y)**2
    L_minus = ((w_minus * x).sum() - y)**2
    numerical = (L_plus - L_minus) / (2*h)
    print(ff"  ∂L/∂w{i} ≈ {numerical.item():.4f}")
# Should match autograd values — if not, there's a bug!

PyTorch Autograd

How PyTorch automatically computes gradients for any computation graph

PyTorch's autograd engine builds a dynamic computation graph during the forward pass. Every tensor operation records how it was computed. When .backward() is called, autograd traverses this graph in reverse, applying the chain rule to compute gradients for every tensor that has requires_grad=True.

Key concepts

  • requires_grad=True: tells PyTorch to track this tensor in the computation graph
  • .backward(): traverses the computation graph backward, computing all gradients
  • .grad: the gradient is stored here after backward()
  • optimizer.zero_grad(): must clear gradients before each step (they accumulate by default)
  • torch.no_grad(): disable gradient tracking for inference (saves memory and compute)
Python · PyTorch autograd complete guide
import torch
import torch.nn as nn

# ── requires_grad: which tensors to track ─────────────────────────
x = torch.tensor(3.0)                         # NOT tracked (input data)
w = torch.tensor(2.0, requires_grad=True)   # TRACKED (model parameter)
b = torch.tensor(1.0, requires_grad=True)   # TRACKED

y = w * x + b   # = 7.0;  PyTorch records: "y came from w,x,b"
L = y**2        # = 49.0; PyTorch records: "L came from y"

print(ff"y.grad_fn: {y.grad_fn}")   # MulBackward — recorded operation
print(ff"L.grad_fn: {L.grad_fn}")   # PowBackward

L.backward()   # traverse graph backward, apply chain rule

print(ff"∂L/∂w = {w.grad}")   # 2y·x = 2·7·3 = 42
print(ff"∂L/∂b = {b.grad}")   # 2y·1 = 2·7 = 14

# ── Gradients ACCUMULATE — always zero before next backward() ─────
L2 = (w * x + b)**2
L2.backward()
print(ff"∂L/∂w after second backward (accumulated): {w.grad}")  # 42 + 42 = 84!
w.grad.zero_()   # ← MUST zero before next step

# ── torch.no_grad(): inference, no gradient tracking ──────────────
with torch.no_grad():
    pred_inference = w * x + b   # no graph built, no memory for gradients
print(ff"Inference pred: {pred_inference}")  # fast, no gradient overhead

# ── Gradient inspection hook ───────────────────────────────────────
model = nn.Linear(2, 1)
history = []

def save_grad(grad): history.append(grad.clone())
model.weight.register_hook(save_grad)

x_ex = torch.randn(4, 2)
y_ex = torch.randn(4, 1)

for _ in range(3):
    loss = ((model(x_ex) - y_ex)**2).mean()
    loss.backward()
    model.zero_grad()

print(ff"\nGradient magnitudes over 3 steps: {[h.norm().item():.4f for h in history]}")

Vanishing Gradients

When gradients shrink to zero in deep networks — and how to fix it

In a deep network, the chain rule multiplies gradients through every layer. If the per-layer gradient magnitude is consistently less than 1, the gradient shrinks exponentially as it passes through more layers — becoming effectively zero at the early layers. This is the vanishing gradient problem.

Early layers receive near-zero gradients and learn almost nothing. The model's capacity near the input is wasted.

Vanishing gradient in a 10-layer network: ∂L/∂W₁ = ∂L/∂W₁₀ × (∂a₁₀/∂a₉) × (∂a₉/∂a₈) × ... × (∂a₂/∂a₁) If each factor ≈ 0.5: 0.5^9 ≈ 0.002 (small) If each factor ≈ 0.1: 0.1^9 ≈ 10^{-9} (near zero!) If sigmoid: max |σ'| = 0.25, so per-step max = 0.25^9 ≈ 3.8 × 10^{-6}
🛤️

Solution 1: Residual Connections

Add skip connections that bypass layers: x_out = x_in + f(x_in). The gradient always has a direct path back: ∂L/∂x_in = ∂L/∂x_out · (1 + ∂f/∂x_in). The "+1" prevents shrinkage.

  • Used by: ResNet, all modern transformers
  • Gradient highway: direct path through "+1"
  • Enables 100+ layer networks
📊

Solution 2: Layer Normalisation

Normalise activations to zero mean and unit variance before each layer. This keeps activation magnitudes in a range where gradients are well-behaved. Used before every attention and FFN block in transformers.

  • Keeps activations from saturating
  • Prevents very large or very small gradients
  • Pre-norm (LLaMA) vs post-norm (BERT) variants
⚡

Solution 3: GELU / ReLU Activations

Unlike sigmoid/tanh (max derivative 0.25), ReLU has derivative exactly 1 for positive inputs and GELU has near-1 derivative for most inputs. No gradient attenuation in the positive regime.

  • ReLU: gradient = 1 for x > 0 (no shrinkage)
  • GELU: smooth, gradient near 1 for positive x
  • Never use sigmoid/tanh in deep hidden layers

Exploding Gradients

When gradients grow exponentially — and gradient clipping as the fix

The opposite of vanishing gradients: if per-layer gradient magnitudes exceed 1, they grow exponentially through the layers. The result is a catastrophically large gradient update that completely destabilises the weights — often producing NaN values within a few steps.

The standard fix is gradient clipping: if the global gradient norm exceeds a threshold, scale down all gradients proportionally so the norm equals the threshold. This caps the maximum update size without changing the direction of the update.

Python · Gradient clipping in practice
import torch
import torch.nn as nn

model = nn.Sequential(nn.Linear(4, 64), nn.ReLU(),
                      nn.Linear(64, 64), nn.ReLU(),
                      nn.Linear(64, 2))

optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

x = torch.randn(8, 4)
y = torch.randint(0, 2, (8,))

# Forward + backward
loss = nn.functional.cross_entropy(model(x), y)
loss.backward()

# Check gradient norm BEFORE clipping
total_norm_before = sum(p.grad.norm()**2
                        for p in model.parameters()
                        if p.grad is not None) ** 0.5
print(ff"Gradient norm before clipping: {total_norm_before.item():.4f}")

# ── Gradient clipping ─────────────────────────────────────────────
max_norm = 1.0   # clip if norm > 1.0 (standard for LLM training)
nn.utils.clip_grad_norm_(model.parameters(), max_norm=max_norm)

# Check gradient norm AFTER clipping
total_norm_after = sum(p.grad.norm()**2
                       for p in model.parameters()
                       if p.grad is not None) ** 0.5
print(ff"Gradient norm after clipping:  {total_norm_after.item():.4f}")
# If norm was > max_norm: now equals max_norm
# If norm was ≤ max_norm: unchanged

optimizer.step()
optimizer.zero_grad()

# In practice: clip_grad_norm_ is always called in LLM training
# Example from LLaMA training config:
#   torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
LLM Training Practice

Almost every LLM training run uses gradient clipping with max_norm=1.0. Without it, outlier batches with unusually high loss can produce enormous gradients that corrupt the model. The clipping threshold is almost always 1.0 — a standard that has proven robust across GPT, LLaMA, PaLM, and most other large models.

Learning Rate — Scaling the Gradient

How big a step to take in the direction of the negative gradient

The learning rate η controls how much each gradient update changes the weights. It is the most important hyperparameter in training. Too large: the model overshoots the minimum and diverges. Too small: training is extremely slow or gets stuck in local minima.

Rule of Thumb: 3e-4

A learning rate of 3×10⁻⁴ (0.0003) with AdamW is the empirical starting point for most transformer training. Adjust based on batch size: double the batch → try 2× learning rate (linear scaling rule).

Warmup Prevents Early Divergence

The first few hundred steps are the most unstable — gradients are largest and weights are most sensitive. Linearly ramping the learning rate from 0 to peak over ~2000 steps prevents large early updates from destabilising training.

Cosine Decay — Standard for LLMs

After warmup, decay the learning rate following a cosine curve to about 10% of peak. This allows large updates early (escaping bad local minima) and fine-grained updates late (converging precisely to a good minimum).

AdamW — Per-Parameter Adaptive LR

AdamW adapts the effective learning rate per parameter based on its gradient history. Parameters with consistently large gradients get smaller effective rates; sparse parameters get larger ones. Makes the global learning rate less sensitive.

Summary — The Complete Picture

From gradient definition to billion-parameter training

The Neural Network Training Loop Step 1: Forward Pass x → Layer₁ → Layer₂ → ... → Layerₙ → ŷ Store all activations (needed for backward pass) Step 2: Compute Loss L = loss(ŷ, y_true) ← e.g. CrossEntropy, MSE Step 3: Backward Pass (Backpropagation) loss.backward() ← applies chain rule from loss back to every weight ← stores ∂L/∂W in W.grad for every parameter W Step 4: Update Weights (Gradient Descent) clip_grad_norm_(model.params, 1.0) ← prevent exploding gradients optimizer.step() ← w ← w − η·∂L/∂w optimizer.zero_grad() ← clear for next step Repeat millions of times → model learns Gradient Interpretation: ∂L/∂w > 0 → decrease w (gradient descent: subtract positive → smaller) ∂L/∂w < 0 → increase w (gradient descent: subtract negative → larger) ∂L/∂w ≈ 0 → leave w unchanged (already near optimal)
Key Takeaways

A gradient tells every weight exactly how to change to reduce the loss. Backpropagation computes all gradients in one efficient backward pass using the chain rule. Vanishing gradients (prevented by residual connections and GELU/ReLU) and exploding gradients (prevented by gradient clipping) are the two failure modes to guard against. The learning rate scales each gradient update — too large diverges, too small stalls. These concepts underpin every neural network ever trained.
