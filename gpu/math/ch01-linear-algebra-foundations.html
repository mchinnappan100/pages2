01

Why Linear Algebra for LLMs?

The mathematical language every LLM speaks

When you type a word into an LLM, the very first thing that happens is that word gets turned into a list of numbers — a vector. Everything the model does from that point on — attention, feed-forward layers, output probabilities — is linear algebra. Understanding these operations is not just academic: it tells you why models behave the way they do.

Here is the full pipeline in linear algebra terms:

"hello" → token id 4521 ↓ Embedding lookup (row of weight matrix) v = [0.21, -0.34, 0.88, ..., 0.12] shape: (768,) ↓ Add positional encoding (vector addition) v' = v + p where p encodes position ↓ 12 transformer layers, each doing: QKV projections (matrix × vector) Attention scores (dot products) FFN (matrix × vector + nonlinearity) ↓ Final linear layer (matrix × vector) logits: one score per vocabulary token shape: (32000,) ↓ Softmax (converts to probabilities) P("world") = 0.34, P("there") = 0.18, …
Key Insight

Every layer of an LLM is just a composition of linear maps (matrices) and nonlinearities. Linear algebra is the entire vocabulary of neural network computation.

Scalars, Vectors & Notation

The basic objects we work with

Scalar

A single number. Examples: temperature 98.6, learning rate 0.001, loss value 2.34. Written as a plain lowercase letter: x = 3.14

Vector

An ordered list of numbers. Think of it as a point in space, or as a direction with magnitude. Written as a bold lowercase letter: v.

A 3-dimensional vector: v = [2, -1, 4] means "go 2 steps right, 1 step back, 4 steps up." In LLMs, vectors have hundreds or thousands of dimensions — but the same intuition applies.

Why ordered lists?

Because the position matters. [2, -1, 4] and [-1, 4, 2] are different vectors even though they contain the same numbers.

Column vector (n × 1) ⎡ 2 ⎤ v = ⎢ -1 ⎥ ← n = 3 dimensions ⎣ 4 ⎦ Each number is a component. v[0] = 2, v[1] = -1, v[2] = 4 In Python / NumPy: import numpy as np v = np.array([2, -1, 4]) print(v.shape) # (3,)
Geometric intuition (2D) ↑ y │ 3 │ ● (2, 3) │ ↗ 2 │ / │ / 1 │ / │ / 0 └──────────→ x 0 1 2 Vector [2, 3] is an arrow from origin (0,0) to (2,3). LLM token embeddings are the same idea but in 768 dimensions instead of 2.

Vector Operations

Addition, subtraction, and scalar multiplication — the basic building blocks

Vector Addition

Add corresponding components. Geometrically: place the second vector at the tip of the first — the sum is the arrow from start to the new tip.

LLM use: Positional encoding is added to token embeddings: x' = x_embedding + x_positional

Scalar Multiplication

Multiply every component by the same number. Scales the vector's length without changing its direction (unless scalar is negative, which flips it).

LLM use: The attention scale factor 1/√d_k is scalar multiplication on the attention scores.

Python · Vector Operations
import numpy as np

# Two token embedding vectors (simplified to 4 dims)
token_embed    = np.array([0.5, -0.2,  0.8, 0.1])
pos_encoding   = np.array([0.1,  0.3, -0.1, 0.4])

# Addition: combine token meaning + position
x = token_embed + pos_encoding
# → [0.6, 0.1, 0.7, 0.5]

# Scalar multiplication: scale by 1/sqrt(d_k)
d_k = 4
scale = 1 / np.sqrt(d_k)    # = 0.5
scaled = x * scale           # each element multiplied by 0.5

# Subtraction: compute difference between two embeddings
king   = np.array([0.9,  0.1, 0.8])
queen  = np.array([0.8,  0.9, 0.8])
man    = np.array([0.9, -0.1, 0.2])

# Famous word2vec analogy: king - man + woman ≈ queen
woman = np.array([0.8, 0.85, 0.2])
result = king - man + woman
print(result)   # ≈ [0.8, 0.95, 0.8] — close to queen!
Intuition: Word Embeddings

The famous king − man + woman ≈ queen result works because word embeddings capture semantic relationships as directions in vector space. "Gender" is a consistent direction; "royalty" is another. This is pure vector arithmetic.

Dot Product

The most important operation in all of deep learning

The dot product (also called inner product) of two vectors is a single number that captures how much they "agree" — how aligned they are.

For vectors a = [a₁, a₂, ..., aₙ] and b = [b₁, b₂, ..., bₙ]:

a · b = a₁b₁ + a₂b₂ + ... + aₙbₙ = Σᵢ aᵢbᵢ Also written as: a · b = ‖a‖ ‖b‖ cos(θ) where θ is the angle between the two vectors.

What does the dot product tell us?

  • Large positive → vectors point in the same direction → they are similar
  • Zero → vectors are perpendicular → completely unrelated
  • Negative → vectors point in opposite directions → they are dissimilar

Dot product in LLMs — Attention

The attention score between query q and key k is literally a dot product: score = q · k / √d_k. A high score means "this query is asking about something very similar to what this key represents" — so it should attend strongly to it.

Python · Dot product and attention score
import numpy as np

# Two 4-dimensional vectors
a = np.array([1.0, 2.0, 3.0, 4.0])
b = np.array([1.0, 0.5, 0.5, 1.0])

# Manual dot product: multiply pairwise, then sum
dot = sum(ai * bi for ai, bi in zip(a, b))
# = 1×1 + 2×0.5 + 3×0.5 + 4×1 = 1 + 1 + 1.5 + 4 = 7.5

# NumPy shorthand
dot_np = np.dot(a, b)       # 7.5
dot_einsum = a @ b          # 7.5 (@ operator)

# Cosine similarity (normalise first, then dot product)
def cosine_sim(x, y):
    return np.dot(x, y) / (np.linalg.norm(x) * np.linalg.norm(y))

# Attention score between query q and key k
d_k = 4
q = np.array([0.8, 0.2, 0.5, 0.9])   # query vector
k = np.array([0.7, 0.3, 0.4, 0.8])   # key vector

score = np.dot(q, k) / np.sqrt(d_k)
print(f"Attention score: {score:.4f}")  # how much q attends to k

Norms & Distance

Measuring the length of a vector and the distance between two vectors

A norm measures the "length" or "size" of a vector. The most common is the L2 norm (Euclidean length).

L2 norm: ‖v‖₂ = √(v₁² + v₂² + ... + vₙ²) L1 norm: ‖v‖₁ = |v₁| + |v₂| + ... + |vₙ| L∞ norm: ‖v‖∞ = max(|v₁|, |v₂|, ..., |vₙ|) Euclidean distance between vectors a and b: d(a, b) = ‖a − b‖₂ = √(Σᵢ (aᵢ − bᵢ)²)

Why norms matter in LLMs

  • Layer Normalisation — before each transformer block, activations are divided by their norm to keep values in a stable range.
  • Weight decay — the L2 norm of weight matrices is penalised during training to prevent overfitting.
  • Cosine similarity — used to compare embeddings: cos(θ) = (a·b) / (‖a‖ ‖b‖). Normalise both vectors to unit length, then their dot product equals cosine similarity.
  • RMSNorm (used in LLaMA/GPT) — divides by the root-mean-square of the vector: ‖v‖_rms = √(mean(v²))
Python · Norms, normalisation, RMSNorm
import numpy as np

v = np.array([3.0, 4.0])   # classic 3-4-5 right triangle

l2 = np.linalg.norm(v)          # 5.0  = √(9 + 16)
l1 = np.linalg.norm(v, ord=1)   # 7.0  = 3 + 4

# Normalise to unit vector (length = 1)
v_unit = v / np.linalg.norm(v)   # [0.6, 0.8]
print(np.linalg.norm(v_unit))    # exactly 1.0

# RMSNorm (used in LLaMA, GPT-NeoX)
def rms_norm(x, eps=1e-8):
    rms = np.sqrt(np.mean(x**2) + eps)
    return x / rms

x = np.array([1.0, 2.0, 3.0, 4.0])
print(rms_norm(x))   # [0.365, 0.730, 1.095, 1.461]

# Cosine similarity between two embeddings
cat = np.array([0.9, 0.1, 0.8])
dog = np.array([0.8, 0.2, 0.7])
car = np.array([0.1, 0.9, 0.1])

def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(f"cat vs dog: {cosine(cat, dog):.3f}")  # high: similar animals
print(f"cat vs car: {cosine(cat, car):.3f}")  # lower: different domains

Vector Spaces & Linear Independence

The geometry of high-dimensional embedding spaces

Vector Space

A vector space is a collection of vectors where you can add any two vectors together and multiply any vector by a scalar — and the result is always still in the space. The embedding space of an LLM is a vector space: ℝ⁷⁶⁸ (or ℝ⁴⁰⁹⁶ for larger models).

Linear Independence

A set of vectors is linearly independent if no vector in the set can be written as a combination of the others. This matters for embeddings: if all word vectors were linearly dependent, you could express all of them with just one vector — and the model would have no way to distinguish between words.

Basis

A basis is a minimal set of linearly independent vectors that spans the whole space. ℝⁿ has a basis of n vectors. In 768-dimensional embedding space, every token embedding can be written as a weighted sum of 768 basis vectors.

Span

The span of a set of vectors is all points you can reach by combining them. If your embedding model only captures 10 "concepts," all embeddings lie in a 10-dimensional subspace of the full 768-dimensional space.

📐

Subspaces in LLMs

Research shows LLM embeddings have meaningful subspaces. The "gender subspace" captures male/female relationships; the "tense subspace" captures past/present/future.

  • World knowledge is encoded in different subspaces
  • Linear probes can extract facts from specific dimensions
  • Attention heads often specialise in specific subspaces
  • LoRA fine-tuning updates low-rank subspaces only
🎯

Orthogonality

Two vectors are orthogonal (perpendicular) if their dot product is zero. Orthogonal vectors are maximally "different" — knowing one tells you nothing about the other.

  • Orthogonal attention heads can specialise independently
  • Unrelated concepts should ideally be orthogonal in embedding space
  • Softmax pushes token probabilities to be spread across the vocabulary
  • QR decomposition produces orthogonal basis vectors

Chapter Summary

Vectors are the atoms of everything in LLMs. Every token, every layer output, every attention query, key, and value is a vector. The dot product measures similarity between vectors. Norms measure magnitudes. These three ideas — vectors, dot products, norms — appear in every single layer of every transformer ever built.
