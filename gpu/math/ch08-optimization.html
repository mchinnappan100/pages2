Optimization &
Adam

LLM training is an optimization problem: find the weights that minimize the cross-entropy loss over trillions of tokens. Gradient descent is the strategy; Adam is the algorithm that makes it work at scale. Understanding Adam means understanding why each design choice — momentum, adaptive learning rates, weight decay — exists.

Gradient Descent

The core idea: move weights in the direction that reduces the loss

Gradient descent updates each weight by subtracting a small multiple of its gradient. The gradient points uphill (toward higher loss), so subtracting it moves downhill (toward lower loss). The learning rate η controls the step size.

Intuition: imagine you are blindfolded on a hilly landscape and want to find the valley. You can feel the slope under your feet (gradient). Each step, you move in the steepest downhill direction. That's gradient descent.

Gradient descent update rule: θ ← θ - η · ∇L(θ) Where: θ = all weights (billions of parameters for an LLM) η = learning rate (e.g. 3e-4 for AdamW on LLMs) ∇L(θ) = gradient of loss with respect to all weights Convergence condition: ∇L(θ) → 0 (gradient becomes zero at a minimum)
02

Stochastic & Mini-batch Gradient Descent

Why we use batches — not the full dataset — for each update

Full-batch gradient descent: compute the gradient over the entire dataset. Exact but impossibly slow for trillion-token datasets.

Stochastic gradient descent (SGD): compute gradient on a single example. Fast but very noisy — gradient from one example is a poor estimate of the true gradient.

Mini-batch gradient descent: use a batch of B examples. The batch gradient is an unbiased estimate of the true gradient with variance reduced by a factor of B. This is what every LLM uses.

For LLMs: typical batch size is 256–2048 sequences × 2048 tokens = ~500K–4M tokens per step. This is "global batch size"; in distributed training it's split across thousands of GPUs.

Mini-batch gradient estimate: ĝ = (1/B) Σᵢ₌₁ᴮ ∇L(θ; xᵢ) ← average over B examples Properties: E[ĝ] = ∇L(θ) ← unbiased estimate of true gradient Var(ĝ) = Var(∇L) / B ← variance shrinks with batch size Why not use batch_size=∞? Memory: each example requires activations in memory during forward pass Diminishing returns: 2B examples halves variance but costs 2× compute Noise can help: stochastic noise helps escape sharp local minima

Momentum & RMSProp

Two ideas that led to Adam

Momentum

Plain SGD oscillates and is slow in narrow valleys. Momentum adds a "velocity" term — rather than stepping purely based on the current gradient, it accumulates past gradients (like a ball rolling downhill building up speed). This smooths oscillations and accelerates convergence in the right direction.

RMSProp

RMSProp adapts the learning rate per parameter based on the history of squared gradients. Parameters that consistently receive large gradients get smaller effective learning rates; parameters with small gradients get larger ones. This prevents sparse gradients (like word embeddings for rare words) from being updated by too-small steps.

Momentum: v ← β·v + (1-β)·g ← exponential moving average of gradients θ ← θ - η·v RMSProp: s ← β·s + (1-β)·g² ← EMA of squared gradients θ ← θ - η · g / (√s + ε) ← scale by inverse RMS of gradient history
04

Adam & AdamW

The optimizer that trains almost every LLM — momentum + RMSProp combined

Adam (Adaptive Moment Estimation) combines momentum (first moment) and RMSProp (second moment). For each parameter, it maintains a running average of the gradient AND of the squared gradient, and adapts the step size accordingly.

AdamW fixes a subtle bug in Adam: it decouples weight decay from the adaptive gradient update. In Adam, L2 regularisation interacts with the adaptive scaling and doesn't actually regularise correctly. AdamW applies weight decay directly to the weights, not through the gradient. All modern LLMs use AdamW.

Adam update (at step t): m ← β₁·m + (1-β₁)·g ← first moment (gradient EMA) v ← β₂·v + (1-β₂)·g² ← second moment (squared gradient EMA) m̂ = m / (1 - β₁ᵗ) ← bias correction (important early in training) v̂ = v / (1 - β₂ᵗ) θ ← θ - η · m̂ / (√v̂ + ε) AdamW adds: θ ← θ · (1 - η·λ) - η · m̂ / (√v̂ + ε) ↑ weight decay applied directly, not through gradient Typical hyperparameters for LLMs: β₁ = 0.9 (gradient momentum) β₂ = 0.95 (squared gradient momentum; sometimes 0.999) ε = 1e-8 (numerical stability) η = 3e-4 (peak learning rate, decayed by cosine schedule) λ = 0.1 (weight decay)
Python · AdamW from scratch
import numpy as np

class AdamW:
    def __init__(self, lr=3e-4, beta1=0.9, beta2=0.95, eps=1e-8, weight_decay=0.1):
        self.lr = lr; self.b1 = beta1; self.b2 = beta2
        self.eps = eps; self.wd = weight_decay
        self.m = {}; self.v = {}; self.t = 0

    def step(self, params, grads):
        self.t += 1
        for i, (p, g) in enumerate(zip(params, grads)):
            if i not in self.m:
                self.m[i] = np.zeros_like(p)   # first moment
                self.v[i] = np.zeros_like(p)   # second moment

            # Update biased moment estimates
            self.m[i] = self.b1 * self.m[i] + (1 - self.b1) * g
            self.v[i] = self.b2 * self.v[i] + (1 - self.b2) * g**2

            # Bias correction
            m_hat = self.m[i] / (1 - self.b1**self.t)
            v_hat = self.v[i] / (1 - self.b2**self.t)

            # AdamW: weight decay applied directly, then gradient step
            p *= (1 - self.lr * self.wd)               # weight decay
            p -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)  # gradient step

# Usage
W = np.random.randn(768, 768) * 0.02
optimizer = AdamW(lr=3e-4)
for step in range(100):
    grad = np.random.randn(*W.shape) * 0.01   # simulated gradient
    optimizer.step([W], [grad])

Learning Rate Schedules

Why the learning rate must change during training

Using a fixed learning rate throughout training is suboptimal. A large learning rate at the start causes instability; a large learning rate at the end prevents convergence to a good minimum. The solution: schedule the learning rate.

Warmup + Cosine Decay

Used by GPT, LLaMA, and virtually all modern LLMs:

  • Warmup phase (first ~2000 steps): linearly increase lr from near-zero to peak. This prevents large gradient updates from distorting the model before it has stabilised.
  • Cosine decay: smoothly decrease from peak lr to a small fraction (typically 10% of peak) following a cosine curve.
Python · Cosine LR schedule with warmup
import numpy as np

def cosine_lr_schedule(step, warmup_steps, total_steps, lr_max, lr_min):
    """
    Linear warmup then cosine decay.
    Used by GPT-3, LLaMA, and most modern LLM training runs.
    """
    if step < warmup_steps:
        # Linear warmup: 0 → lr_max over warmup_steps
        return lr_max * step / warmup_steps
    else:
        # Cosine decay: lr_max → lr_min over remaining steps
        progress = (step - warmup_steps) / (total_steps - warmup_steps)
        cosine   = 0.5 * (1 + np.cos(np.pi * progress))
        return lr_min + (lr_max - lr_min) * cosine

# Example: 100K steps training run
total_steps = 100_000
warmup_steps = 2_000
lr_max = 3e-4
lr_min = 3e-5   # 10% of peak

steps = np.arange(total_steps)
lrs   = [cosine_lr_schedule(s, warmup_steps, total_steps, lr_max, lr_min)
         for s in steps]

print(ff"Step 0:      lr = {lrs[0]:.2e}")       # near zero
print(ff"Step 2000:   lr = {lrs[2000]:.2e}")    # peak
print(ff"Step 50000:  lr = {lrs[50000]:.2e}")   # halfway down
print(ff"Step 100000: lr = {lrs[-1]:.2e}")     # at minimum

Loss Landscape & Convergence

What the optimization problem looks like geometrically

The loss landscape of an LLM is a surface in billions of dimensions where the height is the loss value. Training navigates this landscape looking for a low point (good model). Key features:

  • Local minima — in high dimensions, almost all local minima are nearly as good as global minima (empirical finding). LLMs don't get stuck in bad local minima.
  • Saddle points — points where the gradient is zero but it's not a minimum. Gradient descent can slow near saddle points; Adam's adaptive scaling helps escape them.
  • Sharp vs flat minima — flat minima generalise better (the loss is low over a wide region). Techniques like weight averaging (EMA of weights) and larger batch sizes tend toward flat minima.
  • Loss spikes — sudden loss increases during training, often caused by outlier batches. Gradient clipping and learning rate warmup help prevent these.
Chapter Summary

LLM training is AdamW + cosine LR schedule + gradient clipping. Adam combines momentum (smooth gradient direction) with adaptive per-parameter learning rates (scale by inverse gradient history). AdamW adds proper weight decay. The cosine schedule warms up learning rate to prevent early instability, then decays it to converge to a flat minimum. These three components together make training stable and efficient at billion-parameter scale.
