Probability Basics

What probability means and the axioms it must satisfy

A probability is a number between 0 and 1 that expresses how likely an event is. P(event) = 0 means impossible; P(event) = 1 means certain.

For a language model, the key events are: "the next token is word X." The model assigns a probability to every word in the vocabulary (50,000+ words), and these must all sum to exactly 1.0 — this is enforced by the softmax function.

Three axioms of probability

  • Non-negativity: P(A) ≥ 0 for any event A
  • Normalisation: P(entire sample space) = 1
  • Additivity: P(A or B) = P(A) + P(B) if A and B can't both happen
Discrete probability distribution: P(X = x) for each possible value x Must satisfy: Σₓ P(X = x) = 1 and P(X = x) ≥ 0 Language model next-token distribution: P(next = "cat" | context) = 0.12 P(next = "dog" | context) = 0.08 P(next = "the" | context) = 0.31 ... Sum over all 50,257 vocabulary tokens = 1.0
Python · Softmax converts raw scores to probabilities
import numpy as np

# Raw logit scores from the LLM's final layer (unnormalized)
logits = np.array([2.1, 0.8, -0.5, 1.4, 3.2])

# Softmax: converts to probabilities that sum to 1
def softmax(x):
    # Subtract max for numerical stability (prevents overflow)
    e = np.exp(x - x.max())
    return e / e.sum()

probs = softmax(logits)
print("Probabilities:", np.round(probs, 4))
print("Sum:", probs.sum())   # exactly 1.0
# → [0.1762, 0.0589, 0.0162, 0.1087, 0.6399]
#   The 5th token ("the") gets ~64% probability

# Sample from this distribution (not greedy decode)
next_token = np.random.choice(5, p=probs)
print(f"Sampled token index: {next_token}")

Conditional Probability

The probability of X given that we know Y — the essence of language modelling

The conditional probability P(A|B) is the probability of event A, given that we already know B happened. It updates our belief about A based on new information.

This is the mathematical heart of language modelling. An LLM computes:

P(next token | all previous tokens)

Given the context "The cat sat on the ___", the LLM computes a conditional probability distribution over every possible next word, conditioned on having seen "The cat sat on the" as context.

Definition: P(A | B) = P(A and B) / P(B) Product rule (chain rule): P(A and B) = P(A | B) · P(B) Language model chain rule: P(w₁, w₂, w₃, ..., wₙ) = P(w₁) · P(w₂|w₁) · P(w₃|w₁,w₂) · ... · P(wₙ|w₁...wₙ₋₁) This means: the probability of a sentence = product of each token's probability given all previous tokens. This is exactly what LLMs model!
Python · Sentence probability using the chain rule
import numpy as np

# Simulate conditional probabilities for a tiny vocabulary
# P(next | context) for each position
# Sentence: "The cat sat"

p_the        = 0.05     # P("The")           — starts many sentences
p_cat_the    = 0.02     # P("cat" | "The")
p_sat_thecat = 0.30     # P("sat" | "The cat")

# Chain rule: sentence probability = product of conditionals
p_sentence = p_the * p_cat_the * p_sat_thecat
print(ff"P('The cat sat') = {p_sentence:.6f}")

# Log probability (to avoid underflow with long sequences)
log_p = np.log(p_the) + np.log(p_cat_the) + np.log(p_sat_thecat)
print(ff"Log P('The cat sat') = {log_p:.4f}")

# Per-token cross-entropy loss (what LLMs minimise during training)
# = -log P(correct next token | context)
loss = -log_p / 3   # average over 3 tokens
print(ff"Average cross-entropy loss: {loss:.4f}")

Bayes' Theorem

How to update beliefs when new evidence arrives

Bayes' theorem is one of the most important equations in all of statistics and AI. It tells you how to update your probability estimate for hypothesis H when you observe evidence E.

Bayes' Theorem: P(H | E) = P(E | H) · P(H) / P(E) Where: P(H) = prior probability of H (before seeing evidence) P(E | H) = likelihood: how probable is this evidence if H is true? P(E) = marginal probability of evidence (normalising constant) P(H | E) = posterior: updated probability after seeing evidence In language modelling: H = a hypothesis about meaning/intent E = the tokens in the prompt P(H|E) = what the model "believes" given the context

Bayesian thinking in LLMs

  • Prompt as evidence — each new token in a prompt updates the model's distribution over possible next tokens. This is Bayesian updating in action.
  • In-context learning — few-shot examples shift the model's "prior" about what task to perform. Adding "Q: ... A: ..." examples changes the posterior over the format of the response.
  • RLHF — the reward model learns P(human prefers this response | prompt, response). The policy is then updated toward responses with high posterior probability of being preferred.

Expectation & Variance

The average behaviour of a random variable — and how spread out it is

The expectation (or expected value) E[X] is the probability-weighted average of all possible values of X. It answers: "what value would I get on average if I ran this random process many times?"

The variance Var(X) measures how spread out the distribution is around its mean.

Expectation (discrete): E[X] = Σₓ x · P(X = x) Variance: Var(X) = E[(X - E[X])²] = E[X²] - (E[X])² Standard deviation: σ = √Var(X) Linearity of expectation: E[aX + b] = a·E[X] + b E[X + Y] = E[X] + E[Y] (always true, even for dependent X,Y) LLM connections: E[loss over dataset] = (1/N) Σᵢ loss(xᵢ) ← training minimises this E[output token] = Σ_vocab token · P(token|context) ← not used directly Var(gradients) = key concern for stable training
Python · Expectation and variance of LLM outputs
import numpy as np

# Token probabilities from a small vocabulary
vocab = ["cat", "dog", "the", "sat"]
probs = np.array([0.3, 0.2, 0.4, 0.1])

# Expected loss if we assign these probabilities to the true next token
# Cross-entropy = -E[log P(correct token)]
log_probs = np.log(probs)
entropy = -np.sum(probs * log_probs)
print(ff"Distribution entropy: {entropy:.4f}")   # uncertainty measure

# Variance of training losses (want this to be small = stable training)
losses = np.array([2.3, 1.9, 2.1, 3.5, 2.0])
print(ff"Mean loss: {losses.mean():.4f}")
print(ff"Loss variance: {losses.var():.4f}")
print(ff"Loss std dev: {losses.std():.4f}")

# Monte Carlo estimate of expected value via sampling
samples = np.random.choice([0, 1, 2, 3], size=10000, p=probs)
mc_estimate = np.mean(samples)
true_expectation = np.dot(np.arange(4), probs)   # = 0×0.3 + 1×0.2 + 2×0.4 + 3×0.1
print(ff"MC estimate: {mc_estimate:.4f}, True E[X]: {true_expectation:.4f}")

Maximum Likelihood Estimation (MLE)

The mathematical justification for why we minimise cross-entropy loss

MLE asks: given the data we observed, what parameter values make that data most probable? For a language model, this means: find weights θ that maximise P(training text | θ).

Maximising likelihood is equivalent to minimising negative log-likelihood — which is exactly the cross-entropy loss used to train every LLM.

Maximum Likelihood: θ* = argmax_θ P(data | θ) = argmax_θ Π_i P(xᵢ | θ) (assuming i.i.d. samples) = argmax_θ Σ_i log P(xᵢ | θ) (log turns product → sum) = argmin_θ -Σ_i log P(xᵢ | θ) (flip sign: minimise NLL) For a language model on a sequence of tokens w₁...wₙ: NLL = -Σᵢ log P(wᵢ | w₁, ..., wᵢ₋₁; θ) This IS the cross-entropy training loss. MLE = cross-entropy minimisation.
Python · MLE and cross-entropy loss
import numpy as np

def softmax(x):
    e = np.exp(x - x.max())
    return e / e.sum(axis=-1, keepdims=True)

# Logits from LLM for 3 tokens, vocab size=5
logits = np.array([[2.1, 0.8, 3.2, -0.5, 1.4],
                   [0.3, 2.9, 0.1,  1.2, 0.5],
                   [1.5, 0.2, 0.8, 2.8, 0.4]])

# True next tokens (ground truth from training data)
targets = np.array([2, 1, 3])   # indices of correct tokens

# Step 1: Convert logits to probabilities
probs = softmax(logits)   # shape (3, 5)

# Step 2: Extract probability assigned to the CORRECT token
correct_probs = probs[np.arange(3), targets]
print("P(correct token):", np.round(correct_probs, 4))

# Step 3: Cross-entropy loss = -mean(log P(correct))
# = negative log-likelihood = what we minimise during training
ce_loss = -np.mean(np.log(correct_probs + 1e-10))
print(ff"Cross-entropy loss: {ce_loss:.4f}")

# Perplexity = exp(cross-entropy) — used to compare LLMs
perplexity = np.exp(ce_loss)
print(ff"Perplexity: {perplexity:.4f}")

Chapter Summary

An LLM is a conditional probability distribution P(next token | context). Training it is MLE — maximise P(training data) which equals minimise cross-entropy loss. Inference is sampling from this distribution. Every probability concept from this chapter maps directly to a concrete part of the training or inference pipeline.
