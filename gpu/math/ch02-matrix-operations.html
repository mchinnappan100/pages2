What Is a Matrix?

A rectangular array of numbers with rows and columns

A matrix is a 2-dimensional array of numbers arranged in rows and columns. A matrix with m rows and n columns has shape (m × n). Each element is identified by its row and column index.

In LLMs, matrices are everywhere: the embedding table is a matrix of shape (vocab_size × d_model), the weight matrix of a linear layer is (d_in × d_out), the attention score matrix is (seq_len × seq_len).

Matrix notation column 0 column 1 column 2 row 0 ⎡ 2 7 1 ⎤ row 1 ⎢ 3 4 8 ⎥ ← shape (3 × 3): 3 rows, 3 columns row 2 ⎣ 9 1 5 ⎦ A[1, 2] = 8 (row 1, column 2) Embedding table shape: (50257 × 768) 50257 rows = one per vocabulary token 768 cols = embedding dimension A[4521, :] = the embedding vector for token 4521

Basic Matrix Operations

Addition, subtraction, and scalar multiplication

Matrix addition: add corresponding elements. Only works if both matrices have the same shape.

Scalar multiplication: multiply every element by the same number.

Element-wise multiplication (Hadamard): multiply corresponding elements. Symbol: A ⊙ B. Same shape required. Used in gated activations like SwiGLU and in the forget/input gates of LSTMs.

Python · Matrix basics
import numpy as np

A = np.array([[1, 2],
              [3, 4]])

B = np.array([[5, 6],
              [7, 8]])

# Addition  (element-wise)
C = A + B          # [[6,8],[10,12]]

# Scalar multiplication
D = 3 * A           # [[3,6],[9,12]]

# Hadamard (element-wise) product  — used in SwiGLU gating
E = A * B           # [[5,12],[21,32]]

# SwiGLU: output = SiLU(gate) ⊙ value
gate  = np.array([1.5, -0.5, 2.0])
value = np.array([0.8,  1.2, 0.3])
silu  = gate * (1 / (1 + np.exp(-gate)))   # SiLU activation
swiglu_out = silu * value                      # element-wise gate

Matrix Multiplication

The most compute-intensive operation in every LLM forward pass

Matrix multiplication (matmul) takes two matrices A (m×k) and B (k×n) and produces C (m×n). The inner dimension k must match. Each element C[i,j] is the dot product of row i of A with column j of B.

This is not element-wise multiplication. It is a fundamentally different operation that mixes information across all dimensions.

C = A × B C[i, j] = Σₖ A[i, k] · B[k, j] = (row i of A) · (col j of B) Shapes: A is (m × k), B is (k × n) → C is (m × n) Example: (4 × 768) × (768 × 3072) → (4 × 3072) ↑ 4 tokens ↑ expanded FFN dimension
Visual: how C[1,2] is computed A (2×3) B (3×2) C (2×2) ⎡ 1 2 3 ⎤ × ⎡ 7 8 ⎤ = ⎡ ? ? ⎤ ⎣ 4 5 6 ⎦ ⎢ 9 10⎥ ⎣ ? C[1,2]⎦ ⎣11 12⎦ C[1,2] = row 1 of A · col 2 of B = [4,5,6] · [8,10,12] = 4×8 + 5×10 + 6×12 = 32 + 50 + 72 = 154
Python · Matrix multiplication — the transformer's core operation
import numpy as np

# Simple 2×3 × 3×2 example
A = np.array([[1, 2, 3],
              [4, 5, 6]])       # shape (2, 3)

B = np.array([[7,  8],
              [9,  10],
              [11, 12]])         # shape (3, 2)

C = A @ B                         # shape (2, 2)
# C = [[58, 64], [139, 154]]

# Transformer linear layer: project tokens through weight matrix
batch_size, seq_len, d_model = 2, 4, 8
d_ff = 32   # feed-forward expansion (typically 4× d_model)

tokens = np.random.randn(batch_size, seq_len, d_model)   # (2,4,8)
W1     = np.random.randn(d_model, d_ff)                   # (8,32)

# Batch matmul: each of the 2×4=8 token vectors is projected
hidden = tokens @ W1   # (2,4,8) @ (8,32) → (2,4,32)
print(hidden.shape)    # (2, 4, 32)

# Einsum notation (used in many transformer implementations)
# 'bsd,df->bsf' = batch,seq,d_model × d_model,d_ff → batch,seq,d_ff
hidden2 = np.einsum('bsd,df->bsf', tokens, W1)
print(np.allclose(hidden, hidden2))   # True
Shape Rule — Critical to Remember

For A @ B to work, A's last dimension must equal B's second-to-last dimension. In PyTorch/NumPy: A has shape (..., m, k) and B has shape (..., k, n), giving result shape (..., m, n). Getting this wrong is the #1 source of shape errors in transformer code.

Transpose & Inverse

Flipping a matrix and undoing a linear transformation

Transpose

The transpose of matrix A (written Aᵀ) flips rows and columns: element [i,j] becomes element [j,i]. A matrix of shape (m×n) becomes (n×m).

LLM use: In attention, the key matrix K is transposed to compute scores: scores = Q @ Kᵀ. This pairs each query row (shape d_k) with each key row (shape d_k) via dot product.

Symmetric Matrices

A matrix is symmetric if A = Aᵀ. Attention score matrices tend to be asymmetric (Q and K differ), but covariance matrices in statistics are always symmetric.

Inverse

The inverse A⁻¹ satisfies A × A⁻¹ = I (identity matrix). Only square matrices can have inverses, and not all do. In practice, LLMs almost never compute matrix inverses directly — they use gradient descent instead. But the concept of "undoing a transformation" is fundamental to understanding what a network learns.

Python · Transpose and attention scores
import numpy as np

# Transpose
A = np.array([[1, 2, 3],
              [4, 5, 6]])  # shape (2, 3)

At = A.T           # shape (3, 2) — rows and cols swapped
# [[1, 4], [2, 5], [3, 6]]

# Attention: Q @ K^T → score matrix
seq_len, d_k = 5, 4
Q = np.random.randn(seq_len, d_k)   # (5, 4)  queries
K = np.random.randn(seq_len, d_k)   # (5, 4)  keys

# Score[i,j] = how much token i attends to token j
scores = Q @ K.T / np.sqrt(d_k)     # (5,4) @ (4,5) → (5,5)
print(scores.shape)                 # (5, 5)

# Verify: scores[2, 3] = dot(Q[2], K[3]) / sqrt(d_k)
manual = np.dot(Q[2], K[3]) / np.sqrt(d_k)
print(np.isclose(scores[2, 3], manual))  # True

Broadcasting

How NumPy and PyTorch handle operations on different-shaped tensors

Broadcasting allows operations between tensors of different shapes by automatically "expanding" the smaller tensor to match the larger one — without actually copying data. This is essential for adding biases, applying masks, and scaling attention scores across batches.

Broadcasting rules

  • Align shapes from the right (trailing dimensions)
  • A dimension of size 1 can be broadcast to match any size
  • Missing leading dimensions are treated as size 1
Broadcasting examples Shape (4, 3) + shape (3,) → (4, 3) The (3,) vector is added to every row of the (4,3) matrix. This is exactly how bias vectors are added in linear layers! Shape (2, 1, 5) + shape (1, 3, 5) → (2, 3, 5) Dim 1 of first tensor (1) broadcasts to 3. Dim 0 of second tensor (1) broadcasts to 2. In attention — causal masking: scores shape: (batch, heads, seq, seq) = (2, 8, 10, 10) mask shape: (1, 1, seq, seq) = (1, 1, 10, 10) mask broadcasts over batch and heads dimensions automatically
Python · Broadcasting in transformer operations
import numpy as np

# Add bias to linear layer output
x = np.random.randn(4, 768)      # 4 tokens, each 768-dim
bias = np.random.randn(768)        # one bias per output dim
out = x + bias                       # (4,768) + (768,) → (4,768)
# bias is broadcast: same bias added to every token

# Attention masking with broadcasting
scores = np.random.randn(2, 8, 10, 10)  # (batch, heads, seq, seq)

# Causal mask: prevent attending to future positions
causal_mask = np.tril(np.ones((10, 10)))  # (10, 10) lower triangular
mask = (causal_mask == 0)                   # True where we want -inf

# Broadcast mask over batch and head dims
scores_masked = scores.copy()
scores_masked[mask[None, None, :, :]] = -1e9   # broadcast applied

# Scale factor broadcast over all dimensions
d_k = 64
scores_scaled = scores / np.sqrt(d_k)   # scalar broadcasts over everything

Matrices as Linear Maps

Every matrix is a transformation — a way of reshaping space

A matrix doesn't just store numbers — it transforms vectors. When you compute y = Ax, you are applying a linear transformation to x. The matrix A is a function that takes a vector in ℝⁿ and maps it to a vector in ℝᵐ.

This geometric view is crucial for understanding what transformer layers do:

  • Rotation — attention heads rotate embeddings into a subspace where similarity is meaningful
  • Scaling — different dimensions are amplified or suppressed
  • Projection — the Q, K, V projection matrices project the full embedding into a lower-dimensional subspace for each head

Rank of a matrix

The rank of a matrix is the number of linearly independent rows (or columns). A full-rank matrix can represent any transformation in its space. A low-rank matrix compresses information — this is exactly what LoRA exploits.

Geometric intuition: W projects 3D → 2D 3D embedding space 2D attention subspace ↑ ↑ [0.8] [1.2] [0.3] → multiply by W → [0.5] [0.6] (768-dim) (64-dim per head) LoRA: learn a low-rank update instead of full W W_updated = W_original + B × A = (d × d) + (d × r)(r × d) where r << d Example: d=768, r=8 → only 768×8 + 8×768 = 12,288 params vs full matrix: 768×768 = 589,824 params (48× fewer)

LLM Weight Matrices

Every matrix in a transformer — what it is and what it does

MatrixShapeWhat it does
Embedding table E(vocab × d_model)Converts token id to embedding vector via row lookup
W_Q (query)(d_model × d_k)Projects token embedding into query space per head
W_K (key)(d_model × d_k)Projects token embedding into key space per head
W_V (value)(d_model × d_v)Projects token embedding into value space per head
W_O (output)(h·d_v × d_model)Projects concatenated head outputs back to d_model

Parameter Count

A GPT-2 small model (d_model=768, 12 heads, 12 layers): each layer has 4 attention matrices + 2 FFN matrices ≈ 7M parameters per layer × 12 layers = ~84M. Add embedding table (50257×768 ≈ 38M) → ~122M total. Every single one of these is a matrix multiply at inference time.
