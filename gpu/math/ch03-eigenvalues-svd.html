Eigenvalues & Eigenvectors

Special directions a matrix doesn't rotate — only scales

For a square matrix A, an eigenvector v is a special vector that, when A is applied to it, only gets scaled — not rotated. The scaling factor is the eigenvalue λ.

Eigenvalue equation: A · v = λ · v A is the matrix, v is the eigenvector, λ (lambda) is the eigenvalue. Interpretation: λ > 1 → the matrix stretches v λ = 1 → v is unchanged 0 < λ < 1 → v is compressed λ < 0 → v is flipped and scaled λ = 0 → v is collapsed to zero (the matrix is singular)

Why do eigenvalues matter for LLMs?

  • Gradient flow — the eigenvalues of the Hessian (second derivative matrix) determine how fast or slow different directions in parameter space learn. Vanishing/exploding gradients are an eigenvalue problem.
  • Attention head analysis — researchers analyse eigenvalues of attention weight matrices to understand what each head "focuses on".
  • Stability — weight matrices with large eigenvalues can cause exploding activations. Weight initialisation schemes (Xavier, Kaiming) are designed to keep eigenvalues in a stable range.
Python · Eigendecomposition
import numpy as np

A = np.array([[4, 1],
              [2, 3]])

eigenvalues, eigenvectors = np.linalg.eig(A)
print("Eigenvalues:", eigenvalues)       # [5. 2.]
print("Eigenvectors:\n", eigenvectors)   # columns are eigenvectors

# Verify: A @ v = λ @ v
λ, v = eigenvalues[0], eigenvectors[:, 0]
print(np.allclose(A @ v, λ * v))   # True

# Largest eigenvalue of a weight matrix (spectral norm)
W = np.random.randn(64, 64)
vals = np.linalg.eigvals(W)
spectral_norm = np.max(np.abs(vals))
print(ff"Spectral norm: {spectral_norm:.2f}")
# Large spectral norm → risk of exploding activations

Singular Value Decomposition (SVD)

Decompose any matrix into rotation, scale, rotation

SVD works on any matrix (not just square ones). It factorises a matrix M of shape (m×n) into three matrices:

M = U · Σ · Vᵀ U: (m × m) — left singular vectors (orthonormal) Σ: (m × n) — diagonal, singular values σ₁ ≥ σ₂ ≥ ... ≥ 0 Vᵀ:(n × n) — right singular vectors transposed (orthonormal) Geometric meaning: Vᵀ: rotate input space Σ: scale each dimension by its singular value U: rotate output space Truncated SVD (rank-r approximation): M ≈ U_r · Σ_r · Vᵀ_r (keep only top-r singular values) This is the best rank-r approximation of M (Eckart-Young theorem)
Python · SVD and low-rank approximation
import numpy as np
import matplotlib.pyplot as plt

# SVD of a weight matrix
W = np.random.randn(768, 768)
U, S, Vt = np.linalg.svd(W)

print("Singular values (first 5):", S[:5])
print("How fast they decay:", S[:10] / S[0])

# Rank-r approximation: keep top r singular values
r = 32
W_approx = U[:, :r] @ np.diag(S[:r]) @ Vt[:r, :]
error = np.linalg.norm(W - W_approx, 'fro')
print(ff"Rank-{r} approx error (Frobenius): {error:.4f}")
print(ff"Parameters: full={768*768}, rank-{r}={768*r + r + r*768}")

# Explained variance: how much info is in top-k singular values
explained = np.cumsum(S**2) / np.sum(S**2)
print(ff"Top-32 explains {explained[31]:.1%} of variance")

PCA & Dimensionality Reduction

Find the directions of most variation in high-dimensional data

Principal Component Analysis (PCA) uses SVD to find the directions of maximum variance in a dataset. Applied to LLM embeddings, PCA can compress 768-dimensional token vectors into 2D for visualisation — and the resulting plot reveals semantic clusters: animals cluster together, countries cluster together, verbs cluster together.

How PCA uses SVD

  • Collect a matrix X of shape (n_samples × d_model)
  • Centre the data: subtract the mean embedding
  • Compute SVD: X = U Σ Vᵀ
  • The columns of V are the principal components (the most important directions)
  • Project data onto top-k components: X_reduced = X @ V[:, :k]
Python · PCA on LLM embeddings
import numpy as np

# Simulate 100 token embeddings (768-dim)
embeddings = np.random.randn(100, 768)

# Step 1: Centre (subtract mean)
mean = embeddings.mean(axis=0)
X_centred = embeddings - mean

# Step 2: SVD
U, S, Vt = np.linalg.svd(X_centred, full_matrices=False)
# U: (100, 100), S: (100,), Vt: (100, 768)

# Step 3: Project to 2D (for visualisation)
X_2d = X_centred @ Vt[:2].T   # (100, 768) @ (768, 2) → (100, 2)
print("2D projection shape:", X_2d.shape)

# Explained variance per component
var_explained = (S**2) / (S**2).sum()
print(ff"PC1 explains {var_explained[0]:.1%} of variance")
print(ff"PC2 explains {var_explained[1]:.1%} of variance")

# sklearn shortcut (more numerically stable)
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_2d_sk = pca.fit_transform(embeddings)
print("Explained variance ratio:", pca.explained_variance_ratio_)

Low-Rank Approximation & LoRA

Why fine-tuning updates are low-rank — and how LoRA exploits this

The key insight behind LoRA (Low-Rank Adaptation) is an empirical observation: when you fine-tune a large language model, the change in each weight matrix ΔW turns out to have a very low intrinsic rank. In other words, the update lives in a small subspace of the full parameter space.

Instead of storing the full ΔW (which has the same size as W itself), LoRA decomposes it as: ΔW = B × A where A has shape (r × d_in) and B has shape (d_out × r), with r ≪ min(d_in, d_out).

Standard fine-tuning: W_new = W_pretrained + ΔW cost: d_in × d_out extra parameters LoRA: W_new = W_pretrained + B × A A: (r × d_in) initialized with random Gaussian B: (d_out × r) initialized to zero (so ΔW=0 at start) cost: r × d_in + d_out × r = r(d_in + d_out) parameters Example: d_in=768, d_out=768, r=8 Full ΔW: 768 × 768 = 589,824 params LoRA: 8×768 + 768×8 = 12,288 params (48× reduction!)
Python · LoRA layer from scratch
import numpy as np

class LoRALinear:
    def __init__(self, d_in, d_out, rank=8, alpha=16):
        # Pretrained weight (frozen)
        self.W = np.random.randn(d_in, d_out) * 0.02
        self.W_frozen = True

        # LoRA matrices (trainable)
        self.A = np.random.randn(rank, d_in) * 0.01  # random init
        self.B = np.zeros((d_out, rank))               # zero init → ΔW=0 at start
        self.scale = alpha / rank                       # LoRA scaling factor

    def forward(self, x):
        # Frozen path: x @ W
        base = x @ self.W

        # LoRA path: x @ Aᵀ @ Bᵀ  (scaled)
        lora = x @ self.A.T @ self.B.T * self.scale

        return base + lora   # ΔW = B @ A (after training)

    def trainable_params(self):
        return self.A.size + self.B.size

layer = LoRALinear(768, 768, rank=8)
x = np.random.randn(4, 768)   # 4 tokens
out = layer.forward(x)
print(ff"Output shape: {out.shape}")
print(ff"Trainable params: {layer.trainable_params():,} vs full: {768*768:,}")

Matrix Conditioning

Why some matrices are numerically unstable — and how to fix it

The condition number of a matrix is the ratio of its largest to smallest singular value: κ(A) = σ_max / σ_min. A high condition number means the matrix is nearly singular — small changes in input cause large changes in output. This makes gradient-based optimisation unstable.

Connections to LLM training

  • LayerNorm / RMSNorm — normalising activations keeps condition numbers low, making the loss landscape smoother and easier to optimise.
  • Weight initialisation — Xavier and Kaiming init choose initial weights so singular values are in a healthy range.
  • Gradient clipping — clips the gradient norm to prevent any single update from distorting weight matrix conditioning severely.
  • Attention softmax temperature — the 1/√d_k scale prevents attention logits from becoming too large, which would make the softmax nearly a one-hot vector (a "peaked" distribution with poor conditioning).
Chapter Summary

SVD reveals the intrinsic rank of a matrix. LoRA is pure applied SVD: fine-tuning updates have low rank, so represent them as a product of two thin matrices. Eigenvalues tell you about the dynamics of a transformation. Condition numbers tell you whether a matrix is numerically well-behaved. All of these show up constantly in LLM research papers.
