Calculus &
Backpropagation

Backpropagation is the algorithm that trains neural networks — and it is pure calculus. The chain rule computes how much each weight contributed to the loss, so we can nudge every weight in the direction that reduces it. Understanding this makes LLM training intuitive, not magical.

01

Derivatives — Rate of Change

How sensitive is the output to a small change in the input?

The derivative df/dx measures how much the function f changes when you increase x by a tiny amount. Geometrically, it's the slope of the function at that point.

For training neural networks, what matters is: if I increase this weight by a tiny amount, does the loss go up or down, and by how much? That is the derivative of the loss with respect to the weight.

Definition: df/dx = lim_{h→0} (f(x+h) - f(x)) / h Key derivatives (memorise these — they appear in every LLM layer): d/dx [x²] = 2x d/dx [xⁿ] = n·xⁿ⁻¹ d/dx [exp(x)] = exp(x) d/dx [ln(x)] = 1/x d/dx [sigmoid(x)] = sigmoid(x)·(1 - sigmoid(x)) d/dx [tanh(x)] = 1 - tanh²(x) d/dx [max(0,x)] = 1 if x>0, 0 if x<0 ← ReLU derivative
Python · Numerical derivatives vs analytical
import numpy as np

# Numerical derivative (finite differences)
def numerical_grad(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)   # central differences

# Analytical derivatives of common activation functions
sigmoid   = lambda x: 1 / (1 + np.exp(-x))
d_sigmoid = lambda x: sigmoid(x) * (1 - sigmoid(x))

relu      = lambda x: np.maximum(0, x)
d_relu    = lambda x: (x > 0).astype(float)

# GELU (used in GPT, BERT) — no clean closed form, approximated
gelu      = lambda x: x * sigmoid(1.702 * x)
d_gelu    = lambda x: numerical_grad(gelu, x)

x = 1.5
print(ff"sigmoid({x}) = {sigmoid(x):.4f},  d/dx = {d_sigmoid(x):.4f}")
print(ff"relu({x})    = {relu(x):.4f},  d/dx = {d_relu(x):.4f}")
print(ff"gelu({x})    = {gelu(x):.4f},  d/dx ≈ {d_gelu(x):.4f}")
02

The Chain Rule

How to differentiate compositions of functions — the engine of backpropagation

The chain rule says: if z depends on y, and y depends on x, then the rate at which z changes with x = (rate z changes with y) × (rate y changes with x).

This is backpropagation. Every transformer layer is a composition of functions. The chain rule propagates the gradient of the loss backward through each function, layer by layer.

Chain rule: dz/dx = dz/dy · dy/dx Multi-variable (gradient of composite): If z = f(g(x)): dz/dx = f'(g(x)) · g'(x) Transformer example — trace the gradient through one step: x → W → z=Wx → a=GELU(z) → y=Va → L=cross_entropy(y) ∂L/∂V = ∂L/∂y · ∂y/∂V = δy · aᵀ ∂L/∂a = ∂L/∂y · ∂y/∂a = Vᵀ · δy ∂L/∂z = ∂L/∂a · ∂a/∂z = δa ⊙ GELU'(z) ∂L/∂W = ∂L/∂z · ∂z/∂W = δz · xᵀ ∂L/∂x = ∂L/∂z · ∂z/∂x = Wᵀ · δz ← gradient to previous layer
Python · Chain rule through a simple 2-layer network
import numpy as np

np.random.seed(42)
# Tiny network: x (3,) → W1 (3,4) → ReLU → W2 (4,2) → softmax → loss
x  = np.array([1.0, 2.0, -1.0])
W1 = np.random.randn(3, 4) * 0.1
W2 = np.random.randn(4, 2) * 0.1
target = 1   # correct class

# ── FORWARD PASS ──────────────────────────────────────
z1   = x @ W1              # (3,) @ (3,4) → (4,)
a1   = np.maximum(0, z1)   # ReLU
z2   = a1 @ W2             # (4,) @ (4,2) → (2,)
e    = np.exp(z2 - z2.max())
probs = e / e.sum()        # softmax
loss = -np.log(probs[target] + 1e-10)
print(ff"Loss: {loss:.4f}")

# ── BACKWARD PASS (chain rule) ─────────────────────────
# Gradient of loss wrt softmax output (dL/dz2)
dz2          = probs.copy()
dz2[target] -= 1          # ← gradient of cross-entropy + softmax combined

# Gradient wrt W2: dL/dW2 = a1^T @ dz2
dW2 = np.outer(a1, dz2)

# Gradient wrt a1: dL/da1 = W2 @ dz2
da1 = W2 @ dz2

# Gradient wrt z1 through ReLU (chain rule): dL/dz1 = da1 * ReLU'(z1)
dz1 = da1 * (z1 > 0).astype(float)   # ⊙ element-wise

# Gradient wrt W1: dL/dW1 = x^T @ dz1
dW1 = np.outer(x, dz1)

print(ff"dW1 norm: {np.linalg.norm(dW1):.6f}")
print(ff"dW2 norm: {np.linalg.norm(dW2):.6f}")

Gradients & Jacobians

Extending derivatives to multi-dimensional functions

The gradient ∇f of a scalar-valued function f(x₁, ..., xₙ) is the vector of all partial derivatives. It points in the direction of steepest ascent.

The Jacobian J of a vector-valued function f: ℝⁿ → ℝᵐ is an (m×n) matrix of all partial derivatives. Each row i contains the gradient of output fᵢ with respect to all inputs.

In practice, PyTorch's autograd computes the vector-Jacobian product (VJP) efficiently — you never materialise the full Jacobian for large models (it would have trillions of entries).

Gradient (scalar f → vector): ∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ] Jacobian (vector f → matrix): J[i,j] = ∂fᵢ/∂xⱼ For a linear layer y = Wx + b: ∂L/∂W = (∂L/∂y)ᵀ · xᵀ ← outer product ∂L/∂x = Wᵀ · (∂L/∂y) ← gradient to previous layer ∂L/∂b = ∂L/∂y ← same as output gradient
04

Backpropagation

The full algorithm for computing all gradients in one backward pass

Backpropagation through a transformer block FORWARD PASS (left to right, store activations) input x → LayerNorm → x_norm → W_Q, W_K, W_V projections → Q, K, V → Attention scores = Q @ Kᵀ / √d_k → Softmax → attention weights A → Context = A @ V → W_O projection → attn_out → Residual add → x + attn_out → LayerNorm → FFN (W1 → GELU → W2) → Residual add → output BACKWARD PASS (right to left, apply chain rule) ∂L/∂output → through residual: ∂L/∂ffn_out = ∂L/∂output → through W2: ∂L/∂W2 = hidden_stateᵀ @ ∂L/∂output → through GELU: ∂L/∂z = ∂L/∂hidden * GELU'(z) → through W1: ∂L/∂W1 = x_normᵀ @ ∂L/∂z → ... continues back through attention and LayerNorm ...

Activation Functions & Their Gradients

Why LLMs use GELU instead of ReLU

ActivationFormulaDerivativeUsed in

Why GELU over ReLU?

ReLU has a zero gradient for all negative inputs — neurons with negative activations receive no gradient and don't learn (the "dying ReLU" problem). GELU is smooth everywhere and has a small non-zero gradient for slightly negative inputs, allowing those neurons to recover and continue learning. This matters especially in deep transformers where gradient flow through many layers is critical.

06

Vanishing & Exploding Gradients

Why deep networks are hard to train — and how transformers solve it

Vanishing gradients: when gradients are multiplied together through many layers (chain rule), small values (<1) shrink exponentially. By layer 1, the gradient is effectively zero — early layers don't learn.

Exploding gradients: similarly, large values (>1) grow exponentially. Gradients become NaN and training diverges.

Transformer solutions

  • Residual connections — x_out = x_in + f(x_in). The gradient flows through both paths: ∂L/∂x_in = ∂L/∂x_out · (1 + ∂f/∂x_in). The "+1" guarantees at least one unattenuated gradient path to every layer.
  • LayerNorm / RMSNorm — normalises activations to unit variance, keeping gradients in a stable range.
  • Gradient clipping — if ‖∇L‖ > threshold, scale gradient down: ∇L ← ∇L · threshold / ‖∇L‖
  • Careful initialisation — Xavier/Kaiming keep initial gradients well-scaled.
Chapter Summary

Backpropagation is the chain rule applied recursively from the loss backward through every operation. Each weight receives a gradient telling it exactly how to change to reduce the loss. Residual connections, LayerNorm, and gradient clipping are specifically engineered to keep these gradients well-behaved through 100+ layers. Once you understand the chain rule, the entire transformer training procedure is transparent.
