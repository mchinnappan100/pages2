What Is JAX?

NumPy + automatic differentiation + XLA compilation + TPU support

JAX is a Python library developed by Google for high-performance machine learning and scientific computing. It looks like NumPy on the surface, but under the hood every operation is compiled by XLA (Accelerated Linear Algebra) — Google's optimising compiler that generates highly efficient machine code for CPUs, GPUs, and TPUs.

The key insight: JAX treats Python functions as mathematical functions in the pure sense — no hidden state, no mutation. This makes it possible to automatically differentiate, vectorise, JIT-compile, and parallelise them in principled ways.

JAX — Core Flax — Neural Nets Optax — Optimisers Orbax — Checkpointing XLA — Compiler
🔢

NumPy-compatible API

JAX's primary API mirrors NumPy exactly. Any NumPy code can be migrated by changing the import. The same array operations, broadcasting rules, and indexing work identically — but execute on GPUs and TPUs.

  • jax.numpy replaces numpy — drop-in compatible
  • jnp.array, jnp.dot, jnp.sum — identical signatures
  • Immutable arrays: no in-place mutation (x[0] = 1 is forbidden)
  • DeviceArray type lives on GPU/TPU automatically
🧮

Functional Transformations

JAX's power comes from four composable function transformations that convert Python functions into faster, differentiated, vectorised, or parallelised versions.

  • jax.grad: automatic differentiation
  • jax.jit: XLA compilation to GPU/TPU machine code
  • jax.vmap: vectorise over a batch dimension
  • jax.pmap: parallelise across multiple devices
🔧

XLA Compilation

Every JAX operation is compiled by XLA, which performs operator fusion, memory layout optimisation, and hardware-specific code generation. JIT'd JAX code runs at near-peak hardware efficiency on both NVIDIA GPUs and Google TPUs.

  • Operator fusion: no intermediate buffers for chained ops
  • Memory layout: row-major vs column-major optimised per op
  • TPU-native: XLA is the native language of Google TPUs
  • Compilation cache: compile once, run many times

JAX → XLA → TPU Stack

How Python code becomes optimised TPU instructions

Full JAX Compilation Stack Python code │ (your training loop, model forward pass) ▼ JAX tracing │ JAX traces the Python function symbolically, │ recording operations on abstract values (ShapedArrays). │ No actual computation yet. ▼ Jaxpr (JAX's intermediate representation) │ A functional program with explicit data flow. │ e.g.: { lambda a b ; dot(a, b) |> relu } ▼ HLO (High-Level Operations) │ XLA's computation graph representation. │ Hardware-independent, ready for optimisation. ▼ XLA Optimisations │ Fusion: y = relu(dot(a,b)) → single kernel │ Layout: choose optimal memory layout per op │ Tiling: tile for TPU 128×128 systolic array ▼ Machine Code │ ├─→ TPU XLA Binary (runs on Google Cloud TPUs) ├─→ CUDA PTX / SASS (runs on NVIDIA GPUs) └─→ CPU LLVM IR (runs on CPUs)
Why XLA Matters for LLMs

A transformer layer involves dozens of chained operations: LayerNorm → Q/K/V projections → attention → softmax → output projection → FFN → residuals. Without XLA fusion, each operation reads and writes a separate buffer in HBM memory. XLA fuses them into far fewer kernel launches with minimal memory traffic — exactly the optimisation that makes training 100B+ parameter models tractable.

Four Core Transforms

grad, jit, vmap, pmap — composable function transformations

Python · JAX's four transforms at a glance
import jax
import jax.numpy as jnp

# ── grad: automatic differentiation ──────────────────────────────
def loss(params, x, y):
    pred = jnp.dot(x, params['w']) + params['b']
    return jnp.mean((pred - y)**2)

grad_fn = jax.grad(loss)         # ∂loss/∂params (first arg by default)
grads   = grad_fn(params, x, y)  # grads has same structure as params

# ── jit: compile with XLA ─────────────────────────────────────────
fast_loss = jax.jit(loss)        # first call: compile. subsequent: cached.
L = fast_loss(params, x, y)

# ── vmap: vectorise over batch dimension ──────────────────────────
def single_example_loss(params, x, y):   # written for ONE example
    pred = jnp.dot(x, params['w']) + params['b']
    return (pred - y)**2

# vmap lifts it to handle a whole batch, no loop needed:
batched_loss = jax.vmap(single_example_loss, in_axes=(None, 0, 0))
losses = batched_loss(params, X_batch, Y_batch)  # shape: (batch_size,)

# ── pmap: parallelise across devices ─────────────────────────────
def train_step(params, batch):
    grads = jax.grad(loss)(params, batch['x'], batch['y'])
    return jax.tree_util.tree_map(lambda p,g: p - 0.01*g, params, grads)

parallel_step = jax.pmap(train_step, axis_name='devices')
# Each device processes one shard of the batch simultaneously

# ── Composing transforms ──────────────────────────────────────────
# All transforms are composable:
fast_grad = jax.jit(jax.grad(loss))          # JIT the gradient computation
batch_grad = jax.jit(jax.vmap(jax.grad(loss), in_axes=(None,0,0)))  # JIT+vmap+grad

Functional Programming in JAX

Why explicit state makes distributed LLM training dramatically simpler

In PyTorch, a model is a stateful object: calling model(x) uses weights stored inside the model. In JAX, everything is explicit: the weights are a separate data structure passed as an argument.

This functional design — no hidden state, no mutation — is what makes JAX's transforms work. A pure function f(params, x) can be safely differentiated, vectorised, parallelised, and compiled without worrying about side effects.

PyTorch — stateful (OOP) model = MyTransformer() ← state inside object # Weights live inside model object output = model(x) # Training: model mutates itself loss.backward() optimizer.step() ← updates model.parameters() in place Problem for distribution: "model" must be synchronized across devices. Mutation makes this tricky.
JAX — functional (explicit state) params = init(key, x) ← state is just a dict/array # Weights passed explicitly output = apply(params, x) # Training: returns NEW params (no mutation) grads = jax.grad(loss)(params, x, y) new_params = jax.tree_util.tree_map( lambda p, g: p - lr*g, params, grads) Benefit for distribution: params is just data — shard it, replicate it, send it anywhere. No object synchronisation needed.
Python · Functional vs OOP — training loop comparison
import jax
import jax.numpy as jnp

# ── JAX functional training loop ─────────────────────────────────
def linear_model(params, x):
    """Pure function: output depends only on params and x."""
    return jnp.dot(x, params['W']) + params['b']

def mse_loss(params, x, y):
    pred = linear_model(params, x)
    return jnp.mean((pred - y) ** 2)

# Initialise (random key is explicit — JAX is deterministically random)
key    = jax.random.PRNGKey(42)
k1, k2 = jax.random.split(key)
params = {
    'W': jax.random.normal(k1, (4, 2)),
    'b': jax.random.normal(k2, (2,))
}

# Training step: pure function, no mutation
@jax.jit
def train_step(params, x, y, lr=0.01):
    loss_val, grads = jax.value_and_grad(mse_loss)(params, x, y)
    # tree_map applies update to every leaf (weight tensor)
    new_params = jax.tree_util.tree_map(
        lambda p, g: p - lr * g,
        params, grads
    )
    return new_params, loss_val

# Synthetic data
X = jax.random.normal(jax.random.PRNGKey(0), (100, 4))
Y = jax.random.normal(jax.random.PRNGKey(1), (100, 2))

# Train for 100 steps
for step in range(100):
    params, loss = train_step(params, X, Y)
    if step % 20 == 0:
        print(ff"Step {step}: loss = {loss:.4f}")
# Note: params is always immutable — each step returns new params

Automatic Differentiation

jax.grad, value_and_grad, and higher-order derivatives

JAX computes exact derivatives of any differentiable Python function using automatic differentiation. Unlike PyTorch's dynamic tape (which builds the computation graph during the forward pass), JAX uses source code transformation — it traces through the function symbolically and generates code for the derivative directly.

This approach supports both forward-mode (jvp) and reverse-mode (vjp/grad) differentiation, and arbitrarily high-order derivatives — you can differentiate a gradient to get a Hessian-vector product, or differentiate that to get third-order information.

Python · Differentiation — grad, jacobian, hessian
import jax
import jax.numpy as jnp

# ── Scalar gradient ────────────────────────────────────────────────
def f(x): return jnp.sin(x) * x**2

df_dx = jax.grad(f)
print(f"f'(1.0) = {df_dx(1.0):.4f}")   # exact derivative at x=1

# ── value_and_grad: get both loss and gradients in one pass ────────
def loss_fn(params, x, y):
    pred = jnp.dot(x, params)
    return jnp.mean((pred - y)**2)

params = jnp.array([0.5, -0.3, 1.2])
x = jnp.ones(3); y = jnp.array(2.0)

loss, grads = jax.value_and_grad(loss_fn)(params, x, y)
print(ff"Loss: {loss:.4f},  Grads: {grads}")

# ── Gradient w.r.t. specific argument (argnums) ───────────────────
# grad(f, argnums=0) differentiates w.r.t. first argument (default)
# grad(f, argnums=1) differentiates w.r.t. second argument
grad_wrt_x = jax.grad(loss_fn, argnums=1)   # ∂loss/∂x (not ∂loss/∂params)

# ── Higher-order derivatives ──────────────────────────────────────
def g(x): return x**3 + 2*x**2

dg    = jax.grad(g)          # first derivative: 3x² + 4x
d2g   = jax.grad(dg)         # second derivative: 6x + 4
d3g   = jax.grad(d2g)        # third derivative: 6

print(ff"g'''(5.0) = {d3g(5.0)}")   # should be 6.0

# ── Jacobian (for vector-valued functions) ────────────────────────
def h(x): return jnp.array([x[0]**2 + x[1], x[0]*x[1]])

J = jax.jacobian(h)(jnp.array([2.0, 3.0]))   # 2×2 Jacobian
print(f"Jacobian:\n{J}")

JIT Compilation

Convert Python to optimised machine code with jax.jit

Python function calls are slow — each call has Python interpreter overhead, and intermediate values create memory allocations. jax.jit eliminates this by tracing the function, building an XLA computation graph, compiling it to machine code, and caching the result. Subsequent calls skip Python entirely and run compiled code at hardware speed.

The critical rule: jit-compiled functions must be pure — their output depends only on their inputs. JAX cannot compile functions with Python-level side effects (printing, writing to global state). JAX does allow dynamic shapes, but recompilation is triggered when shapes change.

Python · jit — compilation, static arguments, shape recompilation
import jax
import jax.numpy as jnp
import time

# ── Basic JIT ─────────────────────────────────────────────────────
def slow_matmul(A, B):
    return jnp.dot(A, B).sum()

fast_matmul = jax.jit(slow_matmul)

A = jax.random.normal(jax.random.PRNGKey(0), (1024, 1024))
B = jax.random.normal(jax.random.PRNGKey(1), (1024, 1024))

# First call: traces + compiles (slow)
t0 = time.perf_counter()
result = fast_matmul(A, B).block_until_ready()  # wait for async GPU op
print(ff"First call (compile + run): {time.perf_counter()-t0:.3f}s")

# Second call: uses compiled code (fast)
t0 = time.perf_counter()
result = fast_matmul(A, B).block_until_ready()
print(ff"Second call (cached):       {time.perf_counter()-t0:.3f}s")

# ── Decorator syntax ──────────────────────────────────────────────
@jax.jit
def train_step(params, x, y):
    loss, grads = jax.value_and_grad(lambda p: jnp.mean((jnp.dot(x,p)-y)**2))(params)
    return params - 0.01*grads, loss

# ── Static arguments: not traced, trigger recompilation if changed ─
@jax.jit
def forward(params, x, *, training=True):
    h = jnp.dot(x, params)
    if training:
        h = h * 0.9   # dropout simulation
    return h
# JAX will trace once for training=True and once for training=False

# ── Avoid recompilation: keep shapes fixed ────────────────────────
# Good: always pass (batch=32, seq=512) tensors
# Bad: vary batch size → triggers recompilation each time
# Tip: pad sequences to a fixed max length for transformers

Parallelism at Scale

pmap, pjit, and shard_map — from 8 GPUs to 4096-chip TPU pods

JAX was designed from day one for multi-device parallelism. Training a 1 trillion-parameter model like Gemini Ultra requires thousands of accelerators with tensor, pipeline, and data parallelism all active simultaneously. JAX's device mesh abstraction makes this composable.

pmap — Data Parallelism

jax.pmap replicates a function across all available devices. Each device processes a different shard of the batch. Collective operations like AllReduce are expressed with lax.pmean / lax.psum.

pjit / jit with sharding — Model Parallelism

The modern approach uses jax.jit with explicit PartitionSpec annotations. You describe how each tensor is partitioned across a device mesh, and JAX automatically inserts the necessary collective communication ops (AllGather, ReduceScatter) during compilation.

Python · pmap data parallelism + device mesh sharding
import jax
import jax.numpy as jnp
from jax.sharding import Mesh, PartitionSpec, NamedSharding
from jax.experimental import mesh_utils

# ── pmap: data parallel across all devices ──────────────────────
n_devices = jax.device_count()

@jax.pmap
def parallel_train_step(params, batch):
    """Runs on each device with its own shard of the batch."""
    def loss_fn(p):
        pred = jnp.dot(batch['x'], p)
        return jnp.mean((pred - batch['y'])**2)

    loss, grads = jax.value_and_grad(loss_fn)(params)
    # Average gradients across all devices (AllReduce)
    grads = jax.lax.pmean(grads, axis_name='batch')
    loss  = jax.lax.pmean(loss,  axis_name='batch')
    return params - 0.01 * grads, loss

# ── Modern approach: device mesh + PartitionSpec ─────────────────
# Define a 2D device mesh: (data_parallel, tensor_parallel)
devices   = mesh_utils.create_device_mesh((2, 4))   # 2 DP × 4 TP = 8 devices
mesh      = Mesh(devices, axis_names=('dp', 'tp'))

# Shard a 4096×4096 weight matrix along the tensor-parallel axis
W_sharding = NamedSharding(mesh, PartitionSpec(None, 'tp'))   # shard columns
X_sharding = NamedSharding(mesh, PartitionSpec('dp', None))   # shard batch

W = jax.device_put(jnp.zeros((4096, 4096)), W_sharding)
X = jax.device_put(jnp.zeros((64,   4096)), X_sharding)

print(ff"W sharding: {W.sharding}")   # columns split across 4 TP devices
print(ff"X sharding: {X.sharding}")   # batch split across 2 DP devices

# jit automatically handles AllGather/ReduceScatter when ops need full tensors
@jax.jit
def sharded_matmul(W, X):
    return X @ W   # JAX infers necessary collectives from sharding specs

result = sharded_matmul(W, X)
print(ff"Output shape: {result.shape},  sharding: {result.sharding}")

Flax — Neural Network Library

Google's JAX-native neural network framework

Flax provides neural network building blocks (layers, modules) on top of JAX, while respecting JAX's functional style. A Flax model defines its parameters via init() and applies them via apply() — both are pure functions. This makes Flax models first-class citizens of JAX's transform ecosystem.

The newer Flax NNX API (2024) provides a more familiar OOP interface while keeping the functional backend — bridging the ergonomics of PyTorch with the power of JAX's compilation and parallelism.

Python · Transformer block in Flax (LLM-style)
import jax
import jax.numpy as jnp
import flax.linen as nn

class MultiHeadAttention(nn.Module):
    d_model:  int
    n_heads:  int

    @nn.compact
    def __call__(self, x, mask=None):
        B, T, D = x.shape
        d_head  = D // self.n_heads

        qkv = nn.Dense(D * 3, use_bias=False)(x)   # (B, T, 3D)
        q, k, v = jnp.split(qkv, 3, axis=-1)

        # Reshape to (B, n_heads, T, d_head)
        q = q.reshape(B, T, self.n_heads, d_head).transpose(0, 2, 1, 3)
        k = k.reshape(B, T, self.n_heads, d_head).transpose(0, 2, 1, 3)
        v = v.reshape(B, T, self.n_heads, d_head).transpose(0, 2, 1, 3)

        scores = jnp.matmul(q, k.transpose(0, 1, 3, 2)) / jnp.sqrt(d_head)
        if mask is not None:
            scores = jnp.where(mask, scores, jnp.finfo(jnp.float32).min)
        attn = jax.nn.softmax(scores, axis=-1)
        out  = jnp.matmul(attn, v).transpose(0, 2, 1, 3).reshape(B, T, D)
        return nn.Dense(D, use_bias=False)(out)


class TransformerBlock(nn.Module):
    d_model: int; n_heads: int; d_ff: int

    @nn.compact
    def __call__(self, x, mask=None, training=False):
        # Pre-norm attention
        x = x + MultiHeadAttention(self.d_model, self.n_heads)(
            nn.RMSNorm()(x), mask=mask)
        # Pre-norm FFN
        h = nn.RMSNorm()(x)
        x = x + nn.Dense(self.d_model, use_bias=False)(
            nn.gelu(nn.Dense(self.d_ff, use_bias=False)(h)))
        return x


# ── Initialise and run ────────────────────────────────────────────
block = TransformerBlock(d_model=512, n_heads=8, d_ff=2048)
x     = jnp.zeros((2, 16, 512))  # (batch=2, seq=16, d=512)
key   = jax.random.PRNGKey(0)

# Init: returns dict of parameters (the functional state)
params = block.init(key, x)
print(f"Param keys: {list(params['params'].keys())}")

# Apply: pure function, same interface as all JAX transforms
out = block.apply(params, x)
print(ff"Output shape: {out.shape}")   # (2, 16, 512)

# JIT the apply (compiles to XLA graph)
fast_apply = jax.jit(block.apply)
out_fast   = fast_apply(params, x)   # first call compiles, rest are fast

Optax — Optimisers

Composable gradient transformations for LLM training

Optax is Google DeepMind's optimiser library for JAX. Like JAX itself, Optax follows the functional paradigm: an optimiser is not a stateful object but a pair of functions — init (create the optimiser state) and update (compute parameter updates from gradients and state). This makes optimisers composable and JIT-able.

Python · Optax optimisers for LLM training
import jax
import jax.numpy as jnp
import optax

# ── AdamW with cosine LR schedule + warmup ─────────────────────────
# Standard for LLM training (matches PyTorch LLaMA/GPT configs)
total_steps  = 100_000
warmup_steps = 2_000
peak_lr      = 3e-4

schedule = optax.warmup_cosine_decay_schedule(
    init_value=0.0,
    peak_value=peak_lr,
    warmup_steps=warmup_steps,
    decay_steps=total_steps,
    end_value=peak_lr * 0.1  # decay to 10% of peak
)

optimizer = optax.chain(
    optax.clip_by_global_norm(1.0),   # gradient clipping
    optax.adamw(learning_rate=schedule, weight_decay=0.1)
)

# ── Functional training loop with Optax ───────────────────────────
params    = {'W': jnp.zeros((512, 512)), 'b': jnp.zeros(512)}
opt_state = optimizer.init(params)   # optimiser state: Adam m,v + step count

@jax.jit
def train_step(params, opt_state, grads):
    # optax.update: compute updates from gradients + state
    updates, new_opt_state = optimizer.update(grads, opt_state, params)
    # Apply updates to parameters (functional: returns new params)
    new_params = optax.apply_updates(params, updates)
    return new_params, new_opt_state

# Simulate one training step
fake_grads = jax.tree_util.tree_map(jnp.ones_like, params)
new_params, new_opt_state = train_step(params, opt_state, fake_grads)
print(f"LR at step 0:    {schedule(0):.2e}")
print(f"LR at warmup end: {schedule(warmup_steps):.2e}")
print(f"LR at end:        {schedule(total_steps):.2e}")

# ── Composable optimisers ──────────────────────────────────────────
# Chain combines multiple gradient transformations in sequence:
lion = optax.lion(learning_rate=1e-4, weight_decay=0.1)   # EvoLved Sign Momentum
sophia = optax.chain(optax.clip_by_global_norm(1.0), optax.sgd(1e-3))
# Any transform can be chained, wrapped, or scheduled

Orbax — Checkpointing

Save and restore multi-host distributed model state

Orbax is Google's checkpointing library for JAX, designed for the unique challenges of distributed LLM training: saving petabytes of sharded model state across thousands of devices efficiently, and restoring them reliably when training resumes after a failure.

At the scale of Gemini training (thousands of TPUs running for weeks), checkpoint reliability is critical — a crash without a recent checkpoint can mean days of wasted compute. Orbax handles parallel saving from all devices, atomic writes, and flexible restoration that allows changing the sharding strategy between training and inference.

Python · Orbax checkpoint save and restore
import orbax.checkpoint as ocp
import jax
import jax.numpy as jnp

# ── Save checkpoint ────────────────────────────────────────────────
state = {
    'params': {'W': jnp.ones((4096, 4096)), 'b': jnp.zeros(4096)},
    'opt_state': {'step': jnp.array(1000)},
    'step': 1000
}

checkpointer = ocp.StandardCheckpointer()
checkpointer.save('/tmp/my_checkpoint', args=ocp.args.StandardSave(state))
print("Checkpoint saved.")

# ── Restore checkpoint ─────────────────────────────────────────────
restored = checkpointer.restore(
    '/tmp/my_checkpoint',
    args=ocp.args.StandardRestore(state)   # provide abstract target for shape inference
)
print(ff"Restored step: {restored['step']}")

# ── AsyncCheckpointer: save in background while training continues ─
async_checkpointer = ocp.AsyncCheckpointer(ocp.PyTreeCheckpointHandler())

# This returns immediately — saving happens asynchronously
async_checkpointer.save('/tmp/async_ckpt', state)

# Wait for the async save to complete before closing
async_checkpointer.wait_until_finished()
print("Async checkpoint complete.")

Complete Transformer in JAX

GPT-style causal language model — end-to-end

Python · GPT-style causal LM in JAX + Flax
import jax
import jax.numpy as jnp
import flax.linen as nn
import optax
from functools import partial

class CausalSelfAttention(nn.Module):
    d_model: int; n_heads: int

    @nn.compact
    def __call__(self, x):
        B, T, D = x.shape; dh = D // self.n_heads
        qkv = nn.Dense(D*3, use_bias=False)(x)
        q,k,v = [a.reshape(B,T,self.n_heads,dh).transpose(0,2,1,3)
                 for a in jnp.split(qkv, 3, -1)]
        s = jnp.matmul(q, k.transpose(0,1,3,2)) / jnp.sqrt(dh)
        mask = jnp.tril(jnp.ones((T,T), dtype=bool))   # causal mask
        s = jnp.where(mask[jnp.newaxis,jnp.newaxis], s, jnp.finfo(s.dtype).min)
        a = jax.nn.softmax(s, -1)
        out = jnp.matmul(a,v).transpose(0,2,1,3).reshape(B,T,D)
        return nn.Dense(D, use_bias=False)(out)

class GPTBlock(nn.Module):
    d_model: int; n_heads: int; d_ff: int

    @nn.compact
    def __call__(self, x):
        x = x + CausalSelfAttention(self.d_model, self.n_heads)(nn.LayerNorm()(x))
        h = nn.LayerNorm()(x)
        x = x + nn.Dense(self.d_model)(nn.gelu(nn.Dense(self.d_ff)(h)))
        return x

class GPT(nn.Module):
    vocab_size: int; d_model: int; n_heads: int
    d_ff: int; n_layers: int; max_seq: int

    @nn.compact
    def __call__(self, tokens):
        B, T = tokens.shape
        tok_emb = nn.Embed(self.vocab_size, self.d_model)(tokens)
        pos_emb = nn.Embed(self.max_seq,    self.d_model)(jnp.arange(T))
        x = tok_emb + pos_emb
        for _ in range(self.n_layers):
            x = GPTBlock(self.d_model, self.n_heads, self.d_ff)(x)
        x = nn.LayerNorm()(x)
        return nn.Dense(self.vocab_size, use_bias=False)(x)  # logits

# ── Train ─────────────────────────────────────────────────────────
model  = GPT(vocab_size=1000, d_model=128, n_heads=4,
             d_ff=512, n_layers=4, max_seq=64)
key    = jax.random.PRNGKey(0)
tokens = jax.random.randint(key, (2, 64), 0, 1000)
params = model.init(key, tokens)

optimizer  = optax.adamw(3e-4)
opt_state  = optimizer.init(params)

@jax.jit
def step(params, opt_state, tokens):
    def loss_fn(p):
        logits = model.apply(p, tokens[:, :-1])         # (B, T-1, vocab)
        targets = tokens[:, 1:]                           # (B, T-1)
        return optax.softmax_cross_entropy_with_integer_labels(
            logits.reshape(-1, 1000), targets.reshape(-1)).mean()
    loss, grads    = jax.value_and_grad(loss_fn)(params)
    updates, opt_s = optimizer.update(grads, opt_state, params)
    return optax.apply_updates(params, updates), opt_s, loss

for i in range(5):
    params, opt_state, loss = step(params, opt_state, tokens)
    print(ff"Step {i}: loss={loss:.4f}")

When to Use JAX

Practical guidance for choosing JAX vs PyTorch for LLM work

Choose JAX: Training at Scale

Building very large models (70B+) that need to train on TPU pods or large GPU clusters. JAX's device mesh + XLA auto-sharding handles tensor/pipeline/data parallelism with minimal code.

Choose JAX: TPU Access

Working on Google Cloud TPUs (v4, v5). JAX + XLA is the native TPU programming model — it achieves near-peak TPU utilisation that PyTorch/torch_xla cannot match.

Choose JAX: Research Papers

Reproducing or extending Google DeepMind research. Most papers from DeepMind, Google Brain, and Google Research include JAX code. The functional style makes ablations and transforms easy.

Choose JAX: Scientific Computing

ML beyond neural networks: differentiable physics, protein structure, scientific simulation. JAX's composable transforms (grad, vmap, scan) are uniquely suited for these use cases.

Choose PyTorch: Fine-tuning OSS Models

Working with HuggingFace Transformers, LLaMA, Mistral, or any open-source model. Almost all open-source LLMs are PyTorch; fine-tuning, inference, and deployment tooling is PyTorch-native.

Choose PyTorch: Production Serving

Deploying models in production. vLLM, TensorRT-LLM, TorchServe, and the entire inference optimisation ecosystem is PyTorch. JAX serving requires custom infrastructure.

Book Summary

JAX is the engine behind Gemini, PaLM, and Google's frontier LLMs. Its combination of NumPy-compatible API, XLA compilation, functional transforms (grad, jit, vmap, pmap), and native TPU support makes it uniquely powerful for training very large models at scale. The ecosystem — Flax for neural networks, Optax for optimisers, Orbax for checkpointing — provides everything needed to train a modern LLM. If you're building at Google scale or working with TPUs, JAX is the right tool. For everything else in the open-source LLM ecosystem, PyTorch remains the dominant choice.
