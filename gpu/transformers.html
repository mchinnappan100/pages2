Transformer Architecture

Building the full GPT-style decoder block in PyTorch

GPT-style Decoder-only Transformer Input Tokens [B, T] ↓ Token Embedding + Positional Encoding Hidden States [B, T, d_model] ↓ ┌─── Transformer Block × N ──────────────────────┐ │ RMSNorm │ │ Multi-Head Causal Self-Attention │ │ Q = xW_Q K = xW_K V = xW_V │ │ A = softmax(QKᵀ/√d_k) · V │ │ out = concat(heads) · W_O │ │ + residual │ │ RMSNorm │ │ Feed-Forward Network (SwiGLU) │ │ gate = SiLU(xW₁) │ │ value = xW₂ │ │ out = (gate ⊙ value) W₃ │ │ + residual │ └────────────────────────────────────────────────┘ ↓ RMSNorm → Linear → Logits [B, T, vocab_size]
Python · GPT Block from scratch
import torch
import torch.nn as nn
import math

class RMSNorm(nn.Module):
    def __init__(self, d, eps=1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(d))
        self.eps = eps

    def forward(self, x):
        norm = x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        return self.weight * norm


class SwiGLUFFN(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()
        self.w1 = nn.Linear(d_model, d_ff, bias=False)
        self.w2 = nn.Linear(d_ff, d_model, bias=False)
        self.w3 = nn.Linear(d_model, d_ff, bias=False)

    def forward(self, x):
        return self.w2(torch.nn.functional.silu(self.w1(x)) * self.w3(x))


class CausalSelfAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.n_heads = n_heads
        self.d_head  = d_model // n_heads
        self.qkv     = nn.Linear(d_model, 3*d_model, bias=False)
        self.proj    = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x):
        B, T, C = x.shape
        q, k, v = self.qkv(x).split(C, dim=2)
        q = q.view(B, T, self.n_heads, self.d_head).transpose(1, 2)
        k = k.view(B, T, self.n_heads, self.d_head).transpose(1, 2)
        v = v.view(B, T, self.n_heads, self.d_head).transpose(1, 2)
        # Flash Attention 2 (PyTorch 2.0+): fused, O(N) memory
        out = torch.nn.functional.scaled_dot_product_attention(
            q, k, v, is_causal=True)
        out = out.transpose(1, 2).contiguous().view(B, T, C)
        return self.proj(out)


class TransformerBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.norm1 = RMSNorm(d_model)
        self.attn  = CausalSelfAttention(d_model, n_heads)
        self.norm2 = RMSNorm(d_model)
        self.ffn   = SwiGLUFFN(d_model, d_ff)

    def forward(self, x):
        x = x + self.attn(self.norm1(x))  # pre-norm + residual
        x = x + self.ffn(self.norm2(x))
        return x

Attention Mechanism

Scaled dot-product, multi-head, GQA, MQA, and Flash Attention

Scaled Dot-Product Attention

The fundamental operation: Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V. The scale factor 1/√d_k prevents dot products from growing large and pushing softmax into regions of near-zero gradient.

Multi-Query & Grouped-Query Attention

Standard Multi-Head Attention (MHA) has H query, H key, and H value heads. Multi-Query Attention (MQA) uses 1 K/V head shared across all Q heads — reduces KV cache size by H×. Grouped-Query Attention (GQA) is a middle ground: G groups of Q heads share one K/V head each. Llama 3 uses GQA with 8 KV heads for 70B queries.

Python · GQA (Grouped-Query Attention)
class GroupedQueryAttention(nn.Module):
    def __init__(self, d_model, n_q_heads, n_kv_heads):
        super().__init__()
        assert n_q_heads % n_kv_heads == 0
        self.n_q   = n_q_heads
        self.n_kv  = n_kv_heads
        self.n_rep = n_q_heads // n_kv_heads  # each KV head serves n_rep Q heads
        self.d_h   = d_model // n_q_heads
        self.wq    = nn.Linear(d_model, d_model, bias=False)
        self.wk    = nn.Linear(d_model, n_kv_heads * self.d_h, bias=False)
        self.wv    = nn.Linear(d_model, n_kv_heads * self.d_h, bias=False)
        self.wo    = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x):
        B, T, _ = x.shape
        q = self.wq(x).view(B, T, self.n_q,  self.d_h).transpose(1, 2)
        k = self.wk(x).view(B, T, self.n_kv, self.d_h).transpose(1, 2)
        v = self.wv(x).view(B, T, self.n_kv, self.d_h).transpose(1, 2)
        # Repeat K/V heads to match Q head count
        k = k.repeat_interleave(self.n_rep, dim=1)
        v = v.repeat_interleave(self.n_rep, dim=1)
        out = torch.nn.functional.scaled_dot_product_attention(q, k, v, is_causal=True)
        return self.wo(out.transpose(1, 2).contiguous().view(B, T, -1))
⚡

Flash Attention

Reorders the attention computation to tile across SRAM, never materialising the full N×N matrix in HBM. Reduces attention memory from O(N²) to O(N) and speeds up by 2–4× by improving arithmetic intensity from bandwidth-bound to compute-bound.

  • Flash Attention 1 (Dao et al. 2022): tiled forward + backward
  • Flash Attention 2: parallelise over sequence dim, better work partitioning
  • Flash Attention 3 (H100): uses async wgmma + pingpong pipelines
  • PyTorch 2.0+: F.scaled_dot_product_attention dispatches to FA automatically
🗜️

Sliding Window & Sparse Attention

Reduce O(N²) complexity by restricting which tokens each query can attend to. Mistral uses sliding window attention (each token attends to W neighbours). Longformer adds global tokens for task tokens plus local windows.

  • Sliding window: O(N·W) complexity, W = window size
  • Dilated attention: exponentially spaced positions
  • BigBird: combines random + window + global
  • Ring attention: sequence parallelism over attention

Positional Encoding

Sinusoidal, RoPE, ALiBi — and extending context beyond training length

Python · RoPE (Rotary Position Embedding)
import torch

def precompute_freqs_cis(d_head, seq_len, theta=10000.0):
    # Frequencies: θ_i = 1 / (theta ^ (2i / d_head))
    freqs = 1.0 / (theta ** (torch.arange(0, d_head, 2).float() / d_head))
    t     = torch.arange(seq_len)
    freqs = torch.outer(t, freqs)       # [T, d_head/2]
    return torch.polar(torch.ones_like(freqs), freqs)  # complex: e^(iθ)

def apply_rope(q, k, freqs_cis):
    # q/k: [B, T, H, d_head]
    q_ = torch.view_as_complex(q.float().reshape(*q.shape[:-1], -1, 2))
    k_ = torch.view_as_complex(k.float().reshape(*k.shape[:-1], -1, 2))
    freqs = freqs_cis[:q.shape[1]].unsqueeze(0).unsqueeze(2)   # [1,T,1,d/2]
    q_rot = torch.view_as_real(q_ * freqs).flatten(3)
    k_rot = torch.view_as_real(k_ * freqs).flatten(3)
    return q_rot.type_as(q), k_rot.type_as(k)

Pre-training

Next-token prediction, tokenisation, data mixture, and training stability

Python · Pre-training loop (GPT-style)
import torch, torch.nn as nn
from torch.amp import autocast

model = GPT(vocab_size=32000, d_model=4096, n_layers=32, n_heads=32).cuda()
model = torch.compile(model)   # ~20-40% speedup via Triton kernel fusion

optimizer = torch.optim.AdamW(
    model.parameters(), lr=3e-4, betas=(0.9, 0.95),
    weight_decay=0.1, fused=True
)

def cosine_lr(step, warmup, total, lr_min, lr_max):
    if step < warmup: return lr_max * step / warmup
    t = (step - warmup) / (total - warmup)
    return lr_min + 0.5 * (lr_max - lr_min) * (1 + math.cos(math.pi * t))

for step, (x, y) in enumerate(dataloader):
    lr = cosine_lr(step, warmup=2000, total=100000, lr_min=3e-5, lr_max=3e-4)
    for g in optimizer.param_groups: g["lr"] = lr

    with autocast(device_type="cuda", dtype=torch.bfloat16):
        logits = model(x)                            # [B, T, vocab]
        loss   = nn.functional.cross_entropy(
            logits.view(-1, logits.size(-1)), y.view(-1))

    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    optimizer.zero_grad(set_to_none=True)

    if step % 100 == 0:
        print(ff"step {step}  loss {loss.item():.4f}  lr {lr:.2e}")
🔤

Tokenisation

BPE (Byte-Pair Encoding) and SentencePiece are the dominant tokenisers. Vocabulary size 32K–128K trades off model size vs token efficiency. The tokeniser is frozen before pre-training starts.

  • BPE: greedily merge most frequent byte pairs
  • WordPiece: maximise training data likelihood
  • Unigram (SentencePiece): subword LM, probabilistic
  • Byte-level BPE (GPT-2): no UNK token, handles any text
  • tiktoken (OpenAI): fast Rust implementation of BPE
📚

Data Mixture

Pre-training data quality dominates model quality. Modern recipes mix web (Common Crawl → C4, FineWeb), code (GitHub), books, papers, and curated knowledge with domain-specific weights.

  • Deduplication: MinHash + LSH removes near-duplicate documents
  • Quality filtering: perplexity scoring, heuristic rules
  • Domain weighting: up-weight code/math for capability
  • Curriculum: start with diverse general data, anneal on high-quality

Inference & KV Cache

Autoregressive decoding, KV cache mechanics, speculative decoding

Autoregressive Decoding with KV Cache Prefill phase (process prompt tokens in parallel) ┌────────────────────────────────────────┐ │ tokens: [T₁ T₂ T₃ T₄] │ │ → compute K,V for all positions │ │ → store in KV Cache [layers, 2, T, d] │ └────────────────────────────────────────┘ Decode phase (one token per step, memory-bandwidth-bound) Step 1: input T₄ → attend to KV[0..4] → sample T₅ → append K₅,V₅ to cache Step 2: input T₅ → attend to KV[0..5] → sample T₆ → append K₆,V₆ ... KV cache size per token = 2 × n_layers × n_kv_heads × d_head × dtype_bytes Llama-3 70B, BF16: 2 × 80 × 8 × 128 × 2 = 327 680 bytes ≈ 0.3 MB / token
Python · Simple KV-cache inference
class KVCache:
    def __init__(self, max_len, n_layers, n_kv_heads, d_head, device):
        shape = (n_layers, 2, max_len, n_kv_heads, d_head)
        self.cache = torch.zeros(shape, dtype=torch.bfloat16, device=device)
        self.pos   = 0

    def update(self, layer, k, v):
        T = k.size(1)
        self.cache[layer, 0, self.pos:self.pos+T] = k
        self.cache[layer, 1, self.pos:self.pos+T] = v
        return self.cache[layer, 0, :self.pos+T], self.cache[layer, 1, :self.pos+T]

@torch.no_grad()
def generate(model, prompt_ids, max_new=200, temperature=1.0):
    device = next(model.parameters()).device
    x = prompt_ids.to(device).unsqueeze(0)   # [1, T_prompt]
    cache = KVCache(4096, model.n_layers, model.n_kv_heads, model.d_head, device)

    # Prefill
    logits, cache = model(x, kv_cache=cache)
    cache.pos += x.size(1)

    # Decode
    tokens = [logits[0, -1].div(temperature).softmax(-1).multinomial(1).item()]
    for _ in range(max_new - 1):
        x = torch.tensor([[tokens[-1]]], device=device)
        logits, cache = model(x, kv_cache=cache)
        cache.pos += 1
        tok = logits[0, -1].div(temperature).softmax(-1).multinomial(1).item()
        tokens.append(tok)
        if tok == model.eos_id: break
    return tokens

Speculative Decoding

Run a small draft model (e.g. 7B) to propose K tokens, then verify all K with the large model in parallel in one forward pass. Accepted tokens are free; rejected tokens cost only the verifier call. 2–3× speedup for large models where memory BW is the bottleneck.

Continuous Batching

vLLM and TGI batch decode steps across requests of different lengths by treating the sequence dimension as a ragged batch. New requests join mid-flight; completed requests leave without stalling others. Dramatically higher GPU utilisation vs static batching.

Quantisation (GPTQ / AWQ)

Post-training quantisation to INT4/INT8 reduces model size and memory bandwidth. AWQ finds salient weights and protects them; GPTQ minimises layer-wise reconstruction error with second-order information. 4-bit Llama-3 70B fits in 35 GB.

Paged KV Cache

vLLM manages KV cache in non-contiguous physical pages (like OS virtual memory), eliminating fragmentation and enabling sharing identical prompt prefixes across requests (prefix caching). Enables 24× higher throughput than Hugging Face Transformers baseline.

Fine-tuning & PEFT

Full fine-tuning, LoRA, QLoRA, RLHF, and DPO

Python · LoRA (Low-Rank Adaptation)
import torch, torch.nn as nn

class LoRALinear(nn.Module):
    """Wraps a frozen Linear and adds trainable low-rank delta W = B·A"""
    def __init__(self, linear, r=8, alpha=16):
        super().__init__()
        d_out, d_in = linear.weight.shape
        self.base  = linear
        self.base.weight.requires_grad_(False)   # freeze base
        self.A     = nn.Parameter(torch.randn(r, d_in) * 0.02)  # init A ~N(0,σ)
        self.B     = nn.Parameter(torch.zeros(d_out, r))           # init B = 0
        self.scale = alpha / r                   # LoRA scaling factor

    def forward(self, x):
        return self.base(x) + self.scale * (x @ self.A.T @ self.B.T)

# Inject LoRA into all attention projection layers
def inject_lora(model, r=8, alpha=16, target_modules=("wq", "wk", "wv", "wo")):
    for name, module in model.named_modules():
        if isinstance(module, nn.Linear) and any(t in name for t in target_modules):
            parent = model  # navigate to parent and replace
            *parts, attr = name.split(".")
            for p in parts: parent = getattr(parent, p)
            setattr(parent, attr, LoRALinear(module, r, alpha))
    return model

# Only LoRA parameters are trainable — typically 0.1-1% of model params
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total     = sum(p.numel() for p in model.parameters())
print(ff"Trainable: {trainable/total*100:.2f}%")

RLHF — Reinforcement Learning from Human Feedback

Three-stage pipeline: (1) supervised fine-tuning on demonstrations; (2) train a reward model on human preference pairs; (3) optimise policy with PPO to maximise reward while staying close to SFT policy via KL divergence penalty.

  • SFT: standard cross-entropy on curated demonstrations
  • RM: Bradley-Terry model on preference pairs
  • PPO: clip objective + value baseline + KL penalty
  • Memory: need 3 models live (policy, ref, reward) — ~3× memory
🔄

DPO — Direct Preference Optimisation

Bypasses the explicit reward model. Optimises a classification loss directly on preference pairs using the LLM itself as an implicit reward function. Simpler, more memory-efficient, and often competitive with PPO.

  • Loss: -log σ(β · (log π(y_w|x)/π_ref(y_w|x) − log π(y_l|x)/π_ref(y_l|x)))
  • Only needs policy + frozen reference model
  • β controls divergence from reference (typically 0.1–0.5)
  • SimPO, ORPO: further simplifications (no reference model)

Scaling Laws

Chinchilla, compute-optimal training, and emergence

Chinchilla Scaling Laws (Hoffmann et al. 2022)

Given a fixed compute budget C (in FLOPs), the loss L depends on model size N and training tokens D as:

L(N, D) = E + A/N^α + B/D^β

Chinchilla showed that prior models (GPT-3 175B) were undertrained — they used too many parameters for the compute budget. The optimal compute allocation is ~20 tokens per parameter. Llama models took this seriously: Llama-3 8B was trained on 15 trillion tokens.

Emergent Abilities

Certain capabilities appear abruptly above a threshold scale — few-shot in-context learning, chain-of-thought reasoning, arithmetic. This is debated: some researchers argue emergence is an artefact of discontinuous metrics; others see genuine phase transitions.

Practical Takeaway

For a given inference budget (you'll serve this model for months), train a smaller model for much longer. A 7B model trained on 2T tokens can outperform a 70B model trained on 200B tokens at 1/10th the inference cost. This is why Llama 3 8B is so capable despite its small parameter count.
