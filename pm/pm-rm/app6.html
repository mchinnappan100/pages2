🚀
Release Command Center
A single source of truth for every moving part of your release — development, DevOps, data migration, QA, and stakeholder management — all in one place.
What's in this app
📋
Tracker
A structured spreadsheet interface organised by release section and subsection. Click any cell to edit inline. Click a status badge to cycle through states. Right-click a row for more options.
📊
Dashboard
Live KPI cards, charts, and visualisations that aggregate data from all your sections. Updates instantly as you edit. Charts can be downloaded as PNG.
💾
Persistent storage
All data is saved automatically to your browser's localStorage. Nothing is sent to a server — your data stays local. Export to JSON to back it up or share it.
🔄
Google Sheets compatible
The JSON export format is compatible with the companion Google Sheets version (ReleaseCommandCenter.gs). Import/export freely between both.
Quick start
1
Pick a section from the left panel
The sidebar lists all your release areas. Click one to load its tracker grid on the right.
2
Add or edit items
Click + Add Row to open the form, or click directly into any cell in the table to edit inline.
3
Update status by clicking the badge
Each status badge is a toggle — click it to cycle: Pending → In Progress → Done → Blocked → N/A.
4
Check the Dashboard for the big picture
Switch to the Dashboard tab at any time to see live KPIs and charts across all sections.
Working with the grid
✏️
Inline editing
Click any cell in the Name, Owner, Date, Progress, or Notes columns to edit it directly. Press Enter or click away to save.
🔄
Cycling status
Click a status badge once to advance it to the next state in the cycle:

Pending → In Progress → Done → Blocked → N/A → Pending

Setting an item to Done automatically sets its progress to 100%.
🖱️
Right-click context menu
Right-click any row to access: Edit (full form), Duplicate, quick-set status (Done / In Progress / Blocked), and Delete.
➕
Adding rows
Use the + Add Row button in the top-right to open the full form. Use the small + Add link at the end of each subsection header to add directly into that subsection.
✕
Deleting rows
Hover over a row to reveal the ✕ delete button on the far right, or use the right-click context menu.

Progress bar
📈
Editing progress
Click the percentage number at the end of the progress bar to edit it inline (0–100). The bar colour changes automatically: red below 40%, purple/indigo 40–74%, green/cyan 75–100%.
💡The ⬇ CSV button in the section header exports just the currently visible section, including all its subsections, as a comma-separated file you can open in Excel or Sheets.

Managing sections
+
Add a section
Click + Add Section in the sidebar. Give it a name, an emoji icon, and a colour. A single default subsection is created automatically — you can rename it inline.
✕
Delete a section
Right-click a section in the sidebar. You'll be asked to confirm before the section and all its data is removed.
💡The sidebar footer always shows your overall progress — the percentage of all items across every section that are marked Done.
💡The done/total counter on each section item in the sidebar updates live as you change statuses.

🍩

📊

🔷
Priority Distribution
Bar chart of High / Medium / Low item counts across all sections.
🌀

Polar area chart showing the workload volume in each section.
🕸️

Radar chart overlaying Done % (green) vs Blocked % (red) for every section — ideal for a release readiness review.
Downloading charts
📸
⬇ PNG
Downloads every chart as a PNG image file — great for pasting into release reports, slide decks, or emails. The ⬇ SVG button also downloads PNG (Canvas-based charts are rasterised).
💡The Dashboard updates live — switch to it at any time while editing the tracker and the charts will already reflect your latest changes.
Exporting data
📦
⬇ JSON (header button)
Exports your entire dataset — all sections, subsections, and rows — as a single structured JSON file. Use this to back up your data, share it with a colleague, or import it into the Google Sheets version.
📄
⬇ CSV (section header)
Exports the currently active section as a CSV file you can open in Excel, Google Sheets, or any spreadsheet tool.
Importing JSON
1
Click ⬆ Import in the header
Opens the import panel with a drag-and-drop zone.
2
Drop or browse for your JSON file
Accepts .json files exported from this app or from the Google Sheets companion script. A preview of sections and item counts is shown before you commit.
3
Choose your import mode
Replace all data — full reset from the file.
Merge sections — append imported sections alongside existing ones.
Merge rows — find matching section names and slot rows in.
4
Click Import Data
Data is loaded into localStorage immediately. The tracker and dashboard update automatically.
⚠️Replace all data is irreversible within the app. Export your current data as JSON before importing if you want a backup.
💡The JSON format is compatible with the Google Sheets version (ReleaseCommandCenter.gs). Export from either, import into either.

Tips for release managers
🎯Use the Section Health Radar in the Dashboard during your go/no-go meeting — it shows at a glance which areas are complete vs still blocked.
📋Right-click → Mark Done is the fastest way to batch-close items during a release call without opening each row.
🔄Export JSON at the end of each release and save it as release-v2.4.0.json. You'll build a historical archive you can import and compare later.
📤Share the HTML file itself with your team — it's fully self-contained. Each person's browser stores their own data locally, so it doubles as a personal release checklist.
