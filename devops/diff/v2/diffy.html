Getting Started
  1. Drop File A (incoming/original) and File B (current/modified) onto the upload zones.
  2. The diff editor opens automatically once both files are loaded.
  3. Click Merge in the toolbar to open the VS Code-style 3-pane merge editor.
  4. Use inline Accept Incoming / Accept Current / Accept Both buttons to resolve each conflict.
  5. Edit the result in the middle pane freely, then Save Merge to download.
Merge Pane Layout
← Incoming (File A)
The original / incoming version. Shown with red highlights for lines that differ from File B.
⭐ Result (File C)
The editable merged output. Updated live as you accept conflicts. Fully editable in Monaco editor.
Current (File B) →
The modified / current version. Shown with green highlights for new/changed lines.
CodeLens Actions (inline per conflict)
Accept Incoming
Uses File A lines for this conflict (blue highlight)
Accept Current
Uses File B lines for this conflict (green highlight)
Accept Both
Appends A lines then B lines (amber highlight)
Ignore (↕)
Undoes a previous choice, marks conflict unresolved
Diff Modes
Semantic
Normalizes whitespace/indentation before diffing — only real content changes appear. Best for Salesforce metadata XML.
Whitespace
Raw byte comparison. Every space, tab, newline difference shown. Best for YAML or exact formatting audits.

Tips
  • The conflict minimap (right edge of the result pane) shows red/green/amber marks. Click a mark to jump to that conflict.
  • Scroll Sync keeps the Incoming and Current panes aligned as you scroll — toggle it off to scroll independently.
  • Use Format → then Semantic for the cleanest Salesforce metadata comparison.
  • The result editor is fully editable — fix anything manually then Save Merge.
  • All Incoming / All Current / Both All buttons resolve every conflict at once.
