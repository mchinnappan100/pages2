DiffForge · Metadata Compare
Getting Started
  1. Drop File A (original) and File B (modified) onto the upload zones, or click to browse.
  2. The diff editor opens automatically once both files are loaded.
  3. Use the toolbar to switch modes, format, and download.
  4. Both editor panes are fully editable — changes are reflected live.
Diff Modes
Inline
Additions and removals interleaved in a single view. Best for focused review of specific changes.
Split
Original on the left, modified on the right. Best for seeing full context of both sides simultaneously.
Diff Types
Semantic
Both files are parsed and re-serialized to canonical form before diffing. Whitespace, indentation, and line-ending differences are hidden — only real content changes appear.
Best for Salesforce metadata XML where reformatting by tools (SFDX, Ant, VS Code) creates noise.
Whitespace
Raw file bytes are compared as-is. Every space, tab, and newline difference is shown. Use this when whitespace itself is significant.
Best for YAML, plain text, or when auditing exact formatting changes.
Format Button

Applies pretty-printing to both editor panes simultaneously:

XML
DOMParser-based — preserves attributes, self-closing tags, namespaces, and processing instructions.
JSON
Parses and re-serializes with 2-space indentation. Reports invalid JSON.
Other
No-op — content is left unchanged.
Tip: Format before switching to Semantic mode to get the cleanest possible diff.

Tips
  • Use Format → then Semantic for the cleanest Salesforce metadata comparison — format first to remove pre-existing indentation drift, then let semantic mode hide any remaining whitespace noise.
  • The word wrap toggle (↵ icon) is useful for long XML attribute lines common in Permission Sets and Profiles.
  • Both sides are editable — you can manually fix differences and then Save B to download the corrected file.
  • Switching diff type or language while editing preserves your in-editor changes before rebuilding the model.
  • The status bar shows live line counts and file sizes, updated after every format or model rebuild.
