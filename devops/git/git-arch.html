Git Object Model

01

Git is fundamentally a content-addressable key-value store. Everything — files, directories, commits, and tags — is stored as one of four object types, each identified by its SHA-1 hash. Two identical files always produce the same hash, meaning Git never stores the same content twice.

Three-Tree Architecture

02

Branching & HEAD

03

A branch is nothing more than a lightweight pointer to a commit — a 41-byte file containing a SHA hash. HEAD is a special pointer that indicates which branch (or commit) is currently checked out. Creating a new branch takes microseconds and zero extra disk space.

Remote Repository & Sync

Git is a Content-Addressable Store

Every object (file, directory, commit) is identified by its SHA-1 hash of its content. Identical files always produce the same hash — Git never stores duplicate content.

Branches Are Cheap Pointers

A branch is simply a 41-byte file containing a commit SHA. Creating, deleting, and renaming branches is instantaneous — there's no data copied, ever.

The Reflog is Your Safety Net

Even after destructive operations like git reset --hard, the reflog preserves every HEAD position for 90 days. "Lost" commits can nearly always be recovered.

Rebase Rewrites History

Rebasing creates new commit objects with new SHA hashes. On a private branch this is great for clean history. On shared branches it causes divergence for every collaborator.

Remote-Tracking Branches are Snapshots

origin/main is a local read-only copy of the remote's main branch at the time of the last fetch. It never updates automatically — only git fetch moves it.

Packfiles Make Git Efficient

Git's packfile format uses delta compression to store only the differences between similar objects. A 1 GB codebase with 10 years of history often packs down to under 100 MB.
