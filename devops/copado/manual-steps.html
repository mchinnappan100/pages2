Copado Manual Steps

1. Overview

In enterprise Salesforce delivery, certain operations require manual execution such as Apex Anonymous scripts for data fixes, backfills, or one-time transformations.

These steps must be managed as first-class deployment artifacts to ensure governance, traceability, and repeatability.

2. What Are Manual Steps?

  • Apex Anonymous scripts
  • Data correction scripts
  • Post-deployment configuration
  • One-time setup activities

4. Link to User Story

  • User Story ID (e.g., US-1234)
  • Script purpose
  • Execution timing (pre/post deployment)

5. Example Apex Script

// Script: post-deploy.apex
// Purpose: Backfill StockKeepingUnit using ProductCode
// Execution: Run after deployment

List products = [
    SELECT Id, ProductCode, StockKeepingUnit
    FROM Product2
    WHERE StockKeepingUnit = NULL
    AND ProductCode != NULL
];

for (Product2 p : products) {
    p.StockKeepingUnit = p.ProductCode;
}

if (!products.isEmpty()) {
    update products;
}

System.debug('Updated records: ' + products.size());

6. Deployment Flow

Commit Script → Link to User Story → Deploy via Copado → Execute Script → Validate

7. Best Practices

Version Control

Always commit scripts to Git. Never share via email.

Idempotency

Ensure scripts can run multiple times safely.

Logging

Add debug logs for verification.

Avoid Hardcoding

Use queries instead of hardcoded IDs.

Rollback Plan

Always define how to revert changes.

Test First

Validate scripts in lower environments.

8. Anti-Patterns

  • Running scripts without version control
  • No documentation
  • Hardcoding production data
  • No rollback plan

9. Summary

Managing Apex Anonymous scripts as structured manual steps ensures strong governance, traceability, and safer deployments. Treat these scripts as production-grade artifacts.
