Copado Promotion Flow

UAT → PROD Validation & Deployment

Q: Does Copado create Git branches for Promotions?

No. Promotions are deployment transaction records inside Copado. Git environment branches (QA/UAT/PROD) remain the source of truth.

Q: When Promotion P1343 is created, where does Copado pull metadata from?

Copado pulls metadata from the Git branch representing UAT, not directly from the UAT Salesforce org.

Q: Does Copado retrieve metadata live from the UAT org?

No. The UAT org is mainly used for testing and validation execution, not as the deployment artifact source.

Q: What is the role of QA and UAT?

QA org validates technical correctness first. After approval, changes are promoted into UAT for business validation. Production promotion uses metadata from the UAT Git branch.

Q: Why use Git instead of the UAT org?

Git ensures reproducibility, consistency, version control, and guarantees the exact tested code is what gets promoted.

Key Takeaways

  • ✅ QA Org = Technical Testing Stage
  • ✅ UAT Org = Business Validation Stage
  • ✅ Metadata source = UAT Git Branch
  • ✅ Validation target = Production Org
  • ✅ Deployment source = Git package build
  • ❌ Promotions are not Git branches
  • ❌ Copado does not create promotion-specific Git branches
  • ❌ Not deployed directly from org metadata
