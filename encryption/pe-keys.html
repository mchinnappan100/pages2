Salesforce Shield Platform Encryption

Knowledge Article – Key Architecture and Secrets Management

Overview

Salesforce Shield Platform Encryption provides advanced data protection by encrypting sensitive data at rest while preserving platform functionality such as search, validation rules, and automation.

  • AES-256 encryption
  • Customer-controlled key management
  • Multiple layers of secrets
  • Secure key derivation architecture
  • Support for Bring Your Own Key (BYOK)

Secrets Architecture Overview

Salesforce uses a multi-layer key derivation model to protect encryption keys. The two primary components are:

Tenant Secret

The Tenant Secret is the primary secret controlled by the customer.

  • Core key material
  • Unique per Salesforce org (tenant)
  • Stored securely in Salesforce database
  • Can be rotated up to once per day
  • Can be generated by Salesforce or provided by customer (BYOK)

Master Secret

The Master Secret is controlled and managed exclusively by Salesforce.

  • Platform-level secret
  • Shared securely across infrastructure
  • Rotated automatically by Salesforce
  • Updated approximately 3 times per year
  • Provides additional protection layer

Key Derivation Process

The actual encryption key used to encrypt data is never stored directly. Instead, Salesforce derives it dynamically using both secrets.

Tenant Secret + Master Secret + Org-specific parameters + Key derivation function = Data Encryption Key (DEK)

This layered model ensures maximum security even if one component is compromised.

Encryption Flow

  1. User enters sensitive data
  2. Platform derives encryption key using:
    • Tenant Secret
    • Master Secret
  3. Data encrypted using AES-256
  4. Encrypted data stored in database
  5. When accessed:
    • Key derived again
    • Data decrypted securely

Bring Your Own Key (BYOK)

BYOK allows customers to generate their own Tenant Secret outside Salesforce and securely import it.

Benefits:
  • Full customer control of encryption keys
  • Compliance with regulatory requirements
  • Ability to revoke access by destroying keys
  • External key lifecycle management

Key Rotation

Tenant Secret Rotation

  • Customer controlled
  • Can rotate daily
  • Creates new derived encryption keys
  • Supports forward secrecy

Master Secret Rotation

  • Managed by Salesforce
  • Occurs automatically
  • Typically 3 times per year
  • Transparent to customers

Security Benefits of Dual-Secret Architecture

  • Defense in depth
  • Customer retains key control
  • Salesforce infrastructure protected
  • Protection against insider threats
  • Strong compliance support (HIPAA, PCI, FedRAMP)
  • Prevents direct access to encryption keys

Best Practices

  • Rotate tenant secrets regularly
  • Use BYOK for highest security environments
  • Protect external keys securely
  • Monitor encryption usage
  • Use encryption for sensitive fields only

Summary

Salesforce Shield Platform Encryption uses a powerful layered encryption architecture combining customer-controlled Tenant Secrets and Salesforce-managed Master Secrets to provide strong, scalable, and secure data protection while maintaining full platform functionality.
