Overview

This server provides tools for sending, listing, marking as read, and replying to Gmail emails via a Python MCP server.

MCP servers config for LLM


      "gmail": {
      "command": "/Users/mchinnappan/.local/bin/uv",
      "args": [
        "--directory",
        "/Users/mchinnappan/MCP_Dev/weather/servers",
        "run",
        "gmail_mcp_server.py"
      ]
},

Sample Usage with LLM supporting MCP servers

- send email via gmail to mohan.chinnappan.n@gmail.com with sender_email as mohan.chinnappan.n5@gmail.com, subject: Hello from greenworld, body: Thank for your kind words! Have a great week!

- list unread emails for mohan.chinnappan.n5@gmail.com top 5 only

- make this UID: 3795 email as READ

- reply to UID: 3854 with reply: Vacation starts from June-15-2025

Sample Usage

Send Email: 
send_email_via_gmail(
    sender_email="mohan.chinnappan.n5@gmail.com",
    recipient_email="mohan.chinnappan.n@gmail.com",
    subject="Hello from greenworld",
    body="Thank for your kind words! Have a great week!"
)
List Unread Emails (top 5): 
list_unread_emails(
    email_address="mohan.chinnappan.n5@gmail.com",
    how_many=5
)
Mark Email as Read (UID 3795): 
mark_email_as_read(
    email_address="mohan.chinnappan.n5@gmail.com",
    uid="3795"
)
Reply to Email (UID 3854): 
reply_to_email_by_uid(
    email_address="mohan.chinnappan.n5@gmail.com",
    uid="3854",
    reply_body="Vacation starts from June-15-2025"
)

Available Tools

send_email_via_gmail

Send an email using Gmail SMTP.

send_email_via_gmail(sender_email, recipient_email, subject, body)
  • sender_email: Gmail address to send from
  • recipient_email: Recipient's email address
  • subject: Email subject
  • body: Email body

list_unread_emails

List the latest unread emails, including a preview of the body.

list_unread_emails(email_address, how_many=10, body_lines=3)
  • email_address: Gmail address to check
  • how_many: Max number of unread emails to list (default: 10)
  • body_lines: Number of lines from the email body to preview (default: 3)

mark_email_as_read

Mark an email as read (seen) using its UID.

mark_email_as_read(email_address, uid)
  • email_address: Gmail address to use
  • uid: UID of the email to mark as read

reply_to_email_by_uid

Reply to an email by UID.

reply_to_email_by_uid(email_address, uid, reply_body)
  • email_address: Gmail address to use for replying
  • uid: UID of the email to reply to
  • reply_body: The body of the reply email

Setup Notes

  • Store your Gmail app passwords in ~/.gmail/access.json as:
    { "your_email@gmail.com": "your-app-password" }
  • Enable IMAP and App Passwords in your Google Account settings.
  • All tools require valid credentials and network access to Gmail servers.
