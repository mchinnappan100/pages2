01 — Overview

WHAT IS MCP?

The Model Context Protocol (MCP) is an open standard published by Anthropic in November 2024 that defines a universal interface for connecting AI assistants to systems where data lives — content repositories, business tools, APIs, databases, and development environments.

Before MCP, every AI integration required a custom, point-to-point connection. An AI connecting to 10 data sources required 10 separate integrations, each with bespoke auth, data-formatting, and error-handling logic. MCP solves this with a single standardized interface that works across any LLM host and any data provider.

Core Analogy

MCP is to AI integrations what USB-C is to device peripherals — one universal connector standard that eliminates incompatible, proprietary adapters. M models + N data sources = M+N implementations, not M×N.

The M×N Problem Solved

02 — Architecture

HOW MCP WORKS

MCP follows a client-server architecture mediated by a host application. The host (e.g. Claude Desktop) manages one or more MCP clients, each maintaining a persistent connection to one MCP server that exposes specific capabilities.

Connection Lifecycle

1. Host spawns client → 2. Client connects via transport → 3. Mutual capability negotiation (initialize / initialized) → 4. Client discovers tools/resources/prompts → 5. LLM invokes tools during inference → 6. Client forwards, streams responses back → 7. Graceful shutdown.

Host

The AI application that orchestrates MCP clients and controls LLM access. The host enforces permissions, manages the security boundary, and decides which servers to expose to the model.

Client

A protocol layer inside the host that maintains exactly one stateful connection to one MCP server. Handles JSON-RPC serialization, transport, and capability negotiation automatically.

Server

A lightweight process (local subprocess or remote HTTP) that exposes tools, resources, and prompts. One server per external system — GitHub, Postgres, Slack, your custom API.

03 — Core Primitives

TOOLS, RESOURCES & PROMPTS

MCP defines six core primitives that cover every possible way an AI might interact with external systems. Each has a clearly defined ownership model.

⚡
Tools
Model-controlled · Executable actions
Functions the LLM can call to perform actions or retrieve computed results. Defined with JSON Schema parameters. The model decides when to call them.
📄
Resources
App-controlled · Data & context
Structured data sources — files, DB records, API responses — that clients read and inject into context. Like GET endpoints: retrieval, not mutation.
💬
Prompts
User-controlled · Reusable templates
Pre-defined prompt templates exposed by servers. Users select them from the host UI. Accept arguments and embed resource contents for consistent interactions.
🌊
Sampling
Server → Client · Agentic recursion
Servers can request LLM completions from the client — enabling nested agentic patterns. The host always retains control and can reject sampling requests.
🔔
Notifications
Bidirectional · Progress & updates
Async progress events from server to client during long operations, and cancellation notifications from client to server. Keeps UIs responsive.
🛠
Roots
Client → Server · Scope boundary
Filesystem paths or URIs the client exposes as authorized scope. Servers use roots to understand which resources they may access — a principled permission boundary.
04 — Transport Layer

TRANSPORT MECHANISMS

MCP separates protocol from transport. All messages are JSON-RPC 2.0 over one of three transports, chosen based on deployment topology.

TransportUse CaseDirectionStatus
stdio Local servers spawned as subprocess by host. Gold standard for IDE integrations and desktop apps. Client writes JSON to stdin, server writes to stdout. Bidirectional via pipes Stable
HTTP + SSE Remote servers over the network. Server pushes events via Server-Sent Events; client sends messages via POST. Good for shared multi-client deployments. SSE (server→client), POST (client→server) Stable
Streamable HTTP Simplified HTTP transport replacing SSE. Single POST endpoint with streaming response. Better for serverless and HTTP/2 environments. Streaming POST 2025 Spec
WebSockets Community extension for persistent bidirectional connections. Useful for high-frequency tool calls or real-time collaborative systems. Full-duplex Community
JSON tools/call request — JSON-RPC 2.0

Example: File System MCP Server

Exposes a directory to Claude — list files, read content, write files, and search for text patterns. Uses stdio transport (default for local servers).

Node.js / TypeScript mcp-server-filesystem.ts 
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";
import fs from "fs/promises";
import path from "path";

const ROOT_DIR = process.env.MCP_ROOT ?? process.cwd();

const server = new McpServer({
  name: "filesystem-server",
  version: "1.0.0",
});

// ── Tool: list_files ─────────────────────────────────
server.tool(
  "list_files",
  "List files and directories at a given path",
  {
    dir: z.string().optional().describe("Subdirectory relative to root"),
    extension: z.string().optional().describe("Filter by extension e.g. '.ts'"),
  },
  async ({ dir = "", extension }) => {
    const target = path.join(ROOT_DIR, dir);
    const entries = await fs.readdir(target, { withFileTypes: true });
    const results = entries
      .filter((e) => !extension || e.name.endsWith(extension))
      .map((e) => ({
        name: e.name,
        type: e.isDirectory() ? "directory" : "file",
        path: path.join(dir, e.name),
      }));
    return { content: [{ type: "text", text: JSON.stringify(results, null, 2) }] };
  }
);

// ── Tool: read_file ──────────────────────────────────
server.tool(
  "read_file",
  "Read the full contents of a file",
  { filepath: z.string().describe("Path relative to root") },
  async ({ filepath }) => {
    const content = await fs.readFile(path.join(ROOT_DIR, filepath), "utf-8");
    return { content: [{ type: "text", text: content }] };
  }
);

// ── Tool: write_file ─────────────────────────────────
server.tool(
  "write_file",
  "Write or overwrite a file with new content",
  {
    filepath: z.string().describe("Path relative to root"),
    content: z.string().describe("Content to write"),
  },
  async ({ filepath, content }) => {
    const fullPath = path.join(ROOT_DIR, filepath);
    await fs.mkdir(path.dirname(fullPath), { recursive: true });
    await fs.writeFile(fullPath, content, "utf-8");
    return { content: [{ type: "text", text: `Written to ${filepath}` }] };
  }
);

// ── Tool: search_files ───────────────────────────────
server.tool(
  "search_files",
  "Recursively search for a text pattern across all files",
  {
    query: z.string().describe("Text to search for"),
    dir: z.string().optional().describe("Directory to search (default: root)"),
  },
  async ({ query, dir = "" }) => {
    const hits: string[] = [];

    async function walk(cur: string) {
      const entries = await fs.readdir(path.join(ROOT_DIR, cur), { withFileTypes: true });
      for (const e of entries) {
        const rel = path.join(cur, e.name);
        if (e.isDirectory() && !e.name.startsWith(".")) {
          await walk(rel);
        } else if (e.isFile()) {
          try {
            const text = await fs.readFile(path.join(ROOT_DIR, rel), "utf-8");
            if (text.includes(query)) {
              const lines = text.split("\n")
                .map((l, i) => ({ n: i + 1, l }))
                .filter(({ l }) => l.includes(query))
                .map(({ n, l }) => `  Line ${n}: ${l.trim()}`);
              hits.push(`${rel}:\n${lines.join("\n")}`);
            }
          } catch {}
        }
      }
    }

    await walk(dir);
    return {
      content: [{
        type: "text",
        text: hits.length ? hits.join("\n\n") : `No matches for "${query}"`,
      }],
    };
  }
);

// ── Resource: root directory listing ─────────────────
server.resource("root-directory", "file:///root", async () => {
  const entries = await fs.readdir(ROOT_DIR, { withFileTypes: true });
  const listing = entries
    .map((e) => `[${e.isDirectory() ? "DIR " : "FILE"}] ${e.name}`)
    .join("\n");
  return {
    contents: [{
      uri: "file:///root",
      mimeType: "text/plain",
      text: `Root: ${ROOT_DIR}\n\n${listing}`,
    }],
  };
});

// ── Start ────────────────────────────────────────────
const transport = new StdioServerTransport();
await server.connect(transport);
console.error("Filesystem MCP Server running on stdio");
Example: PostgreSQL Database Server

Gives Claude read/write access to a Postgres database — schema introspection, safe parameterized SELECT queries, and row insertion.

Node.js / TypeScript mcp-server-postgres.ts 
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";
import pg from "pg";

const pool = new pg.Pool({ connectionString: process.env.DATABASE_URL });
const server = new McpServer({ name: "postgres-server", version: "1.0.0" });

// ── Tool: list_tables ────────────────────────────────
server.tool("list_tables", "List all public tables with sizes", {}, async () => {
  const { rows } = await pool.query(`
    SELECT tablename AS name,
           pg_size_pretty(pg_relation_size(quote_ident(tablename))) AS size
    FROM pg_tables WHERE schemaname = 'public' ORDER BY tablename
  `);
  return { content: [{ type: "text", text: JSON.stringify(rows, null, 2) }] };
});

// ── Tool: describe_table ─────────────────────────────
server.tool(
  "describe_table",
  "Get column schema for a table",
  { table: z.string().describe("Table name") },
  async ({ table }) => {
    const { rows } = await pool.query(
      `SELECT column_name, data_type, is_nullable, column_default
       FROM information_schema.columns
       WHERE table_name = $1 AND table_schema = 'public'
       ORDER BY ordinal_position`,
      [table]
    );
    return { content: [{ type: "text", text: JSON.stringify(rows, null, 2) }] };
  }
);

// ── Tool: query (SELECT only) ────────────────────────
server.tool(
  "query",
  "Run a read-only SQL SELECT query with optional parameters",
  {
    sql: z.string().describe("SELECT query to execute"),
    params: z.array(z.string()).optional().describe("Query params ($1, $2...)"),
  },
  async ({ sql, params = [] }) => {
    const norm = sql.trim().toUpperCase();
    if (!norm.startsWith("SELECT") && !norm.startsWith("WITH")) {
      return {
        content: [{ type: "text", text: "Error: only SELECT queries allowed" }],
        isError: true,
      };
    }
    const { rows } = await pool.query(sql, params);
    return { content: [{ type: "text", text: JSON.stringify(rows, null, 2) }] };
  }
);

// ── Tool: insert_row ─────────────────────────────────
server.tool(
  "insert_row",
  "Insert a row into a table and return the new record",
  {
    table: z.string(),
    data: z.record(z.string(), z.any()).describe("Column-value pairs to insert"),
  },
  async ({ table, data }) => {
    const cols = Object.keys(data);
    const vals = Object.values(data);
    const placeholders = cols.map((_, i) => `$${i + 1}`).join(", ");
    const { rows } = await pool.query(
      `INSERT INTO ${table} (${cols.join(", ")}) VALUES (${placeholders}) RETURNING *`,
      vals
    );
    return { content: [{ type: "text", text: JSON.stringify(rows[0], null, 2) }] };
  }
);

// ── Prompt: data analyst ─────────────────────────────
server.prompt("data-analyst", "Expert SQL analyst for this database", [], async () => ({
  messages: [{
    role: "user",
    content: {
      type: "text",
      text: "You are an expert SQL data analyst. Start by listing all tables, then help the user explore and analyze data. Always use parameterized queries.",
    },
  }],
}));

const transport = new StdioServerTransport();
await server.connect(transport);
console.error("PostgreSQL MCP Server running");
Example: Remote HTTP + SSE Server

A server accessible over the network via Server-Sent Events — suitable for shared team deployments, cloud hosting, and multi-user setups.

Node.js / Express mcp-server-http.ts 
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { SSEServerTransport } from "@modelcontextprotocol/sdk/server/sse.js";
import express from "express";
import { z } from "zod";

const app = express();
app.use(express.json());

// Per-session transport map: sessionId -> transport
const sessions = new Map<string, SSEServerTransport>();

function createServer(): McpServer {
  const server = new McpServer({ name: "remote-server", version: "1.0.0" });

  // ── Tool: get_weather (Open-Meteo, no API key) ─────
  server.tool(
    "get_weather",
    "Get current weather for a location via Open-Meteo",
    {
      city: z.string(),
      lat: z.number(),
      lon: z.number(),
    },
    async ({ lat, lon, city }) => {
      const url = `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}¤t_weather=true`;
      const res = await fetch(url);
      const data = await res.json();
      const w = data.current_weather;
      return {
        content: [{
          type: "text",
          text: `${city}: ${w.temperature}°C, wind ${w.windspeed} km/h`,
        }],
      };
    }
  );

  // ── Tool: fetch_json_api ──────────────────────────
  server.tool(
    "fetch_json_api",
    "Fetch JSON from any public GET endpoint and return the result",
    { url: z.string().url().describe("Public API endpoint URL") },
    async ({ url }) => {
      const res = await fetch(url, {
        headers: { Accept: "application/json" },
      });
      const json = await res.json();
      return {
        content: [{
          type: "text",
          text: JSON.stringify(json, null, 2).slice(0, 4000),
        }],
      };
    }
  );

  return server;
}

// GET /sse — client opens long-lived SSE connection
app.get("/sse", async (req, res) => {
  const transport = new SSEServerTransport("/messages", res);
  const server = createServer();
  sessions.set(transport.sessionId, transport);

  res.on("close", () => {
    sessions.delete(transport.sessionId);
    console.log(`Session closed: ${transport.sessionId}`);
  });

  await server.connect(transport);
  console.log(`New session: ${transport.sessionId}`);
});

// POST /messages — client sends tool requests
app.post("/messages", async (req, res) => {
  const sessionId = req.query.sessionId as string;
  const transport = sessions.get(sessionId);
  if (!transport) return res.status(404).json({ error: "Session not found" });
  await transport.handlePostMessage(req, res);
});

app.get("/health", (_, res) => res.json({ status: "ok", sessions: sessions.size }));

app.listen(3333, () => {
  console.log("Remote MCP Server: http://localhost:3333");
  console.log("  SSE:      GET  /sse");
  console.log("  Messages: POST /messages?sessionId=xxx");
});
Example: Programmatic Client + Full Claude Agentic Loop

Connect to any MCP server, discover its capabilities, and run a complete multi-turn Claude reasoning loop with automatic tool execution.

Node.js / TypeScript mcp-client-claude.ts 
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";
import Anthropic from "@anthropic-ai/sdk";

async function createMCPClient(command: string, args: string[] = []) {
  const client = new Client(
    { name: "my-mcp-client", version: "1.0.0" },
    { capabilities: { tools: {}, resources: {}, prompts: {} } }
  );

  const transport = new StdioClientTransport({
    command,
    args,
    env: { ...process.env, MCP_ROOT: "./workspace" },
  });

  await client.connect(transport);
  return client;
}

async function runAgenticLoop(client: Client, userMessage: string) {
  const anthropic = new Anthropic();

  // Discover and convert MCP tools to Anthropic format
  const { tools: mcpTools } = await client.listTools();
  const claudeTools = mcpTools.map((t) => ({
    name: t.name,
    description: t.description ?? "",
    input_schema: t.inputSchema as Anthropic.Tool["input_schema"],
  }));

  const messages: Anthropic.MessageParam[] = [
    { role: "user", content: userMessage },
  ];

  console.log(`\nUser: ${userMessage}\n`);

  // Agentic loop: reason → act → observe until done
  while (true) {
    const response = await anthropic.messages.create({
      model: "claude-opus-4-5",
      max_tokens: 4096,
      tools: claudeTools,
      messages,
    });

    if (response.stop_reason === "end_turn") {
      const text = response.content
        .filter((b) => b.type === "text")
        .map((b) => (b as Anthropic.TextBlock).text)
        .join("");
      console.log(`Claude: ${text}`);
      return text;
    }

    if (response.stop_reason === "tool_use") {
      messages.push({ role: "assistant", content: response.content });

      const toolResults: Anthropic.ToolResultBlockParam[] = [];

      for (const block of response.content) {
        if (block.type !== "tool_use") continue;

        console.log(`  Tool call: ${block.name}(${JSON.stringify(block.input)})`);

        const result = await client.callTool({
          name: block.name,
          arguments: block.input as Record<string, unknown>,
        });

        const resultText = result.content
          .filter((c) => c.type === "text")
          .map((c) => (c as { text: string }).text)
          .join("\n");

        console.log(`  Result: ${resultText.slice(0, 120)}...`);

        toolResults.push({
          type: "tool_result",
          tool_use_id: block.id,
          content: resultText,
        });
      }

      messages.push({ role: "user", content: toolResults });
    }
  }
}

// Main: connect to filesystem server and run a task
async function main() {
  const client = await createMCPClient("npx", ["ts-node", "mcp-server-filesystem.ts"]);

  const { tools } = await client.listTools();
  console.log(`Connected. Tools: ${tools.map((t) => t.name).join(", ")}`);

  await runAgenticLoop(
    client,
    "Find all TypeScript files, read the main entry point, and write a one-paragraph summary to SUMMARY.md"
  );

  await client.close();
}

main().catch(console.error);

Example: Web Research Server

Lets Claude search the web via DuckDuckGo, fetch URLs, and extract clean article text — turning any AI into a research assistant with no API keys required.

Python mcp_server_web.py 
"""
MCP Web Research Server
Tools: search_web, fetch_url, extract_article
No API keys required - uses DuckDuckGo HTML search + httpx
"""

import asyncio
import json
import re
from urllib.parse import quote

import httpx
from bs4 import BeautifulSoup
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types

app = Server("web-research-server")


@app.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="search_web",
            description="Search the web using DuckDuckGo and return top results with titles, snippets, and URLs",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "Search query"},
                    "max_results": {"type": "integer", "default": 5},
                },
                "required": ["query"],
            },
        ),
        types.Tool(
            name="fetch_url",
            description="Fetch raw content from a URL",
            inputSchema={
                "type": "object",
                "properties": {
                    "url": {"type": "string"},
                    "timeout": {"type": "integer", "default": 10},
                },
                "required": ["url"],
            },
        ),
        types.Tool(
            name="extract_article",
            description="Fetch a URL and extract clean, readable article text (strips ads, nav, scripts)",
            inputSchema={
                "type": "object",
                "properties": {"url": {"type": "string"}},
                "required": ["url"],
            },
        ),
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[types.TextContent]:

    headers = {"User-Agent": "Mozilla/5.0 (compatible; MCPBot/1.0)"}

    if name == "search_web":
        query = arguments["query"]
        max_results = arguments.get("max_results", 5)

        url = f"https://html.duckduckgo.com/html/?q={quote(query)}"
        async with httpx.AsyncClient(follow_redirects=True, timeout=15) as client:
            response = await client.get(url, headers=headers)

        soup = BeautifulSoup(response.text, "html.parser")
        results = []
        for r in soup.select(".result")[:max_results]:
            title = r.select_one(".result__title")
            snippet = r.select_one(".result__snippet")
            link = r.select_one(".result__url")
            if title:
                results.append({
                    "title": title.get_text(strip=True),
                    "snippet": snippet.get_text(strip=True) if snippet else "",
                    "url": link.get_text(strip=True) if link else "",
                })

        return [types.TextContent(type="text", text=json.dumps(results, indent=2))]

    elif name == "fetch_url":
        async with httpx.AsyncClient(
            follow_redirects=True,
            timeout=arguments.get("timeout", 10)
        ) as client:
            response = await client.get(arguments["url"], headers=headers)
        return [types.TextContent(
            type="text",
            text=f"Status: {response.status_code}\n\n{response.text[:6000]}",
        )]

    elif name == "extract_article":
        async with httpx.AsyncClient(follow_redirects=True, timeout=15) as client:
            response = await client.get(arguments["url"], headers=headers)

        soup = BeautifulSoup(response.text, "html.parser")

        # Strip noise elements
        for tag in soup(["script", "style", "nav", "header", "footer", "aside"]):
            tag.decompose()

        # Find main content area
        article = (
            soup.find("article")
            or soup.find(class_=re.compile(r"content|article|post|entry", re.I))
            or soup.find("main")
            or soup.body
        )

        text = article.get_text(separator="\n", strip=True) if article else ""
        text = re.sub(r"\n{3,}", "\n\n", text)  # collapse whitespace
        title = soup.title.string if soup.title else arguments["url"]

        return [types.TextContent(type="text", text=f"# {title}\n\n{text[:5000]}")]

    raise ValueError(f"Unknown tool: {name}")


@app.list_resources()
async def list_resources() -> list[types.Resource]:
    return [types.Resource(
        uri="info://server",
        name="Server Info",
        mimeType="text/plain",
    )]


@app.read_resource()
async def read_resource(uri: str) -> str:
    return "MCP Web Research Server — Tools: search_web, fetch_url, extract_article"


async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write, app.create_initialization_options())


if __name__ == "__main__":
    asyncio.run(main())
Example: Sandboxed Code Execution Server

Lets Claude write and execute Python code in a controlled subprocess with timeout enforcement, stdout/stderr capture, and a safety check layer.

Python mcp_server_executor.py 
"""
MCP Code Execution Server
Executes Python code in an isolated subprocess with timeout and capture.
"""

import asyncio
import json
import os
import subprocess
import sys
import tempfile
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types

app = Server("code-execution-server")

BLOCKED_PATTERNS = [
    "__import__", "exec(", "eval(", "open(", "os.", "sys.",
    "subprocess", "socket", "shutil", "importlib",
]


def is_safe(code: str) -> tuple[bool, str]:
    for pat in BLOCKED_PATTERNS:
        if pat in code:
            return False, f"Blocked: '{pat}'"
    return True, ""


@app.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="execute_python",
            description=(
                "Execute Python code in a sandbox and return stdout/stderr. "
                "Best for: data analysis, algorithms, math, and demonstrations."
            ),
            inputSchema={
                "type": "object",
                "properties": {
                    "code": {"type": "string", "description": "Python code to run"},
                    "timeout": {"type": "integer", "default": 10, "maximum": 30},
                },
                "required": ["code"],
            },
        ),

            name="analyze_data",

            inputSchema={
                "type": "object",

                        "type": "array",
                        "items": {"type": "number"},
                    },
                    "operation": {
                        "type": "string",
                        "enum": ["stats", "histogram", "sort"],
                    },
                },
                "required": ["data", "operation"],
            },
        ),
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[types.TextContent]:

    if name == "execute_python":
        code = arguments["code"]
        timeout = min(int(arguments.get("timeout", 10)), 30)

        ok, reason = is_safe(code)
        if not ok:
            return [types.TextContent(type="text", text=f"Blocked: {reason}")]

        with tempfile.NamedTemporaryFile(mode="w", suffix=".py", delete=False) as f:
            f.write(code)
            tmpfile = f.name

        try:
            result = subprocess.run(
                [sys.executable, tmpfile],
                capture_output=True,
                text=True,
                timeout=timeout,
            )
            parts = []
            if result.stdout:
                parts.append(f"stdout:\n{result.stdout.strip()}")
            if result.stderr:
                parts.append(f"stderr:\n{result.stderr.strip()}")
            if result.returncode != 0:
                parts.append(f"exit code: {result.returncode}")
            return [types.TextContent(
                type="text",
                text="\n\n".join(parts) or "Executed successfully (no output)",
            )]
        except subprocess.TimeoutExpired:
            return [types.TextContent(type="text", text=f"Timeout after {timeout}s")]
        finally:
            os.unlink(tmpfile)

    elif name == "analyze_data":
        import statistics
        data = arguments["data"]
        op = arguments["operation"]

        if op == "stats":
            result = {
                "count": len(data),
                "min": min(data),
                "max": max(data),
                "mean": statistics.mean(data),
                "median": statistics.median(data),
                "stdev": round(statistics.stdev(data), 4) if len(data) > 1 else 0,
            }
            return [types.TextContent(type="text", text=json.dumps(result, indent=2))]

        elif op == "histogram":
            mn, mx = min(data), max(data)
            buckets, width = 8, (mx - mn) / 8 or 1
            counts = [0] * buckets
            for v in data:
                idx = min(int((v - mn) / width), buckets - 1)
                counts[idx] += 1
            lines = [f"Histogram of {len(data)} values:"]
            for i, c in enumerate(counts):
                lo, hi = mn + i * width, mn + (i + 1) * width
                lines.append(f"  {lo:7.2f}–{hi:7.2f} | {'█' * c} ({c})")
            return [types.TextContent(type="text", text="\n".join(lines))]

        elif op == "sort":
            return [types.TextContent(type="text", text=str(sorted(data)))]

    raise ValueError(f"Unknown tool: {name}")

    async with stdio_server() as (read, write):
        await app.run(read, write, app.create_initialization_options())


if __name__ == "__main__":
    asyncio.run(main())
FastMCP — High-Level Decorator Pattern

FastMCP infers JSON Schema from Python type hints and docstrings. Minimal boilerplate, maximum productivity. Includes a working todo manager, unit converter, and reusable prompt.

"""
FastMCP pattern: schemas inferred from type hints + docstrings.
Run: python mcp_server_fastmcp.py
"""

import datetime
from typing import Annotated
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("productivity-server")

# ── In-memory todo store ──────────────────────────────
_todos: list[dict] = []


@mcp.tool()
def add_todo(
    title: Annotated[str, "Task title"],
    priority: Annotated[str, "Priority: low / medium / high"] = "medium",
    due_date: Annotated[str | None, "Due date YYYY-MM-DD"] = None,
) -> str:
    """Add a task to the todo list."""
    todo = {
        "id": len(_todos) + 1,
        "title": title,
        "priority": priority,
        "due_date": due_date,
        "done": False,
        "created": datetime.datetime.now().isoformat(),
    }
    _todos.append(todo)
    return f"Added task #{todo['id']}: {title} [{priority}]"


@mcp.tool()
def list_todos(
    filter_by: Annotated[str, "all / done / pending"] = "all",
    priority: Annotated[str | None, "Filter: low / medium / high"] = None,
) -> str:
    """List tasks with optional filters."""
    items = [t for t in _todos if
             filter_by == "all" or
             (filter_by == "done" and t["done"]) or
             (filter_by == "pending" and not t["done"])]
    if priority:
        items = [t for t in items if t["priority"] == priority]
    if not items:
        return "No tasks found."
    lines = []
    for t in items:
        due = f" (due: {t['due_date']})" if t["due_date"] else ""
        lines.append(f"  {'✓' if t['done'] else '○'} #{t['id']} [{t['priority']}] {t['title']}{due}")
    return f"Tasks ({len(items)}):\n" + "\n".join(lines)


@mcp.tool()
def complete_todo(task_id: Annotated[int, "Task ID to mark done"]) -> str:
    """Mark a task as complete."""
    for t in _todos:
        if t["id"] == task_id:
            t["done"] = True
            return f"Task #{task_id} '{t['title']}' completed."
    return f"Task #{task_id} not found."


@mcp.tool()
def convert_units(
    value: Annotated[float, "Value to convert"],
    from_unit: Annotated[str, "Source unit e.g. km, kg, celsius"],
    to_unit: Annotated[str, "Target unit e.g. miles, lbs, fahrenheit"],
) -> str:
    """Convert between common units of measurement."""
    table = {
        ("km", "miles"): lambda v: v * 0.621371,
        ("miles", "km"): lambda v: v * 1.60934,
        ("kg", "lbs"): lambda v: v * 2.20462,
        ("lbs", "kg"): lambda v: v * 0.453592,
        ("celsius", "fahrenheit"): lambda v: v * 9 / 5 + 32,
        ("fahrenheit", "celsius"): lambda v: (v - 32) * 5 / 9,
        ("meters", "feet"): lambda v: v * 3.28084,
        ("feet", "meters"): lambda v: v * 0.3048,
        ("liters", "gallons"): lambda v: v * 0.264172,
        ("gallons", "liters"): lambda v: v * 3.78541,
    }
    key = (from_unit.lower(), to_unit.lower())
    if key not in table:
        return f"Unsupported: {from_unit} to {to_unit}"
    return f"{value} {from_unit} = {table[key](value):.4f} {to_unit}"


# ── Resource: live todo summary ──────────────────────
@mcp.resource("todos://summary")
def todo_summary() -> str:
    """Inject current todo stats into the model's context."""
    done = sum(1 for t in _todos if t["done"])
    return f"Todos: {len(_todos)} total, {done} done, {len(_todos)-done} pending."


# ── Reusable prompt ───────────────────────────────────
@mcp.prompt()
def productivity_coach() -> str:
    """A focused productivity coach that manages your task list."""
    return (
        "You are a helpful productivity coach with access to the user's task list. "
        f"Current state: {todo_summary()}\n\n"
        "Help the user organize work, prioritize tasks, and hit deadlines. "
        "When adding tasks, always confirm priority and ask about due dates."
    )


if __name__ == "__main__":
    mcp.run()  # defaults to stdio
Example: Complete Python MCP Client + Claude

A full agentic client: connects to a server, discovers tools, builds the Claude agentic loop, and streams results. Drop-in for any MCP server.

Python mcp_client.py 
"""
Complete MCP Client with Claude agentic loop.
Works with any MCP stdio server.
"""

import asyncio
import json
from contextlib import AsyncExitStack

import anthropic
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client


class MCPAgent:
    """Connects to an MCP server and runs a Claude agentic loop."""

    def __init__(self, server_script: str, model: str = "claude-opus-4-5"):
        self.server_script = server_script
        self.model = model
        self.session: ClientSession | None = None
        self.claude = anthropic.Anthropic()
        self._stack = AsyncExitStack()

    async def connect(self):
        params = StdioServerParameters(command="python", args=[self.server_script])
        read, write = await self._stack.enter_async_context(stdio_client(params))
        self.session = await self._stack.enter_async_context(
            ClientSession(read, write)
        )
        await self.session.initialize()

        tools = (await self.session.list_tools()).tools
        print(f"Connected to {self.server_script}")
        print(f"Available tools: {', '.join(t.name for t in tools)}\n")

    async def _get_claude_tools(self) -> list[dict]:
        tools = (await self.session.list_tools()).tools
        return [
            {
                "name": t.name,
                "description": t.description or "",
                "input_schema": t.inputSchema,
            }
            for t in tools
        ]

    async def run(self, prompt: str) -> str:
        """Run a full agentic session for a given user prompt."""
        tools = await self._get_claude_tools()
        messages = [{"role": "user", "content": prompt}]

        print(f"User: {prompt}\n")

        while True:
            response = self.claude.messages.create(
                model=self.model,
                max_tokens=4096,
                tools=tools,
                messages=messages,
            )

            # Final answer
            if response.stop_reason == "end_turn":
                text = "".join(
                    b.text for b in response.content if hasattr(b, "text")
                )
                print(f"Claude: {text}")
                return text

            # Tool calls
            if response.stop_reason == "tool_use":
                messages.append({"role": "assistant", "content": response.content})
                results = []

                for block in response.content:
                    if block.type != "tool_use":
                        continue

                    print(f"  → {block.name}({json.dumps(block.input, indent=2)})")

                    result = await self.session.call_tool(block.name, block.input)
                    result_text = " ".join(
                        c.text for c in result.content if hasattr(c, "text")
                    )

                    print(f"     {result_text[:150]}...\n")

                    results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result_text,
                    })

                messages.append({"role": "user", "content": results})

    async def close(self):
        await self._stack.aclose()


async def main():
    # Example: connect to the web research server
    agent = MCPAgent("mcp_server_web.py")
    await agent.connect()

    tasks = [
        "Search for 'Model Context Protocol 2025' and give me a summary of the latest news.",
        "Fetch https://modelcontextprotocol.io and tell me what MCP is according to the official site.",
    ]

    for task in tasks:
        print("=" * 60)
        await agent.run(task)
        print()

    await agent.close()


if __name__ == "__main__":
    asyncio.run(main())

08 — Security

SECURITY MODEL

MCP operates under a clear trust hierarchy: the host is the root of trust, clients inherit host permissions, and servers are explicitly sandboxed. The protocol itself enforces a clean separation between these layers.

🔐

Human-in-the-Loop

Hosts must expose tool calls to users before execution. Users must be able to review and approve tool invocations — especially for writes, deletions, or external API calls.

🏝️

Process Isolation

Each MCP server runs in its own subprocess. Servers cannot access host memory or other server processes — the only channel is the MCP protocol over stdio or HTTP.

🚧

Roots & Scope

Clients communicate authorized scope via "roots" — a list of filesystem paths or URIs. Servers should respect roots and avoid accessing resources outside them.

🔑

Secret Management

Never hardcode credentials in server code. Pass secrets via environment variables set in the host config. The host is responsible for securely injecting secrets to spawned servers.

⚠️

Prompt Injection Risk

Tool results containing untrusted web content could attempt prompt injection. Validate and sanitize tool outputs. Be especially vigilant with web scraping and user-supplied data.

📋

Audit Logging

Log all tool invocations with arguments and results. Agentic systems taking real-world actions must provide a complete, tamper-evident audit trail for compliance and debugging.

⚠ Agentic Risk Checklist

When deploying agents with broad tool access: (1) Apply minimal scope by default. (2) Require explicit user confirmation for destructive operations. (3) Rate-limit tool calls per session. (4) Implement circuit breakers for unexpected tool usage patterns. (5) Never expose database write tools without human review of the generated SQL.
