The Problem

The M×N Integration Hell

Imagine you have M AI models (GPT-4, Claude, Gemini, Llama, Mistral…) and N tools (GitHub, Slack, databases, file systems, APIs…). Without a standard, each model needs a custom integration for every tool. That's M × N connectors to build and maintain.

The key insight: Instead of every AI model knowing how to talk to every tool, you only need each model to speak one common language (MCP), and each tool to expose itself via the same standard. The complexity drops from multiplicative to additive.

Architecture

How MCP Works

MCP defines a three-layer architecture: a Host (the AI application), a Client (connector inside the host), and one or more Servers (tools/data sources). Communication uses JSON-RPC 2.0 over stdio or HTTP/SSE.

H->>C: Initialize connection C->>S: initialize {protocolVersion, capabilities} S-->>C: {capabilities: {tools, resources}} H->>C: List available tools C->>S: tools/list S-->>C: [{name:"read_file"}, {name:"list_dir"}] C-->>H: Available tools H->>C: Call tool C->>S: tools/call {name:"list_dir", arguments:{path:"."}} S->>T: Execute list_dir(".") T-->>S: ["main.py","README.md","src/"] S-->>C: {content:[{type:"text", text:"..."}]} C-->>H: Tool result H-->>U: "Your project contains: main.py, README.md, src/"
🖥️
Host

The AI application (Claude Desktop, Cursor, your app). Manages MCP clients and orchestrates LLM interactions.

🔌
Client

Runs inside the host. Maintains a 1:1 session with an MCP server, handles protocol negotiation and message routing.

⚙️
Server

Exposes Tools, Resources, and Prompts via the standard protocol. This is where your integrations live.

Core Primitives

What Servers Expose

mindmap root((MCP Server)) Tools Executable functions Has JSON Schema input Model-invoked actions Examples: read_file, query_db, send_email Resources Data sources and files URI-addressed content Application-controlled Examples: file contents, DB rows, API responses Prompts Reusable prompt templates Parameterized workflows User-selectable snippets Examples: code review, summarize doc
Tools

Functions the model can call. Like REST endpoints — they take typed inputs and return results. Enable real-world actions.

Resources

Data the application exposes to the model. Addressed by URI. Like GET endpoints — read-only structured data.

Prompts

Reusable prompt templates controlled by the server. Users can invoke them for consistent, well-crafted workflows.

Code Examples

Building an MCP Server

Write once, connect to any MCP-compatible host. Here's a minimal file-reading MCP server in both Node.js and Python.

npm install @modelcontextprotocol/sdk
server.js — MCP Server with Tools & Resources 
import { McpServer, ResourceTemplate } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { readFile, readdir } from "fs/promises";
import { z } from "zod";

// 1️⃣  Create the MCP server
const server = new McpServer({
  name: "file-explorer",
  version: "1.0.0",
});

// 2️⃣  Expose a TOOL: list directory contents
server.tool(
  "list_directory",
  "List files and folders at a given path",
  { path: z.string().describe("Directory path to list") },
  async ({ path }) => {
    const entries = await readdir(path, { withFileTypes: true });
    const result = entries.map((e) =>
      `${e.isDirectory() ? "📁" : "📄"} ${e.name}`
    ).join("\n");

    return {
      content: [{ type: "text", text: result }],
    };
  }
);

// 3️⃣  Expose a TOOL: read file contents
server.tool(
  "read_file",
  "Read the contents of a file",
  {
    path: z.string().describe("Path to the file"),
    encoding: z.enum(["utf8", "base64"]).default("utf8"),
  },
  async ({ path, encoding }) => {
    const content = await readFile(path, encoding);
    return {
      content: [{ type: "text", text: content }],
    };
  }
);

// 4️⃣  Expose a RESOURCE: project README
server.resource(
  "readme",
  new ResourceTemplate("file://{path}/README.md", { list: undefined }),
  async (uri, { path }) => {
    const text = await readFile(`${path}/README.md`, "utf8");
    return {
      contents: [{ uri: uri.href, mimeType: "text/markdown", text }],
    };
  }
);

// 5️⃣  Start server over stdio (for local use)
const transport = new StdioServerTransport();
await server.connect(transport);
console.error("MCP file-explorer server running…");
client.js — Connecting as an MCP Client 
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";

// Spin up our server as a subprocess
const transport = new StdioClientTransport({
  command: "node",
  args: ["server.js"],
});

const client = new Client({ name: "demo-client", version: "1.0.0" });
await client.connect(transport);

// Discover tools (this is the M+N magic — no hardcoding!)
const { tools } = await client.listTools();
console.log("Available tools:", tools.map((t) => t.name));
// → ["list_directory", "read_file"]

// Call a tool
const result = await client.callTool({
  name: "list_directory",
  arguments: { path: "./src" },
});

console.log(result.content[0].text);
// → 📁 components
// → 📄 index.ts
// → 📄 utils.ts

await client.close();

server.py — MCP Server with Tools & Resources 
import asyncio
import os
from pathlib import Path
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types

# 1️⃣  Create the MCP server
app = Server("file-explorer")


# 2️⃣  List available TOOLS
@app.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="list_directory",
            description="List files and folders at a given path",
            inputSchema={
                "type": "object",
                "properties": {
                    "path": {"type": "string", "description": "Directory path"}
                },
                "required": ["path"],
            },
        ),
        types.Tool(
            name="read_file",
            description="Read the contents of a file",
            inputSchema={
                "type": "object",
                "properties": {
                    "path": {"type": "string", "description": "File path"},
                    "encoding": {"type": "string", "default": "utf-8"},
                },
                "required": ["path"],
            },
        ),
    ]


# 3️⃣  Handle TOOL calls
@app.call_tool()
async def call_tool(
    name: str, arguments: dict
) -> list[types.TextContent]:

    if name == "list_directory":
        path = Path(arguments["path"])
        entries = []
        for entry in sorted(path.iterdir()):
            icon = "📁" if entry.is_dir() else "📄"
            entries.append(f"{icon} {entry.name}")
        return [types.TextContent(type="text", text="\n".join(entries))]

    elif name == "read_file":
        path = Path(arguments["path"])
        encoding = arguments.get("encoding", "utf-8")
        content = path.read_text(encoding=encoding)
        return [types.TextContent(type="text", text=content)]

    raise ValueError(f"Unknown tool: {name}")


# 4️⃣  Expose RESOURCES
@app.list_resources()
async def list_resources() -> list[types.Resource]:
    return [
        types.Resource(
            uri="file://./README.md",
            name="Project README",
            mimeType="text/markdown",
        )
    ]


@app.read_resource()
async def read_resource(uri: str) -> str:
    if uri == "file://./README.md":
        return Path("README.md").read_text()
    raise ValueError(f"Unknown resource: {uri}")


# 5️⃣  Start server over stdio
async def main():
    async with stdio_server() as streams:
        await app.run(
            streams[0], streams[1],
            app.create_initialization_options()
        )

if __name__ == "__main__":
    asyncio.run(main())
client.py — Connecting as an MCP Client 
import asyncio
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

server_params = StdioServerParameters(
    command="python",
    args=["server.py"],
)

async def main():
    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:

            # Initialize the connection
            await session.initialize()

            # Discover tools — the M+N magic!
            tools = await session.list_tools()
            print("Available tools:", [t.name for t in tools.tools])
            # → ['list_directory', 'read_file']

            # Call a tool
            result = await session.call_tool(
                "list_directory",
                arguments={"path": "./src"}
            )
            print(result.content[0].text)
            # → 📁 components
            # → 📄 index.ts
            # → 📄 utils.ts

            # Read a resource
            resource = await session.read_resource("file://./README.md")
            print(resource.contents[0].text[:100])

asyncio.run(main())
Under the Hood

JSON-RPC Message Flow

All MCP communication uses JSON-RPC 2.0. Here's what the wire messages look like for a tool call.

Client → Server (Request) 
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "read_file",
    "arguments": {
      "path": "./src/index.ts",
      "encoding": "utf8"
    }
  }
}
Server → Client (Response) 
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "export const main = () => {\n  // ..."
      }
    ],
    "isError": false
  }
}

Summary

Why MCP Matters

1
Universal Standard

Any MCP-compliant model works with any MCP-compliant server. Build once, integrate everywhere. No more per-model, per-tool custom code.

2
Discovery at Runtime

Models don't need to be pre-trained on your tools. They discover capabilities dynamically via tools/list — making the ecosystem truly extensible.

3
Security Boundaries

MCP servers run in separate processes. The host controls what servers are started and what capabilities are exposed — no direct model-to-tool access.

4
Open Ecosystem

MCP is open-source and already supported by Claude, Cursor, Cline, and dozens of community servers for GitHub, Postgres, Slack, Brave Search, and more.
