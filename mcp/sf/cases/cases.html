Automate Salesforce Case Notifications

Overview

This server provides tools for querying unresolved Salesforce cases and notifying customers via email, using the Python MCP server and Salesforce CLI.

MCP servers config for LLM


     "salesforceCases": {
      "command": "/Users/mchinnappan/.local/bin/uv",
      "args": [
        "--directory",
        "/Users/mchinnappan/MCP_Dev/weather",
        "run",
        "case_mcp_server.py"
      ]
    }

Sample Usage

Get Unresolved Cases and Notify: 
await get_unresolved_cases_and_notify(
    username="your-sf-username-or-alias",
    email_message="Thank you for your patience. We are working on your case."
)

Available Tools

get_unresolved_cases_and_notify

Get unresolved Salesforce cases and notify customers via email.

await get_unresolved_cases_and_notify(username="username", email_message=None)
  • username: Salesforce org username or alias (default: "username")
  • email_message: Optional custom message for the email body

Setup Notes

  • Requires sf (Salesforce CLI) installed and authenticated for your org(s).
  • Uses sf force:org:display -u <username> --json to get connection info.
  • Requires httpx Python package for async HTTP requests.
  • Emails are simulated by the send_email function (customize as needed for real email sending).
