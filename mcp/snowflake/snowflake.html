Overview

This server provides tools for querying and inserting data into Snowflake databases via a Python MCP server.

Sample Usage with LLM supporting MCP servers

- snowflake query SELECT * FROM MY_DATABASE.PUBLIC.DOCUMENTS LIMIT 2

- snowflake insert INSERT INTO MY_DATABASE.PUBLIC.DOCUMENTS (id, source_type, source, content  ) 
VALUES (100, 'url', 'https://www.postgresql.org/', 'PostgreSQL');

- snowflake query SELECT * FROM MY_DATABASE.PUBLIC.DOCUMENTS WHERE Id = 100

Sample Usage

Run a Query: 
snowflake_query(
    query="SELECT * FROM MY_DATABASE.PUBLIC.DOCUMENTS LIMIT 2;",
    profile="default"
)
Insert Data: 
snowflake_insert(
    insert_sql="INSERT INTO MY_DATABASE.PUBLIC.DOCUMENTS (id, name) VALUES (1, 'Sample');",
    profile="default"
)

Available Tools

snowflake_query

Run a SQL query on Snowflake and return results as a list of dicts.

snowflake_query(query, profile="default")
  • query: SQL query string
  • profile: Connection profile name from ~/.snowflake/connection.json (default: "default")

snowflake_insert

Run an INSERT statement on Snowflake.

snowflake_insert(insert_sql, profile="default")
  • insert_sql: Full SQL insert statement (e.g. INSERT INTO t (a, b) VALUES (1, 'foo'))
  • profile: Connection profile name from ~/.snowflake/connection.json (default: "default")

Setup Notes

  • Store your Snowflake connection profiles in ~/.snowflake/connection.json as:
    { "default": { "user": "YOUR_USER", "password": "YOUR_PASSWORD", "account": "YOUR_ACCOUNT", "warehouse": "YOUR_WAREHOUSE", "database": "YOUR_DATABASE", "schema": "YOUR_SCHEMA" } }
  • Each profile can specify user, password, account, warehouse, database, schema, and optionally role.
  • All tools require valid credentials and network access to the Snowflake server.
  • Install the snowflake-connector-python package: pip install snowflake-connector-python
