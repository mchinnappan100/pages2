02 — Context

The Enterprise Challenge

Modern enterprises face mounting pressure from repetitive workflows, fragmented systems, slow response cycles, and human bottlenecks. Traditional automation tools — including early copilot-style AI — addressed only surface-level problems through scripted, linear action chains.

Earlier AI like Salesforce's own Agentforce Assistant (formerly called Copilot) relied on Chain-of-Thought (CoT)CoT generates a linear sequence of steps. If the plan is wrong, it has no mechanism to self-correct — making it fragile at scale. reasoning, which proved too brittle for enterprise-scale use. A new approach was needed.

03 — Core Technology

Atlas Reasoning Engine ✓ Fact-checked

Atlas is the cognitive core of Agentforce — incubated at Salesforce AI Research and formally launched at Dreamforce 2024. Built on a proprietary system designed to simulate human-like thinking, it uses advanced Retrieval-Augmented Generation (RAG)RAG grounds AI responses in real enterprise data, dramatically reducing hallucinations. to analyze information and determine the next best actions across simple or complex tasks.

Unlike its predecessor (which used Chain-of-Thought reasoning), Atlas employs a ReAct (Reason–Act–Observe) loop — allowing it to dynamically evaluate its own reasoning at each step, course-correct mid-task, and ask for clarification when needed. This makes it fundamentally more robust than static, linear automation.

05 — Implementation

Code Examples

Agentforce tools are built using Salesforce Apex and standard APIs. The Atlas Reasoning Engine selects and invokes these tools dynamically based on the current plan step.

Atlas Execution Plan (Illustrative)


const atlasPlan = {
  goal: "Resolve high-priority customer support case",
  reasoning_engine: "Atlas ReAct Loop",

  steps: [
    {
      step: 1,
      action: "Retrieve customer case and history",
      tool: "Salesforce Data Cloud (RAG-based retrieval)"
    },
    {
      step: 2,
      action: "Evaluate case severity and appropriate resolution path",
      tool: "Atlas LLM + System 2 reasoning",
      observe: "Does the plan look correct? Loop if not."
    },
    {
      step: 3,
      action: "Assign to support engineer or autonomous resolution",
      tool: "Salesforce Flow",
      guardrail: "Escalate to human if confidence below threshold"
    },
    {
      step: 4,
      action: "Notify customer with resolution summary",
      tool: "Communication API (Email / Slack / SMS)"
    }
  ]
};

Apex Tool for Atlas


// Apex REST endpoint exposed as an Agentforce tool
// Atlas will invoke this during its execution plan

@RestResource(urlMapping='/atlas/getCases')
global with sharing class AtlasCaseTool {

  @HttpPost
  global static List<Case> execute(String priority) {
    return [
      SELECT Id, CaseNumber, Priority, Status, Subject
      FROM Case
      WHERE Priority = :priority
        AND Status != 'Closed'
      ORDER BY CreatedDate DESC
      LIMIT 10
    ];
  }

}

Agentforce API Call


async function askAgent(question) {
  const response = await fetch("/services/apexrest/agentforce", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${accessToken}`
    },
    body: JSON.stringify({ query: question })
  });

  const result = await response.json();
  console.log("Agent response:", result);
  return result;
}
