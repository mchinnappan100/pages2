📄 The Code
load_gemma.py
from transformers import AutoProcessor, AutoModelForMultimodalLM

MODEL_ID = "google/gemma-4-31B-it"

# Load model
processor = AutoProcessor.from_pretrained(MODEL_ID)
model = AutoModelForMultimodalLM.from_pretrained(
    MODEL_ID,
    dtype="auto",       # → bfloat16 on modern GPUs
    device_map="auto"  # → spread across all available GPUs / CPU
)

🗂️ What Each File Does
processor_config.json
1.69 kB
Config / Metadata
Tells AutoProcessor how to pre-process multimodal inputs — image resize dimensions, pixel normalization values, and how image tokens are coupled with text tokens.
Used by: AutoProcessor
Contains: image_size, mean/std, patch_size
chat_template.jinja
18.7 kB
Config / Metadata
A Jinja2 template that formats conversation turns (user / assistant roles + system prompt) into the exact string format Gemma was fine-tuned on. Wrong template → garbage output.
Used by: tokenizer.apply_chat_template()
Largest config file because it handles many edge cases
config.json
4.62 kB
Config / Metadata
The model's architecture blueprint. Defines every structural hyperparameter so the framework can instantiate the right class with the right dimensions before loading any weights.
Contains: num_layers, hidden_size, num_heads,
vocab_size, attention_type, max_position
tokenizer_config.json
3.08 kB
Config / Metadata
Tokenizer settings — which tokenizer class to instantiate, BOS/EOS token IDs, padding side (left/right), and any special tokens added during instruction-tuning.
Contains: bos_token, eos_token, pad_token,
padding_side, tokenizer_class
tokenizer.json
32.2 MB
Tokenizer
The full SentencePiece / BPE vocabulary — all ~256k tokens, their merge rules, and token→id mappings. Large because Gemma uses a bigger vocabulary than GPT-2-era models to better handle multilingual text and code.
Contains: vocab (~256k entries), merges,
special_tokens, added_tokens
Note: downloaded once then cached
model.safetensors.index.json
120 kB
Weights
A shard manifest — maps every tensor name (e.g. model.layers.0.self_attn.q_proj.weight) to the shard file it lives in. Required because 62.5 GB can't fit in a single file.
Contains: {"weight_map": {"layer.0.w": "shard-00001-of-00015.safetensors", ...}}
*.safetensors shards
62.5 GB total
Weights
The actual 31 billion learned parameters split across multiple shard files. Each shard is a memory-safe binary format (no arbitrary code execution, unlike pickle). Downloaded in parallel at ~52 MB/s.
dtype: bfloat16 (2 bytes/param)
31B × 2 bytes = ~62 GB on disk
device_map="auto" spreads across GPUs+CPU
🔢 Why 62.5 GB?
bfloat16 (dtype="auto")
31,000,000,000 params × 2 bytes = ~62 GB
Modern GPUs (Ampere+) support bfloat16 natively. Same exponent range as float32 — far fewer NaN/inf than float16.
float32 (would be)
31B params × 4 bytes = ~124 GB
bfloat16 halves memory with negligible accuracy loss for inference. float32 would need ~4 A100-80GB cards just for weights.
int8 quantized
31B params × 1 byte = ~31 GB
Pass load_in_8bit=True (bitsandbytes) to halve again at a small accuracy cost.
int4 quantized (GPTQ/AWQ)
31B params × 0.5 bytes = ~15.5 GB
Fits in a single consumer GPU (24 GB). Use load_in_4bit=True or a pre-quantized GPTQ model.
📋 Load Order & Why
1
Architecture config
config.json
Must come first — the framework reads this to know which Python class to instantiate and what tensor shapes to allocate memory for.
2
Tokenizer files
tokenizer_config.json  →  tokenizer.json  →  chat_template.jinja
Loads the full vocabulary and chat formatting rules. Independent of model weights — the processor and tokenizer are ready to encode inputs while weights are still downloading.
3
Processor config
processor_config.json
Configures the image pre-processing pipeline so the AutoProcessor knows how to turn raw images into pixel tensors the model expects.
4
Shard manifest
model.safetensors.index.json
Tells the loader which shard file holds which tensor. With device_map="auto" this also drives which shard goes to which GPU, so the loader can stream shards directly to the right device.
5
Weight shards (parallel download)
model-00001-of-N.safetensors  …  model-NNNN-of-N.safetensors
The heaviest step — 62.5 GB streamed in parallel. HuggingFace Hub downloads multiple shards concurrently (shown as "Fetching 2 files"). Each shard is written to disk cache then memory-mapped into the model.
