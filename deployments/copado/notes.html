QA → UAT Promotion
Without DevOps

How developers and dev leads can independently promote user stories from QA to UAT using Copado — no DevOps ticket required.

Audience: Devs & Dev Leads Tool: Copado Env: QA → UAT Updated: 2026
01 — Context

Why This Matters

Previously, every QA → UAT promotion required a DevOps engineer to intervene. This created bottlenecks, slowed sprint velocity, and added unnecessary hand-offs. This guide gives dev teams the autonomy to promote independently.

🚧
Before: The Old Way
Dev finishes QA → DevOps team waits for Dev to confirm to Promote story to UAT → DevOps promotes → UAT testing begins. Delays measured in hours or days.
⚡
Now: Dev-Led Promotion
Dev or Dev Lead promotes directly in Copado. DevOps is only needed for environment-level infrastructure changes.
ℹ️ Scope: This process applies only to story-level promotions from QA to UAT. Production deployments still require DevOps sign-off.
02 — Key Concepts

Promotion Records & Branches

📦 What is a Promotion Record?

In Copado, a promotion record is the container that holds all deployment state — including resolved merge conflicts, selected user stories, and deployment results. Think of it as a "deployment session."

⚠️ Why You Should NOT Create a New Promotion Record

Creating a new promotion record resets all deployment state — including any merge conflicts you already resolved. This means repeated, duplicated work. Every time you create a new record, you start from scratch.

✅ The Right Approach: Reuse & Reattach

Instead of creating a new record, reuse the existing promotion record and attach a new promotion branch to it. This preserves all resolved conflicts and deployment history, keeping your team's workflow continuous and efficient.

03 — Step-by-Step

How to Promote QA → UAT

  1. Verify the story is QA-complete
    Confirm the user story status in Copado is set to QA Complete and all test cases have passed. The story must not have open defects tagged as blockers.
  2. Navigate to the Promotion Record
    In Copado, go to Promotions and locate the existing promotion record for the current sprint/release. Do not create a new one — reuse the one already associated with your pipeline.
  3. Attach a new Promotion Branch
    On the promotion record, select Add Promotion Branch and attach the feature branch for your story. This keeps all previously resolved merge conflict state intact.
  4. Add the User Story to the promotion
    In the User Stories related list on the promotion record, click Add Stories and select the story (or stories) ready for UAT. Confirm the metadata selections are correct.
  5. Run Merge & resolve any conflicts
    Click Promote → Merge. If conflicts arise, resolve them inline in Copado's conflict resolver. Since you're reusing the record, previously resolved conflicts won't reappear.
  6. Deploy to UAT
    Once the merge is clean, click Deploy. Copado will push the changes to the UAT environment. Monitor the deployment log directly in the promotion record.
  7. Update story status & notify QA
    Set the user story status to UAT in Copado (or your linked ADO/Jira board). Ping the QA team in Slack with the UAT org URL so they can begin acceptance testing.
✅ No DevOps ticket needed. As long as the pipeline and permission sets are configured (see table below), Dev Leads have full authority to execute this flow end-to-end.

05 — Best Practices

Tips for Dev & Dev Leads

🔁
Always reuse the promotion record
Never create a new promotion record mid-sprint. Reuse and reattach branches to preserve conflict resolution state.
🔍
Check metadata before deploying
Review the metadata components included in the promotion. Avoid accidentally pulling in unrelated changes from other stories.
📋
Keep story status in sync
Update the user story status in both Copado and your project tracker (Jira/ADO) immediately after promotion so QA can pick it up.
🚨
Deployment errors? Don't retry blindly
If a deployment fails, read the error log before retrying. Most failures are metadata conflicts — resolve them in the conflict resolver, not by recreating the record.
