Salesforce Deployment using GitHub Actions

Automating CI/CD for Salesforce using modern source-driven development

Big Picture

GitHub Actions allows you to automatically validate and deploy Salesforce metadata when code changes occur in GitHub. GitHub becomes the single source of truth for your Salesforce orgs. 

Developer → GitHub PR → GitHub Actions → Salesforce Org

Prerequisites

  • Source-driven Salesforce project (force-app/main/default)
  • GitHub repository with branching strategy
  • Salesforce CLI (sf)
  • Dedicated Salesforce integration user

Authentication (JWT Flow)

GitHub Actions cannot use browser-based login. The recommended approach is JWT-based authentication using a Connected App.

Salesforce Setup

  • Create Connected App
  • Enable OAuth & JWT Bearer Flow
  • Generate private key

GitHub Secrets

  • SF_CLIENT_ID
  • SF_USERNAME
  • SF_JWT_KEY
  • SF_INSTANCE_URL

GitHub Actions Workflow

A GitHub Actions workflow defines when and how Salesforce deployments run. 

.github/workflows/salesforce-deploy.yml

Common triggers include:

  • Pull request → validation only
  • Merge to main → deploy to production

Recommended Branch Strategy

Branch Action
feature/* Development only
develop Validation
release/* UAT Deployment
main Production Deployment

Best Practices

  • Always validate before deploying
  • Use a dedicated CI integration user
  • Never store secrets in the repo
  • Deploy only source-backed metadata
  • Keep pull requests small and focused
