OwnBackup Permission Set Setup

1. Permission Set: OwnBackupResourcePS

Create a permission set with the following permissions:

  • View and Edit Converted Leads
  • API Only User
  • Access Conversation Entries
  • Customize Application
  • Edit Read Only Fields
  • Manage Experiences
  • Manage Encryption Keys
  • Manage Flow
  • Manage Orchestration Runs and Work Items
  • Manage All Private Reports and Dashboards
  • Manage Users
  • Modify All Data
  • Password Never Expires
  • Query All Files
  • View All Custom Settings
  • View All Lookup Record Names
  • View Encrypted Data

package.xml

<?xml version="1.0" encoding="UTF-8"?>
<Package xmlns="http://soap.sforce.com/2006/04/metadata">

  <types>
      <members>OwnBackupResourcePS</members>
      <name>PermissionSet</name>
  </types>

  <version>64.0</version>
</Package>
Get this package.xml by: 
  curl -O https://raw.githubusercontent.com/mchinnappan100/pages2/refs/heads/main/backup/own/package.xml

2. Create SFDX Project

Create a new Salesforce DX project:

sf project generate -n OwnBackupPSCreation

Navigate into the project:

cd OwnBackupPSCreation

Retrieve the permission set:

sf project retrieve start -x package.xml -o <username>

3. Output Location

The retrieved permission set XML will be available at:

force-app/main/default/permissionsets/OwnBackupResourcePS.permissionset-meta.xml
Commit this in the version control system and use it for deployment to other orgs as needed and assign this permission set to the user performing backup and restore operations with OwnBackup.
