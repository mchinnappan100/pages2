ACME Inc

External Contractor Secure Access Solution Architecture

1. Executive Summary

ACME Inc requires a secure and controlled way to allow external contractors to access internal Windows applications, JIRA, and enterprise systems that require ACME domain authentication.

This document evaluates two primary solutions:

  • Citrix Virtual Desktop Infrastructure (VDI)
  • Microsoft Azure Virtual Desktop (AVD)

Recommended Solution: Azure Virtual Desktop with Zero Trust Security

2. Requirements

Security Requirements

  • MFA authentication
  • No direct network exposure
  • Full audit logging
  • Data exfiltration prevention
  • Session timeout / idle disconnect policy
  • Named accounts only — no shared credentials
  • Device compliance posture check
  • Data residency compliance (e.g., GDPR)

Functional Requirements

  • Access Windows apps
  • Access JIRA (domain restricted)
  • Support Windows and macOS
  • Easy onboarding/offboarding
  • BYOD and ACME-issued device support
  • Time-boxed access — auto-expires at contract end

Advantages

  • Industry-leading security
  • Native session recording capabilities
  • Granular control (clipboard, USB, printing)
  • Supports hybrid infrastructure
  • Best for high compliance industries

Disadvantages

  • High licensing cost (per-user / per-device model)
  • Complex setup and maintenance
  • Requires dedicated Citrix expertise
  • Infrastructure heavy

Implementation Steps

  1. Setup Active Directory and contractor accounts
  2. Install Citrix Delivery Controller and Gateway
  3. Deploy and configure Citrix StoreFront
  4. Create Windows desktop pools
  5. Install internal apps and JIRA
  6. Configure security policies (DLP, clipboard restrictions, USB blocking)
  7. Enable monitoring and session recording

Advantages

  • Cloud-native solution
  • Lower infrastructure cost (consumption-based)
  • Easy deployment and scaling
  • Built-in Azure security integration
  • Minimal maintenance
  • Strong BYOD support via Microsoft Intune

Disadvantages

  • Requires Azure expertise
  • Dependent on Azure availability (99.9% SLA; mitigated by multi-region deployment)
  • Less granular control vs Citrix
  • No native session recording — requires third-party add-ons

Implementation Steps

  1. Create Microsoft Entra ID contractor accounts
  2. Create Virtual Desktop host pool
  3. Create Windows virtual machines with appropriate VM SKU
  4. Install internal applications and JIRA client
  5. Configure MFA and Conditional Access policies
  6. Enable Microsoft Intune for device compliance enforcement
  7. Enable Azure Monitor and Microsoft Sentinel for threat detection
  8. Configure Privileged Identity Management (PIM) for any elevated contractor roles

Final Recommendation

Recommended Solution: Azure Virtual Desktop

  • Best balance of security and cost
  • Easy onboarding/offboarding
  • Supports Windows and macOS
  • Cloud-native and scalable
  • Minimal maintenance

Note: If ACME operates in a regulated industry (healthcare, finance, defence) where built-in session recording, DLP, and granular peripheral control are mandated, Citrix VDI may be the better choice despite its higher cost. This recommendation should be revisited once compliance scope is formally defined.

Security Best Practices

  • Enable MFA on all contractor accounts
  • Use Conditional Access policies to enforce device and location constraints
  • Disable clipboard, USB, and printing in virtual desktop sessions
  • Enable monitoring, logging, and alerting via Azure Monitor / Sentinel
  • Use virtual desktops only — no direct network access to internal systems
  • Disable direct network access; enforce all traffic through AVD Gateway
  • Just-In-Time (JIT) Access — provision VM access only for active sessions; auto-deprovision after session or contract end
  • Named accounts only — no shared contractor credentials; every account must be uniquely identifiable for auditability
  • Regular access reviews — conduct quarterly reviews of all active contractor accounts; immediately revoke access upon contract completion
  • Network segmentation — place VMs in a dedicated subnet with NSG rules to prevent lateral movement to production systems
  • Privileged Identity Management (PIM) — for any elevated contractor roles, enforce time-limited, approved elevation with a full audit trail
  • Session timeout / idle disconnect — automatically lock and disconnect sessions after 15 minutes of inactivity
  • Zero Trust principles — verify identity, device health, and access scope on every connection attempt; never implicitly trust a contractor session
