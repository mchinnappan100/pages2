ACME Inc

External Contractor Secure Access Solution Architecture

1. Executive Summary

ACME Inc requires a secure and controlled way to allow external contractors to access internal Windows applications, JIRA, and enterprise systems that require ACME domain authentication.

This document evaluates two primary solutions:

  • Citrix Virtual Desktop Infrastructure (VDI)
  • Microsoft Azure Virtual Desktop (AVD)

Recommended Solution: Azure Virtual Desktop with Zero Trust Security

2. Requirements

Security Requirements

  • MFA authentication
  • No direct network exposure
  • Full audit logging
  • Data exfiltration prevention

Functional Requirements

  • Access Windows apps
  • Access JIRA (domain restricted)
  • Support Windows and macOS
  • Easy onboarding/offboarding

Option 1: Citrix Virtual Desktop Infrastructure (VDI)

flowchart LR User[Contractor Device] Gateway[Citrix Gateway] Auth[MFA + Active Directory] Controller[Citrix Controller] VDI[Windows Virtual Desktop Pool] Apps[Internal Apps + JIRA] Monitor[Monitoring + Session Recording] User --> Gateway Gateway --> Auth Auth --> Controller Controller --> VDI VDI --> Apps Controller --> Monitor

Advantages

  • Industry-leading security
  • Session recording capabilities
  • Granular control (clipboard, USB, printing)
  • Supports hybrid infrastructure
  • Best for high compliance industries

Disadvantages

  • High licensing cost
  • Complex setup and maintenance
  • Requires Citrix expertise
  • Infrastructure heavy

Implementation Steps

  1. Setup Active Directory and contractor accounts
  2. Install Citrix Delivery Controller and Gateway
  3. Create Windows desktop pools
  4. Install internal apps and JIRA
  5. Configure security policies
  6. Enable monitoring and logging

Option 2: Microsoft Azure Virtual Desktop (AVD)

flowchart LR User[Contractor Device] Internet[Internet] AzureGateway[Azure Virtual Desktop Gateway] AzureAD[Azure AD + MFA] VM[Azure Windows VM Pool] Apps[Internal Apps + JIRA] Monitor[Azure Monitoring] User --> Internet Internet --> AzureGateway AzureGateway --> AzureAD AzureAD --> VM VM --> Apps VM --> Monitor

Advantages

  • Cloud-native solution
  • Lower infrastructure cost
  • Easy deployment and scaling
  • Built-in Azure security integration
  • Minimal maintenance

Disadvantages

  • Requires Azure expertise
  • Dependent on Azure availability
  • Less granular control vs Citrix

Implementation Steps

  1. Create Azure AD contractor accounts
  2. Create Virtual Desktop host pool
  3. Create Windows virtual machines
  4. Install internal applications
  5. Configure MFA and Conditional Access
  6. Enable Azure monitoring

Solution Comparison

Feature Citrix VDI Azure Virtual Desktop
Cost High Lower
Security Excellent Very Good
Maintenance High Low
Scalability Good Excellent
Deployment Complex Easy

Final Recommendation

Recommended Solution: Azure Virtual Desktop

  • Best balance of security and cost
  • Easy onboarding/offboarding
  • Supports Windows and macOS
  • Cloud-native and scalable
  • Minimal maintenance

Security Best Practices

  • Enable MFA
  • Use conditional access policies
  • Disable clipboard and USB
  • Enable monitoring and logging
  • Use virtual desktops only
  • Disable direct network access
