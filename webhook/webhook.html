WebHooks Integration Guide

A practical guide to using WebHooks to integrate GitHub, GitLab, JIRA, Copado, Stack platforms, and other enterprise systems.

What Are WebHooks?

A WebHook is an HTTP callback mechanism that allows one system to notify another system in real time when an event occurs. Instead of polling, the source system pushes event data to a configured URL.

Key Characteristics

  • Event-driven (push-based)
  • Typically uses HTTP POST with JSON payloads
  • Low latency and near real-time
  • Requires secure endpoint validation
POST /webhooks/events Content-Type: application/json { "event": "push", "timestamp": "2026-02-03T12:00:00Z", "payload": { ... } }

GitHub WebHooks

GitHub WebHooks enable integrations for repository events such as pushes, pull requests, releases, and workflow executions.

Common Events

  • push
  • pull_request
  • workflow_run
  • release
X-Hub-Signature-256: sha256=... { "action": "opened", "pull_request": { "id": 123, "title": "Add new feature" } }

GitLab WebHooks

GitLab WebHooks provide event notifications for source control, CI/CD pipelines, and merge request activity.

Common Events

  • Push events
  • Merge request events
  • Pipeline events
  • Job events
X-Gitlab-Event: Merge Request Hook { "object_kind": "merge_request", "object_attributes": { "state": "merged" } }

JIRA WebHooks

JIRA WebHooks notify external systems about issue lifecycle changes such as creation, updates, transitions, and comments.

Common Events

  • Issue created
  • Issue updated
  • Status transitioned
  • Comment added
{ "webhookEvent": "jira:issue_updated", "issue": { "key": "PROJ-101", "fields": { "status": { "name": "Done" } } } }

Copado WebHooks

Copado WebHooks are commonly used in Salesforce DevOps pipelines to notify external systems about deployment, validation, and promotion events.

Typical Use Cases

  • Deployment status notifications
  • Pipeline stage transitions
  • Release tracking integrations
{ "event": "deployment_completed", "environment": "UAT", "status": "SUCCESS" }

Stack & Other Systems

Many enterprise platforms (observability stacks, messaging systems, internal tools) expose WebHooks using similar patterns.

Recommended Design Pattern

  • Central WebHook gateway / listener
  • Event normalization layer
  • Async processing (queues, workers)
  • Retry & idempotency handling
