Git Merge Conflict & Resolution Guide

Overview

A merge conflict happens when Git cannot automatically merge changes from two branches because the same lines were modified.

This guide demonstrates:

  • Create a repository
  • Create main and develop branches
  • Modify the same file in both branches
  • Trigger a merge conflict
  • Resolve the conflict

1. Create a New Git Repository

mkdir git-conflict-demo
cd git-conflict-demo

git init
Create a file. 
echo "Hello from Main branch" > app.txt
Commit it. 
git add .
git commit -m "Initial commit"

2. Create Develop Branch 

git branch develop
git checkout develop
or 
git switch -c develop
Current branches: 
git branch
Output: 
* develop
  main

3. Modify File in Develop Branch

Edit **app.txt** 
Hello from Develop branch
Commit: 
git add app.txt
git commit -m "Change message in develop branch"

4. Switch Back to Main 

git checkout main
Modify the same file differently. Edit **app.txt** 
Hello from Main branch - updated
Commit: 
git add app.txt
git commit -m "Update message in main"

5. Attempt to Merge Develop into Main 

git merge develop
Git will respond: 
Auto-merging app.txt
CONFLICT (content): Merge conflict in app.txt
Automatic merge failed; fix conflicts and then commit the result.

6. Examine the Conflict

Open **app.txt** 
<<<<<<< HEAD
Hello from Main branch - updated
=======
Hello from Develop branch
>>>>>>> develop
Explanation: | Marker | Meaning | |------|------| | `<<<<<<< HEAD` | Current branch (main) | | `=======` | Divider | | `>>>>>>> develop` | Incoming branch changes |

7. Resolve the Conflict

Edit the file and choose the final content. Example resolution: 
Hello from Main and Develop branches
Then stage the file. 
git add app.txt
Complete the merge. 
git commit -m "Resolve merge conflict between main and develop"

8. Verify the Merge

Check the history. 
git log --graph --oneline --all
Example output: 
*   Merge branch 'develop'
|\
| * Change message in develop branch
* | Update message in main
|/
* Initial commit

Best Practices to Avoid Merge Conflicts

  • Pull frequently from main branch
  • Create small feature branches
  • Communicate with teammates
  • Use code reviews
  • Use merge tools (VSCode, GitKraken)
