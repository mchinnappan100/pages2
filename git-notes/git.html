Initial Configuration

Set up your identity before making your first commit:

# Set your name
git config --global user.name "Your Name"

# Set your email
git config --global user.email "your.email@example.com"

# Set default branch name
git config --global init.defaultBranch main

# Enable colored output
git config --global color.ui auto

# View all settings
git config --list

üí° Tip: Use --global for user-wide settings, or omit it for repository-specific configuration.

Git Basics

Creating a Repository

Initialize a New Repository

# Create a new directory
mkdir my-project
cd my-project

# Initialize Git repository
git init

# Output:
# Initialized empty Git repository in /path/to/my-project/.git/

Clone an Existing Repository

# Clone a repository
git clone https://github.com/user/repo.git

# Clone with a different name
git clone https://github.com/user/repo.git my-repo

# Clone a specific branch
git clone -b develop https://github.com/user/repo.git

The Three States of Git

Working Directory

Modified files that haven't been staged

Staging Area

Files marked for the next commit

Repository

Committed files in Git history

# Check status of files
git status

# Example output:
# On branch main
# Changes not staged for commit:
#   modified:   index.html          (Working Directory)
#
# Changes to be committed:
#   new file:   style.css           (Staging Area)

Basic Workflow

# 1. Create or modify files
echo "Hello Git" > hello.txt

# 2. Add files to staging area
git add hello.txt          # Add specific file
git add .                  # Add all files in current directory
git add *.js               # Add all JavaScript files

# 3. Commit changes
git commit -m "Add hello.txt with greeting"

# Shortcut: Add and commit in one step (tracked files only)
git commit -am "Update existing files"

# 4. View commit history
git log
git log --oneline          # Compact view
git log --graph --all      # Visual graph of branches

‚úì Best Practice:

  • Make small, focused commits
  • Write clear, descriptive commit messages
  • Commit often to track your progress

Undoing Changes

# Unstage a file (keep changes in working directory)
git reset HEAD file.txt

# Discard changes in working directory
git checkout -- file.txt

# Undo last commit (keep changes staged)
git reset --soft HEAD~1

# Undo last commit (keep changes in working directory)
git reset HEAD~1

# Undo last commit (discard all changes) ‚ö†Ô∏è DANGEROUS
git reset --hard HEAD~1

# Amend the last commit (fix message or add forgotten files)
git add forgotten-file.txt
git commit --amend -m "Updated commit message"

Viewing Changes

# Show unstaged changes
git diff

# Show staged changes
git diff --staged

# Show changes for a specific file
git diff file.txt

# Show changes between commits
git diff abc123 def456

# Show changes between branches
git diff main feature-branch

# Show who changed each line of a file
git blame file.txt

Branching & Merging

Branches allow you to diverge from the main line of development and work independently without affecting the main codebase.

Working with Branches

# List all branches
git branch                 # Local branches
git branch -r              # Remote branches
git branch -a              # All branches

# Create a new branch
git branch feature-login

# Switch to a branch
git checkout feature-login

# Create and switch in one command
git checkout -b feature-signup

# Modern alternative (Git 2.23+)
git switch feature-login
git switch -c feature-signup

# Rename a branch
git branch -m old-name new-name

# Delete a branch
git branch -d feature-login    # Safe delete (only if merged)
git branch -D feature-login    # Force delete

Merging Branches

Fast-Forward Merge

# Switch to main branch
git checkout main

# Merge feature branch
git merge feature-login

# Output:
# Updating abc123..def456
# Fast-forward
#  login.js | 45 ++++++++++++
#  1 file changed, 45 insertions(+)

No merge commit created; main simply moves forward

Three-Way Merge

# Merge with a merge commit
git merge --no-ff feature-signup

# Output:
# Merge made by the 'recursive' strategy.
#  signup.js | 32 +++++++++++++++++
#  1 file changed, 32 insertions(+)

Creates a merge commit to preserve branch history

Handling Merge Conflicts

When the same part of a file is modified in both branches:

# Attempt to merge
git merge feature-branch

# Output:
# Auto-merging index.html
# CONFLICT (content): Merge conflict in index.html
# Automatic merge failed; fix conflicts and then commit the result.

Conflict markers in file:

<<<<<<< HEAD

Welcome to My Site

=======

Welcome to Our Platform

>>>>>>> feature-branch
# 1. Open the conflicted file and resolve manually
# Remove conflict markers and choose the correct code

# 2. Stage the resolved file
git add index.html

# 3. Complete the merge
git commit -m "Merge feature-branch and resolve conflicts"

# Or abort the merge if needed
git merge --abort

Rebasing

Rebase rewrites history by moving your commits on top of another branch:

# Switch to your feature branch
git checkout feature-branch

# Rebase onto main
git rebase main

# If conflicts occur, resolve them then:
git add resolved-file.txt
git rebase --continue

# Or skip a commit
git rebase --skip

# Or abort the rebase
git rebase --abort

# Interactive rebase (edit commit history)
git rebase -i HEAD~3      # Edit last 3 commits

‚ö†Ô∏è Warning: Never rebase commits that have been pushed to a shared repository, as it rewrites history!

Remote Repositories

Managing Remotes

# List remote repositories
git remote -v

# Add a remote repository
git remote add origin https://github.com/user/repo.git

# Add multiple remotes
git remote add upstream https://github.com/original/repo.git

# Change remote URL
git remote set-url origin https://github.com/user/new-repo.git

# Remove a remote
git remote remove origin

# Rename a remote
git remote rename origin upstream

# Show information about a remote
git remote show origin

Syncing with Remote

Pushing Changes

# Push to remote repository
git push origin main

# Push and set upstream
git push -u origin main

# Push all branches
git push --all origin

# Push tags
git push --tags

# Force push (‚ö†Ô∏è dangerous)
git push --force origin main

# Safer force push
git push --force-with-lease origin main

Pulling Changes

# Fetch and merge changes
git pull origin main

# Fetch without merging
git fetch origin

# Fetch all remotes
git fetch --all

# Pull with rebase
git pull --rebase origin main

# View fetched changes
git log origin/main

# Merge fetched changes
git merge origin/main

Tracking Branches

# Create a tracking branch
git checkout -b feature origin/feature

# Modern way (Git 2.23+)
git switch -c feature origin/feature

# Set upstream for existing branch
git branch --set-upstream-to=origin/main main

# View tracking relationships
git branch -vv

# Pull from tracked branch
git pull    # No need to specify remote and branch

# Push to tracked branch
git push    # No need to specify remote and branch

Pull Request Workflow

1

Fork and Clone

git clone https://github.com/yourname/repo.git

Create Feature Branch

Make Changes and Commit

4

Push to Your Fork

5

Keep Your Fork Updated

git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Advanced Topics

Stashing Changes

Temporarily save changes without committing:

# Stash current changes
git stash

# Stash with a message
git stash save "Work in progress on feature X"

# List all stashes
git stash list
# Output:
# stash@{0}: WIP on main: abc123 Last commit message
# stash@{1}: On feature: def456 Previous stash

# Apply most recent stash (keep it in stash list)
git stash apply

# Apply specific stash
git stash apply stash@{1}

# Apply and remove from stash list
git stash pop

# Show stash contents
git stash show -p stash@{0}

# Drop a stash
git stash drop stash@{0}

# Clear all stashes
git stash clear

# Stash including untracked files
git stash -u

Tagging

Mark specific points in history as important:

# Create a lightweight tag
git tag v1.0.0

# Create an annotated tag (recommended)
git tag -a v1.0.0 -m "Release version 1.0.0"

# Tag a specific commit
git tag -a v0.9.0 abc123 -m "Version 0.9.0"

# List all tags
git tag
git tag -l "v1.*"    # Pattern matching

# Show tag information
git show v1.0.0

# Push a tag to remote
git push origin v1.0.0

# Push all tags
git push --tags

# Delete a local tag
git tag -d v1.0.0

# Delete a remote tag
git push origin --delete v1.0.0

# Checkout a tag
git checkout v1.0.0

Cherry-Picking

Apply specific commits from one branch to another:

# Cherry-pick a single commit
git cherry-pick abc123

# Cherry-pick multiple commits
git cherry-pick abc123 def456

# Cherry-pick a range of commits
git cherry-pick abc123..def456

# Cherry-pick without committing
git cherry-pick -n abc123

# Continue after resolving conflicts
git cherry-pick --continue

# Abort cherry-pick
git cherry-pick --abort

Submodules

Include other Git repositories within your repository:

# Add a submodule
git submodule add https://github.com/user/library.git libs/library

# Initialize submodules after cloning
git submodule init
git submodule update

# Clone with submodules
git clone --recursive https://github.com/user/repo.git

# Update all submodules
git submodule update --remote

# Update specific submodule
git submodule update --remote libs/library

# Remove a submodule
git submodule deinit libs/library
git rm libs/library
rm -rf .git/modules/libs/library

Reflog - Recovery Tool

Recover lost commits or branches:

# View reflog
git reflog

# Output example:
# abc123 HEAD@{0}: commit: Add new feature
# def456 HEAD@{1}: checkout: moving from main to feature
# ghi789 HEAD@{2}: commit: Fix bug

# Recover to a previous state
git reset --hard HEAD@{2}

# Create a branch from a reflog entry
git branch recover-branch HEAD@{3}

# View reflog for specific branch
git reflog show feature-branch

# Reflog expires after 90 days by default

üíæ Lifesaver: Reflog is your safety net. Even if you accidentally delete a branch or reset too hard, reflog can help you recover!

Git Hooks

Automate tasks with Git hooks:

# Hooks are located in .git/hooks/
cd .git/hooks

# Example: Pre-commit hook
# File: .git/hooks/pre-commit
#!/bin/bash
# Run tests before commit
npm test
if [ $? -ne 0 ]; then
    echo "Tests failed. Commit aborted."
    exit 1
fi

# Make hook executable
chmod +x .git/hooks/pre-commit

Common Hooks:

  • pre-commit - Run before commit is created
  • prepare-commit-msg - Modify commit message
  • commit-msg - Validate commit message
  • post-commit - Run after commit
  • pre-push - Run before push

Best Practices

‚úì Good Practices

  • ‚Ä¢ Commit early and often with descriptive messages
  • ‚Ä¢ Use branches for features and bug fixes
  • ‚Ä¢ Pull before you push to avoid conflicts
  • ‚Ä¢ Review changes before committing
  • ‚Ä¢ Use .gitignore for generated files
  • ‚Ä¢ Write meaningful commit messages (present tense)

‚úó Avoid These

  • ‚Ä¢ Don't commit sensitive data (passwords, keys)
  • ‚Ä¢ Don't rebase public/shared branches
  • ‚Ä¢ Avoid large binary files in repositories
  • ‚Ä¢ Don't commit directly to main/master
  • ‚Ä¢ Avoid force pushing to shared branches
  • ‚Ä¢ Don't use vague commit messages like "fix" or "update"

Common Workflows

Feature Branch Workflow

# 1. Update main branch
git checkout main
git pull origin main

# 2. Create feature branch
git checkout -b feature/new-feature

# 3. Make changes and commit
git add .
git commit -m "Implement new feature"

# 4. Push to remote
git push -u origin feature/new-feature

# 5. After code review, merge to main
git checkout main
git merge feature/new-feature
git push origin main

# 6. Delete feature branch
git branch -d feature/new-feature
git push origin --delete feature/new-feature

Hotfix Workflow

# 1. Create hotfix branch from main
git checkout main
git checkout -b hotfix/critical-bug

# 2. Fix the bug
git add fixed-file.js
git commit -m "Fix critical bug in production"

# 3. Merge to main
git checkout main
git merge hotfix/critical-bug
git push origin main

# 4. Merge to develop (if using)
git checkout develop
git merge hotfix/critical-bug
git push origin develop

# 5. Clean up
git branch -d hotfix/critical-bug

Troubleshooting

Problem: Accidentally committed to wrong branch

# Move last commit to a new branch
git branch new-branch
git reset --hard HEAD~1
git checkout new-branch

Problem: Need to undo pushed commit

# Create a new commit that reverses the changes
git revert HEAD
git push origin main

Problem: Diverged branches

# Option 1: Merge
git pull origin main

# Option 2: Rebase (cleaner history)
git pull --rebase origin main

Problem: Large file accidentally committed

# Remove file from Git history
git filter-branch --tree-filter 'rm -f large-file.zip' HEAD

# Or use BFG Repo-Cleaner (faster)
bfg --delete-files large-file.zip
