Recover Deleted Folders in Git

How to Retrieve a Deleted Folder from Git

Restoring a deleted folder depends on whether the deletion has already been committed or not.

1. Folder Deleted but Not Committed

Restore the folder from the latest commit: 

git restore path/to/folder

Older Git versions: 

git checkout -- path/to/folder

2. Folder Deleted and Already Committed

Find the last commit where the folder existed: 

git log -- path/to/folder

Restore the folder from a previous commit: 

git restore --source=<commit-hash> path/to/folder

Older syntax: 

git checkout <commit-hash> -- path/to/folder

Commit the restored files: 

git add path/to/folder
git commit -m "Restore deleted folder"

3. Deleted in the Last Commit

Undo the last commit while preserving changes: 

git reset --soft HEAD~1

Or restore only the deleted folder: 

git restore --source=HEAD~1 path/to/folder

4. Don't Know When It Was Deleted

Find commits where files were deleted: 

git log --diff-filter=D --summary

Search the history of a specific folder: 

git log -- path/to/folder

Restore from a previous commit: 

git checkout <commit> -- path/to/folder

5. Empty Directories

Git does not track empty directories. It only tracks files.

If the folder should remain empty: 

mkdir path/to/folder
touch path/to/folder/.gitkeep
git add path/to/folder/.gitkeep

Example

Suppose src/utils/ was deleted.

View History

git log -- src/utils

Restore Folder

git restore --source=HEAD~2 src/utils

# or

git checkout HEAD~2 -- src/utils

Verify

ls src/utils
git status

Commit

git add src/utils
git commit -m "Restore src/utils"
