Frequently Asked Questions

All data is stored locally in your browser's localStorage. This means:

  • No server: Your data never leaves your device
  • Complete privacy: No one else can access your information
  • Instant access: No network delays or authentication required
  • Browser-specific: Data is tied to the specific browser and device you're using

‚ö†Ô∏è Important: Clear browser data will delete your projects. Always maintain JSON backups!

Not simultaneously in real-time. However, you can share data through export/import:

  1. Person A exports project as JSON (‚Üì JSON button)
  2. Person A shares the JSON file (email, Slack, cloud storage, etc.)
  3. Person B imports the JSON file (‚Üë JSON button)
  4. Person B can now work with the same project data

Best practice: Establish a workflow where one person owns the "master" copy and periodically exports/shares updates.

Backing up is simple and essential:

  1. Click the ‚Üì JSON button in the project view
  2. Save the downloaded file to a secure location:
    • Cloud storage (Google Drive, Dropbox, OneDrive)
    • Local backup folder on your computer
    • Version control system (Git repository)
  3. Establish a backup schedule (daily, weekly, or after major changes)
  4. Keep multiple versions with dates in filenames (e.g., "project-2026-01-25.json")

üí° Pro Tip: Set a recurring calendar reminder to export your data weekly!

‚ö†Ô∏è Warning: Clearing data is permanent and cannot be undone!

To clear all data, you have two options:

Option 1: Browser Settings

  1. Open your browser settings
  2. Navigate to Privacy & Security
  3. Find "Clear browsing data" or "Site settings"
  4. Select this specific domain
  5. Clear site data

Option 2: Developer Tools

  1. Press F12 to open Developer Tools
  2. Go to "Application" or "Storage" tab
  3. Find "Local Storage" in the sidebar
  4. Right-click and select "Clear"

üî¥ Before clearing: Always export and save your JSON backup first!

The smart recurrence system automatically manages your repeating tasks:

  1. Mark as Completed: When you set a recurring activity to "Completed" status
  2. Automatic Creation: A new instance is immediately created with:
    • Same title, category, priority, and assignee
    • Updated due date based on recurrence pattern
    • Status reset to "Open"
    • Fresh notes field for new context
  3. Old Activity: The completed activity remains in your history with completion date

Recurrence Patterns:

  • üìÖ Daily: Next day
  • üìÖ Weekly: Same day next week (+7 days)

Troubleshooting

‚ùå My data disappeared!

Possible causes:

  • Cleared browser data/cache
  • Used different browser or incognito mode
  • Browser updated and reset storage

‚úÖ Solution: Restore from JSON backup

‚ùå Drag & drop not working

Try these fixes:

  • Refresh the page (F5 or Cmd+R)
  • Update to the latest browser version
  • Disable browser extensions temporarily
  • Use a different browser

‚ùå JSON import failed

Common issues:

  • File is corrupted or incomplete
  • Wrong file format (must be .json)
  • File was edited incorrectly

‚úÖ Solution: Use an unmodified JSON export

‚ùå Dark mode not saving

This is expected behavior:

  • Theme preference resets on page reload
  • Toggle theme each session as needed
  • This keeps the app lightweight
