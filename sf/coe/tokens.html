Salesforce OAuth Tokens

Understanding Access Tokens and Refresh Tokens

Introduction

When integrating with Salesforce APIs, understanding OAuth tokens is crucial for maintaining secure and efficient authentication. Salesforce uses two primary types of tokens: Access Tokens and Refresh Tokens. These tokens work together to provide seamless authentication while maintaining security.

Access Tokens

What is an Access Token?

An access token is a short-lived credential that grants your application permission to access Salesforce resources on behalf of a user. Think of it as a temporary key that opens the door to Salesforce data.

Key Characteristics

  • •
    Short-Lived: Typically expires after 15 minutes to 2 hours (configurable via session timeout settings)
  • •
    Bearer Token: Included in the Authorization header of API requests
  • •
    Scope Limited: Access is restricted to the permissions granted during OAuth authorization
  • •
    Single Instance: Tied to a specific Salesforce instance URL

Example Usage

Authorization: Bearer 00D5g000000XXXX!AR8AQJXYZ...

Refresh Tokens

What is a Refresh Token?

A refresh token is a long-lived credential used to obtain new access tokens without requiring the user to re-authenticate. It's your golden ticket to maintaining continuous access.

Key Characteristics

  • •
    Long-Lived: Can last indefinitely until revoked, though policies may vary
  • •
    Single Use (Rotating): When used to get a new access token, Salesforce can issue a new refresh token
  • •
    Secure Storage Required: Must be stored securely (never in client-side code)
  • •
    Revocable: Can be revoked by users or administrators at any time

Refresh Token Flow

POST /services/oauth2/token
Content-Type: application/x-www-form-urlencoded

grant_type=refresh_token
&refresh_token=your_refresh_token
&client_id=your_client_id
&client_secret=your_client_secret

Refresh Token Rotation

Salesforce supports refresh token rotation as a security best practice. When enabled, each time you use a refresh token to obtain a new access token, Salesforce can issue a new refresh token and invalidate the old one.

How It Works

1

Your application uses the current refresh token to request a new access token

2

Salesforce validates the refresh token and returns a new access token

3

Optionally, Salesforce also returns a new refresh token (rotation enabled)

4

Your application stores the new refresh token and discards the old one

Security Benefit

Rotation limits the window of opportunity for a compromised refresh token to be exploited, as each token can only be used once.

Access Token vs Refresh Token

Feature Access Token Refresh Token
Lifespan Short (15 min - 2 hours) Long (until revoked)
Purpose Access Salesforce APIs Obtain new access tokens
Usage Frequency Every API call When access token expires
Storage Location Memory (can be temporary) Secure, encrypted storage
Rotation N/A (expires naturally) Optional (recommended)
Scope API permissions Token generation only

Security Best Practices

For Access Tokens

  • Never store in client-side code or URLs
  • Use HTTPS for all API calls
  • Implement token expiry handling
  • Clear tokens from memory after use

For Refresh Tokens

  • Store encrypted in secure backend
  • Enable rotation when possible
  • Never expose in logs or error messages
  • Implement revocation mechanisms
