SCIM White Paper

Executive Summary

System for Cross-domain Identity Management (SCIM) is a standardized protocol for automating user identity lifecycle management across cloud systems. It simplifies provisioning, improves security, and ensures consistency across platforms like Salesforce.

What is SCIM?

SCIM is a REST/JSON-based open standard that enables automated provisioning, updating, and deprovisioning of users between Identity Providers (IdPs) and service providers like Salesforce.

  • Standardized schema
  • REST APIs
  • Supports CRUD operations

SCIM in Salesforce

Salesforce supports SCIM provisioning through Identity Providers like Azure AD and Okta. It allows automatic user creation, updates, and deactivation.

PEIN (Provisioning External Identifier)

PEIN is a unique identifier used in Salesforce to map external identities from Identity Providers to Salesforce users.

Best Practices

  • Use stable unique IDs (GUIDs)
  • Ensure proper PEIN mapping
  • Test provisioning lifecycle flows
  • Monitor sync logs

Conclusion

SCIM enables scalable identity management, while Salesforce PEIN ensures reliable identity mapping. Together, they provide secure and automated user lifecycle management.
