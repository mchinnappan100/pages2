Salesforce Custom Labels – Best Practices

A concise guide for developers to build scalable, maintainable internationalized Salesforce applications.

Why Use Custom Labels?

Custom Labels provide an essential mechanism for storing text values that can be referenced across Apex, Lightning Web Components, Aura, and Flows. They support localization, promote reuse, and help maintain consistency across applications.

1. Use Clear & Consistent Naming Conventions

  • Prefix labels by domain or feature (e.g., Acct_Error_InvalidStatus).
  • Avoid generic names like Error1 or Message100.
  • Use CamelCase or snake_case consistently.
  • Keep names descriptive but not excessively long.

2. Organize Labels by Category

Group labels logically so your team can find them easily:

  • Error Messages
  • Button Labels
  • Validation Text
  • Help/Tooltip Text
  • API/UI Messaging

Consider creating a shared document listing your categories and conventions.

3. Write Good Label Content

  • Keep messages clear, concise, and user-friendly.
  • Avoid embedding dynamic values directly—support placeholders instead (e.g., {0}).
  • Use complete sentences for user-facing text.
  • Avoid technical jargon unless needed.

4. Plan for Internationalization (i18n)

  • Ensure text is culturally neutral by default.
  • Use complete phrases so translators aren't forced to interpret fragments.
  • Avoid concatenating multiple labels to form a sentence.
  • Let translators handle punctuation and grammar.

6. Maintainability

  • Regularly audit for unused or duplicated labels.
  • Document new labels in a shared repository.
  • Review names and content during pull requests.
  • Use packaging or metadata tracking to manage label lifecycles across orgs.

7. Include Labels in Version Control

Always commit CustomLabels.labels-meta.xml to source control. This ensures that changes to text, translations, and new labels are consistently deployed through DevOps pipelines.
