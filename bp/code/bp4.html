Apex Trigger Framework

Standard Naming Conventions

Trigger File [Object]Trigger
Handler Class [Object]TriggerHandler
Example AccountTrigger, AccountTriggerHandler

1. One Trigger Per Object

Multiple triggers create unpredictable execution order and maintenance nightmares.

2. Logic-less Triggers

Triggers should only route to handler methods. All business logic stays in handler classes.

3. Context-Aware

Use Trigger.isInsert, isBefore, isAfter, etc. to control execution flow precisely.

4. Recursion Control

Implement static flags or use a trigger framework to prevent infinite loops.

Complete Trigger Pattern

Trigger File (AccountTrigger.trigger)

Apex Trigger 
trigger AccountTrigger on Account (before insert, before update, 
                                   after insert, after update, after delete) {
    AccountTriggerHandler handler = new AccountTriggerHandler();
    
    if (Trigger.isBefore) {
        if (Trigger.isInsert) {
            handler.onAfterInsert(Trigger.new);
        }
        if (Trigger.isUpdate) {
            handler.onAfterUpdate(Trigger.new, Trigger.oldMap);
        }
        if (Trigger.isDelete) {
            handler.onAfterDelete(Trigger.old);
        }
    }
}

Handler Class (AccountTriggerHandler.cls)

Apex 
public class AccountTriggerHandler {
    
    // Recursion prevention
    private static Boolean isExecuting = false;
    
    public void onBeforeInsert(List<Account> newAccounts) {
        if (isExecuting) return;
        isExecuting = true;
        
        validateAccounts(newAccounts);
        setDefaultValues(newAccounts);
        
        isExecuting = false;
    }
    
    public void onAfterInsert(List<Account> newAccounts) {
        if (isExecuting) return;
        isExecuting = true;
        
        createRelatedRecords(newAccounts);
        sendNotifications(newAccounts);
        
        isExecuting = false;
    }
    
    private void validateAccounts(List<Account> accounts) {
        // Validation logic
    }
    
    private void setDefaultValues(List<Account> accounts) {
        for (Account acc : accounts) {
            if (acc.Rating == null) {
                acc.Rating = 'Cold';
            }
        }
    }
}

Flow Types & Use Cases

üöÄ Before-Save Flow (Fast Field Updates)

10x faster than after-save. Executes before record is saved.

‚úì Best for: Field updates on the same record, validation, default values

‚úó Cannot: Access record ID, create related records, send emails

üì® After-Save Flow

Executes after record is committed to database.

‚úì Best for: Creating related records, sending emails, callouts, complex logic

‚è∞ Scheduled Flow

Runs on a schedule (daily, weekly, etc.)

‚úì Best for: Batch processing, daily summaries, data cleanup

üñ•Ô∏è Screen Flow

Interactive UI for users to input data.

‚úì Best for: Guided wizards, data collection, custom processes

Flow Optimization

‚úÖ

Always Define Entry Criteria

Prevent unnecessary flow execution. Use specific conditions.

‚úÖ

Bulkify Flow Logic

Use collections and "Get Records" outside of loops.

‚úÖ

Use Decision Elements

Branch logic efficiently instead of creating multiple flows.

‚ùå

Never Put DML Inside Loops

Collect records in a collection, then do single Create/Update outside loop.

‚ö†Ô∏è Critical Mistakes to Avoid

  • 1. DML in Loops: Never put Get/Create/Update Records inside a Loop element.
  • 2. No Error Handling: Always add fault paths to handle failures gracefully.
  • 3. Hardcoded IDs: Use variables and queries instead of hardcoding record IDs.
  • 4. Missing Null Checks: Always check if collections or records are null before using.

import { getRecord } from 'lightning/uiRecordApi';
import getAccounts from '@salesforce/apex/AccountController.getAccounts';

const FIELDS = ['Account.Name', 'Account.Industry'];

export default class AccountCard extends LightningElement {
    // Public properties
    @api recordId;
    @api flexipageRegionWidth;
    
    // Private reactive properties
    @track accounts = [];
    error;
    
    // Wire service to get record data
    @wire(getRecord, { recordId: '$recordId', fields: FIELDS })
    account;
    
    // Lifecycle hooks
    connectedCallback() {
        this.loadAccounts();
    }
    
    // Private methods
    loadAccounts() {
        getAccounts()
            .then(result => {
                this.accounts = result;
            })
            .catch(error => {
                this.error = error;
            });
    }
    
    // Event handlers
    handleClick(event) {
        const selectedId = event.target.dataset.id;
        this.dispatchEvent(new CustomEvent('accountselected', {
            detail: selectedId
        }));
    }
    
    // Getters
    get hasAccounts() {
        return this.accounts && this.accounts.length > 0;
    }
}

HTTP Callout Example

Apex 
public class ExternalServiceCallout {
    
    @future(callout=true)
    public static void makeCallout(String endpoint, String payload) {
        HttpRequest req = new HttpRequest();
        req.setEndpoint(endpoint);
        req.setMethod('POST');
        req.setHeader('Content-Type', 'application/json');
        req.setHeader('Authorization', 'Bearer ' + getAccessToken());
        req.setBody(payload);
        req.setTimeout(120000); // 120 seconds
        
        try {
            Http http = new Http();
            HttpResponse res = http.send(req);
            
            if (res.getStatusCode() == 200) {
                String responseBody = res.getBody();
                // Process response
            } else {
                System.debug('Error: ' + res.getStatusCode() + ' ' + res.getStatus());
            }
        } catch (Exception e) {
            System.debug('Callout exception: ' + e.getMessage());
        }
    }
    
    private static String getAccessToken() {
        // Retrieve from Named Credential or Custom Setting
        return 'your_access_token';
    }
}

with sharing

Enforces record-level security. User sees only records they have access to.

without sharing

Runs in system context. User can access all records regardless of sharing rules.

inherited sharing

Inherits sharing from the calling context. Default if not specified.
