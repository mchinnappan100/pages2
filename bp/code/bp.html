Bulkification

Always write code to handle multiple records at once.

// Bad: trigger.new[0];
// Good: for(Account a : Trigger.new) { ... }

Governor Limits

  • No SOQL inside loops.
  • No DML inside loops.
  • Use Map collections to avoid nested loops.

Apex Trigger Framework

Standard Naming Conventions

Trigger File [Object]Trigger
Handler Class [Object]TriggerHandler
Example AccountTrigger, AccountTriggerHandler

One Trigger Per Object

Do not create multiple triggers for the same object. It creates execution order issues.

Logic-less Triggers

Triggers should only delegate tasks to a Handler Class.

Handler Pattern Example

trigger AccountTrigger on Account (before insert, after update) {
    if (Trigger.isBefore && Trigger.isInsert) {
        AccountHandler.onBeforeInsert(Trigger.new);
    }
}

Salesforce Flow

Standard Naming Conventions

  • Flow Label Object - Function - TriggerType
    e.g., Case - Send Email - Record Triggered
  • Variables (Hungarian Notation - Optional but clean)
    varCount (Variables) rcAccount (Record Single) colContacts (Collection) frmInput (Formula)

Flow Types Best Practice

  • Before-Save Flows (Fast Field Updates): 10x faster. Use for updates to the same record that triggered the flow.
  • After-Save Flows: Use for related records, emails, or complex actions.
  • Entry Criteria: Always define "Entry Conditions" to prevent flows from running unnecessarily.

Critical Mistakes

DML in Loops: Never put "Get Records", "Create Records", or "Update Records" inside a Loop element.

Dynamic Forms

Component Visibility

Hide components not relevant to the user or record status to improve load time.

Lightning Data Service (LDS)

Prefer wire adapters over imperative Apex. It caches data efficiently.

@wire(getRecord, { recordId: '$recordId', fields: [NAME_FIELD] })

Communication Patterns

  • Parent to Child: @api properties.
  • Child to Parent: this.dispatchEvent (Custom Events).
  • Unrelated: Lightning Message Service (LMS).
