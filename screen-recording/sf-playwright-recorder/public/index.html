SF Playwright Recorder — Help

Quick Start

  • Add an Org Profile with your SF username & password.
  • Click ⚡ Quick Login & Record — the browser opens, logs in automatically, and starts recording.
  • Perform your flow in the browser window.
  • Click ■ Stop Recording, give the test a filename, click Save.
  • Switch to the ▶ Run tab and click ▶ Run.

Org Profiles (~/.sfplaywright-recorder)

Profiles are stored in a plain YAML dotfile in your home directory. You can edit it by hand or use the UI.

  • ＋ — add a new profile
  • ✏ — edit (leave password blank to keep existing)
  • ✕ — delete
  • Selecting a profile auto-fills the Login URL field.

Saved Auth State

After logging in, click Save Auth State to store cookies & tokens in generated/sf-auth.json. Tick Use saved auth on future recordings to skip the login page entirely.

Selector Quality

The recorder prefers stable selectors in this order:

  1. getByLabel() — associated <label> text
  2. getByPlaceholder()
  3. getByRole() — ARIA role + accessible name
  4. getByTestId()
  5. getByText() — for buttons & links
  6. Shadow-DOM path — for Salesforce LWC components
  7. CSS selector — last resort

Fill Dedup

Keystroke-by-keystroke fills are collapsed to a single fill(finalValue) per field. Only the last value typed before you leave the field is kept.

Running Tests

Select a saved test in the ▶ Run tab and click ▶ Run. Output streams live. Click ■ Kill to abort.

You can also upload a .spec.js file from outside the project using the ⬆ Upload button.

From the terminal:

npx playwright test generated/my-test.js

CLI Options

node server.js -p 4000Use port 4000
--no-openDon't auto-open browser
--helpShow CLI help

If the requested port is busy the server automatically tries the next one.

Splitter

Drag the vertical bar between the sidebar and the code panel to resize. Double-click to snap back to the default width.

Dotfile Format

profiles:
  - name: my-dev-org
    login-url: https://login.salesforce.com
    username: me@example.com.dev
    password: Secret1
    org-type: developer

Keep this file private — treat it like .env.
