Level 01

Motivation — Why Replace RNNs?

01
🐢

The Sequential Bottleneck

Recurrent models compute hidden states one step at a time: h_t depends on h_{t-1}. This inherently sequential nature precludes parallelization — critical at longer sequences.

h_t = f(h_{t-1}, x_t) — must process in order
O(n) sequential operations — cannot parallelize across positions
Memory constraints limit batch size for long sequences
LSTM/GRU mitigate but don't eliminate the problem
📏

Long-Range Dependency Problem

In RNNs, relating two positions n steps apart requires n sequential operations. Information from early positions must survive through long chains of transformations — prone to vanishing gradients.

RNN maximum path length: O(n) — hard for distant dependencies
CNN (dilated): O(log_k n) — better, but still grows with distance
Transformer: O(1) — every position directly attends to every other
"making...more difficult" — verb relates directly to complement
💡

The Core Idea

Replace sequential processing entirely with attention mechanisms — allowing every position to directly attend to every other position in a single step, regardless of distance.

"The Transformer is the first transduction model relying entirely on self-attention"
No recurrence, no convolution — attention only
Fully parallelizable — all positions processed simultaneously
Enables training in 12 hours vs. weeks for comparable RNN systems
🔍

Interpretability Bonus

Attention mechanisms produce weights over all input positions — making it possible to visualise what the model attends to, providing insight into learned linguistic structure.

Different attention heads capture different syntactic/semantic roles
Head 1: subject-verb agreement; Head 2: anaphora resolution, etc.
Figure 3 shows heads correctly tracking "making...more difficult"
Not a black box — attention weights are inspectable
Level 02

Model Architecture

02
🏗️

Encoder Stack

N = 6 identical layers. Each layer has two sub-layers: multi-head self-attention + position-wise feed-forward network. Residual connections + layer norm around each.

Sub-layer 1: Multi-Head Self-Attention
Sub-layer 2: Position-wise Feed-Forward Network
Each sub-layer: LayerNorm(x + Sublayer(x))
All sub-layers output dimension d_model = 512
Encoder Layer (repeated N=6 times): Input x (shape: seq_len × d_model=512) ↓ [Sub-layer 1] MultiHeadAttention(x, x, x) # self-attention + Residual(x) → LayerNorm ↓ [Sub-layer 2] FFN(x) = ReLU(xW₁+b₁)W₂+b₂ + Residual(x) → LayerNorm ↓ Output (seq_len × 512)
🔓

Decoder Stack

N = 6 identical layers with three sub-layers. The extra third sub-layer performs cross-attention over the encoder output. Self-attention is masked to prevent attending to future positions.

Sub-layer 1: Masked Multi-Head Self-Attention
Sub-layer 2: Multi-Head Cross-Attention (queries from decoder, keys/values from encoder)
Sub-layer 3: Position-wise Feed-Forward Network
Masking: future positions set to −∞ before softmax
➕

Residual Connections & Layer Norm

Every sub-layer output is added back to its input (residual / skip connection) and normalised. This enables training of deep networks by preventing vanishing gradients.

Formula: output = LayerNorm(x + Sublayer(x))
Residual connections: gradient highways through the network
Layer normalisation: stabilises activations across positions
Applied identically in both encoder and decoder
📡

Positional Encoding

Since the model has no recurrence or convolution, position information must be explicitly injected. Sinusoidal positional encodings are added to the input embeddings.

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
Same dimension d_model as embeddings — can be summed
Enables relative-position reasoning: PE(pos+k) is linear function of PE(pos)
Position 0: [sin(0/1), cos(0/1), sin(0/100), ...] Position 1: [sin(1/1), cos(1/1), sin(1/100), ...] Position 2: [sin(2/1), cos(2/1), sin(2/100), ...] Each position gets a unique "fingerprint" vector. The sinusoidal pattern means relative offsets are preserved as linear transformations. Learned PE gives nearly identical results (Table 3).
⚡

Feed-Forward Networks

Each encoder/decoder layer contains a position-wise FFN applied identically to each position. Two linear transformations with a ReLU in between — like two conv layers with kernel size 1.

FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
Input/output dimension: d_model = 512
Inner (hidden) dimension: d_ff = 2048
"Same weights across positions, different weights per layer"
🔤

Embeddings & Output

Learned embeddings convert tokens to d_model-dimensional vectors. A shared weight matrix is used for input embeddings, output embeddings, and the pre-softmax linear transformation.

Input/output tokens → d_model = 512 vectors via learned embedding
Weight sharing: 3 matrices share the same weights (reduces parameters)
Embedding weights scaled by sqrt(d_model) before adding positional encoding
Decoder output → Linear → Softmax → token probabilities
Level 03

Attention Mechanisms

03
Scaled Dot-Product Attention
Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V
Q = queries (d_k dim), K = keys (d_k dim), V = values (d_v dim). Scale by 1/√d_k prevents softmax saturation at large d_k values.
Multi-Head Attention
MultiHead(Q, K, V) = Concat(head₁, ..., headₕ) · Wᴼ head_i = Attention(Q·Wᵢᴼ, K·Wᵢᴷ, V·Wᵢᵛ)
h = 8 heads, d_k = d_v = d_model/h = 64. Each head projects to a different subspace and attends independently.
🔢

Why Scale by 1/√d_k?

For large d_k, dot products grow large in magnitude, pushing softmax into regions with extremely small gradients — causing vanishing gradients during training.

Dot products of random unit vectors have variance proportional to d_k
Dividing by √d_k normalises variance back to ~1
Prevents softmax from becoming a one-hot selector
Additive attention doesn't suffer this — no scaling needed there
🎭

Why Multi-Head?

A single attention head averages over all subspaces. Multi-head attention allows the model to jointly attend to information from different representation subspaces at different positions.

h = 8 heads, each attending to a 64-dimensional subspace
Head 1 might track subject-verb agreement
Head 2 might track coreference resolution
Outputs concatenated then projected through W^O
1️⃣

Encoder Self-Attention

Queries, keys, and values all come from the previous encoder layer output. Each position can attend to all positions in the previous layer — full bidirectional context.

Q = K = V = previous encoder layer output
Every position can attend to every other position
Builds up rich contextual representations
No masking — full access to input sequence
🔗

Encoder-Decoder Cross-Attention

Queries come from the decoder's previous layer; keys and values come from the encoder output. Every decoder position can attend to all encoder positions — classic seq2seq alignment.

Q = decoder layer output; K = V = encoder final output
Decoder "looks up" relevant source positions for each target token
Replaces the attention mechanism in classic seq2seq (Bahdanau 2015)
Generalises alignment beyond a single attention vector
🎭

Masked Decoder Self-Attention

Decoder self-attention is masked to prevent each position from attending to future positions — preserving the auto-regressive property required for generation.

Future positions set to −∞ before softmax → effectively 0 weight
At position i: can attend to positions 0..i only
Ensures output token i only depends on tokens 0..i-1
Enables parallel training while maintaining generation order
📊

Attention Head Ablation

Table 3 shows that the number of heads matters: too few or too many both hurt performance. Eight heads is optimal for the base model.

h=1: BLEU 24.9 (−0.9 vs base)
h=4: BLEU 25.5
h=8: BLEU 25.8 ← optimal
h=16: BLEU 25.8 (same)
h=32: BLEU 25.4 (−0.4, degraded)
Training

Training Details & Hyperparameters

04
Learning Rate Schedule
lrate = d_model⁻⁰·⁵ · min(step⁻⁰·⁵, step · warmup_steps⁻¹·⁵)
Linear warmup for first warmup_steps=4000 steps, then inverse-square-root decay. d_model=512, so peak LR ≈ 0.001 at step 4000.
⚙️

Base vs Big Model

Two model configurations were evaluated, demonstrating that larger models consistently improve both perplexity and BLEU score.

Base: d_model=512, d_ff=2048, h=8, N=6, P_drop=0.1 → 65M params
Big: d_model=1024, d_ff=4096, h=16, N=6, P_drop=0.3 → 213M params
Base: 100K steps, ~12 hours on 8 P100s
Big: 300K steps, ~3.5 days on 8 P100s
🎯

Regularisation

Two regularisation techniques are used: residual dropout and label smoothing. Together they prevent overfitting and improve BLEU scores.

Residual Dropout: P_drop=0.1 applied to each sub-layer output before residual addition
Also applied to embedding+positional encoding sums
Label Smoothing: ε_ls=0.1 — "hurts perplexity but improves BLEU"
P_drop=0.0 ablation: BLEU drops from 25.8 → 24.6
🔧

Optimizer

Adam with a custom learning rate schedule that warms up linearly then decays — essential for stable training of the Transformer architecture.

Adam: β₁=0.9, β₂=0.98, ε=10⁻⁹
warmup_steps = 4000
Linear warmup prevents instability in early training
Inverse-sqrt decay prevents overfitting at convergence
📦

Data & Inference

Trained on standard WMT 2014 EN-DE and EN-FR datasets with byte-pair encoding. Inference uses beam search with checkpoint averaging.

EN-DE: 4.5M sentence pairs, ~37K BPE vocab (shared src/tgt)
EN-FR: 36M sentences, 32K word-piece vocab
Beam size = 4, length penalty α = 0.6
Checkpoint averaging: last 5 (base) or last 20 (big)

Layer Type Comparison (Table 1)

06
Layer TypeComplexity per LayerSequential OpsMax Path Length
Self-AttentionO(n² · d)O(1)O(1)
Recurrent (RNN/LSTM)O(n · d²)O(n)O(n)
ConvolutionalO(k · n · d²)O(1)O(log_k n)
Self-Attention (restricted, r)O(r · n · d)O(1)O(n/r)

n = sequence length, d = dimension, k = convolution kernel size, r = restriction neighbourhood. Self-attention wins on sequential operations and maximum path length — the two axes that matter most for training speed and long-range dependencies.
