Models

Models in Detail

02
🖼️

1. I-JEPA — Image JEPA

Introduced by Yann LeCun's team in 2023. Instead of reconstructing pixels (like MAE), I-JEPA predicts latent semantic representations — the core conceptual advance over generative self-supervised methods.

Learns semantic features without pixel reconstruction
No hand-crafted augmentations required
Excellent visual representations for downstream tasks
Over 10× more compute-efficient than MAE ViT-H/14

Applications: Image classification · Detection · Segmentation · Medical imaging

Meta AI Blog ↗
🎬

2. V-JEPA — Video JEPA

Extends the JEPA principle to video. Rather than predicting future pixels, V-JEPA predicts future embeddings — making it dramatically better at understanding temporal dynamics and interactions.

Object motion and physical interactions
Action recognition across time
No pixel prediction — predicts latent future states
Strong temporal reasoning capabilities

Applications: Video search · Action recognition · Robotics · Autonomous systems

GitHub ↗
🚀

3. V-JEPA 2

Released in 2025. A major leap — trained on over one million hours of video with much larger models, enabling strong world modeling, robotic planning, and video question answering.

1M+ hours of video training data
Significantly larger model capacity
Robotic planning and physical reasoning
High-quality visual embeddings for LLM integration
Video question answering

Applications: World modeling · Robotic control · Video QA · Downstream reasoning

Announcement ↗
🔬

4. V-JEPA 2.1

The newest iteration of V-JEPA. Focuses on improving dense visual features and spatial localization — particularly for robotic manipulation tasks requiring fine-grained spatial understanding.

Dense visual features for fine-grained tasks
Improved spatial localization
Robotic manipulation (pick, place, assemble)
Temporal consistency across frames
Currently Meta's strongest self-supervised vision model

Applications: Robotic manipulation · Spatial reasoning · Embodied AI

arXiv ↗
🌍

5. JEPA World Models (JEPA-WM)

The most architecturally ambitious variant. Instead of "what an image looks like," JEPA-WM learns dynamics, physics, and planning — supporting full latent-space simulation for physical tasks.

Learns physical dynamics and causal structure
Planning entirely in latent space (no pixel simulation)
Environments: RoboCasa · DROID · MetaWorld · Push-T · PointMaze
Pretrained checkpoints and training code released
Most direct instantiation of LeCun's 2022 vision

Applications: Physical planning · Robotic control · Embodied reasoning

Usability

Are They Usable Today?

03

Yes. All five JEPA models are available as open-source PyTorch implementations with pretrained weights.

↓ Download pretrained weights

All models release pretrained checkpoints on HuggingFace or Meta's GitHub — ready to use as feature extractors with no additional training.

🎯 Fine-tune for your task

All models support standard fine-tuning on downstream tasks: classification, detection, segmentation, action recognition, and manipulation.

📦 Feature extractors

Extract high-quality visual embeddings from images or video frames and feed them into classifiers, retrieval systems, or language models.

🤖 Robotic planners

JEPA-WM and V-JEPA 2 support latent-space planning for robotic control — the agent plans without ever simulating in pixel space.

🔗 Integrate with language models

V-JEPA 2 produces visual embeddings that downstream LLMs can directly reason over — enabling multimodal understanding without a generative vision decoder.

🐍 PyTorch throughout

All implementations are standard PyTorch. No proprietary frameworks required. Compatible with HuggingFace Transformers ecosystem for easy integration.

JEPA vs. LLMs — What JEPA Is Not

04

JEPA is not a replacement for large language models. The two approaches are complementary — not competitive.

DimensionLLM (e.g. GPT-4)JEPA (e.g. V-JEPA 2)
What it predictsNext token (word/subword)Next latent representation
Primary modalityTextImages, video, physical state
World modelImplicit, text-basedExplicit, grounded in perception
PlanningToken-level chain-of-thoughtLatent-space trajectory optimisation
Physical understandingLearned from text descriptionsLearned from raw visual observation
Best useReasoning, language, codingPerception, physical simulation, robotics
ComplementarityJEPA perceptual representations + LLM reasoning = powerful multimodal agent

"JEPA addresses several limitations of autoregressive models: no need to predict every pixel or token, focuses on abstract semantics, better sample efficiency, better suited for planning, more robust representations, and a natural fit for robotics and embodied AI."

Enterprise AI

JEPA + LLM + Agent Frameworks

05

A compelling direction for enterprise automation — combining JEPA's structured state representations with LLM reasoning and agentic execution.

📊

JEPA for State Representation

Use JEPA to learn structured state representations of complex systems — workflow states, document layouts, dashboard configurations — and predict how they evolve.

Encode current system state as a rich latent vector
Predict future states given proposed actions
Identify anomalies or at-risk states before they occur
🧠

LLM for Reasoning & Planning

Pair JEPA embeddings with an LLM (Claude, GPT-4, Llama) for natural-language reasoning over the structured world state — combining perception with language.

Feed JEPA embeddings as context into an LLM
LLM reasons over structured state in natural language
Generate multi-step action plans grounded in reality
⚡

Agent Frameworks for Execution

Use Agentforce, LangChain, or similar frameworks to execute the LLM's action plan across enterprise systems — Salesforce, databases, APIs — in a governed loop.

JEPA perceives and models workflow state
LLM reasons and plans next actions
Agent executes across Salesforce, ERP, APIs
Loop repeats with updated JEPA state observation
