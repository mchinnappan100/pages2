Level 01

Motivation & Limitations of Current AI

01
🤔

Why Current AI Falls Short

Despite spectacular successes — AlphaFold, GPT, DALL-E — society still lacks Level 5 self-driving, domestic robots, and truly general virtual assistants. The fundamental gap is common sense.

LLMs have no understanding of physical reality — knowledge comes only from text
Self-driving only works in fully-mapped environments with good weather
Systems are brittle and specialized — they "make stupid mistakes"
Cannot reason or plan across long horizons
👶

What Babies Can Do That AI Cannot

A human infant develops a rich world model far more efficiently than any current AI system — through observation alone, before any linguistic description.

Object permanence by month 6
Gravity, inertia, momentum by month 10
Learns from observation — no labelled data, no trial-and-error rewards
Generalises to new situations immediately
⚠️

The Blurry Prediction Problem

When a model is trained to predict a single output in raw signal space but the true target is multimodal, gradient descent averages all possibilities — producing a blurry, unrealistic output.

Predict next video frame when object could go left or right → ghostly average
Root cause: stochasticity + limited perception make full-detail prediction impossible
Mode-dropping: generative model represents only some modes, ignores others
Solution: predict in abstract representation space, not pixel space
📊

The Four Learning Paradigms

LeCun analyses four learning paradigms and argues for the dominance of self-supervised learning with world models over supervised or RL approaches.

Supervised: data-hungry, requires massive labelled datasets
Reinforcement: sample-inefficient, billions of simulated steps for simple tasks
Self-supervised: predicts parts of input from other parts — "central element for next AI revolution"
Model-based / World-model: plan via differentiable simulation — no real-world trial-and-error needed
SSL learns: "What information bits are most predictable from other parts?" → Works beautifully for text (next-word prediction → GPT) → Fails for images if predicting pixels: too many valid completions → Solution: predict in representation space (I-JEPA, H-JEPA)

The paper argues SSL will provide ~10⁵× more bits of training signal per sample than supervised learning, making it the key to sample efficiency.

🔢

Why Probabilistic Models Are Intractable

Normalising distributions in high-dimensional continuous spaces requires computing a partition function that is computationally intractable — making probabilistic models unsuitable as world models.

Word prediction works: probability over a finite dictionary is tractable
Video frame prediction: ∫ exp[−βE(y)] dy over all possible frames is intractable
Solution: give up representing probability distributions; use energy functions instead
EBMs assign scalar energy to (x,y) pairs — no normalisation needed for inference
🎯

Three Core Challenges

LeCun identifies three fundamental challenges that must be solved on the path to autonomous AI.

1. Representations: Learning world representations via SSL without supervision
2. Reasoning: Reasoning in a way compatible with deep learning (gradient-based)
3. Planning: Planning complex hierarchical action sequences across multiple timescales
Level 02

The Six-Module Architecture

02

"The main premise is that intelligent behavior emerges from the interplay between a world model, a cost module driving behavior through intrinsic motivation, and a hierarchical joint-embedding predictive architecture trained with self-supervised learning."

— Yann LeCun, A Path Towards Autonomous Machine Intelligence (2022)
⚙️

Configurator

The top-level controller. Sets the overall mode, goals, and priorities for the entire system at any given time — configuring all other five modules based on the current task or behavioral objective.

Directs attention and weighting across modules
Adjusts the relative weighting of intrinsic cost vs. learned critic
Enables task-switching without retraining
Analogous to executive control in cognitive science
👁️

Perception Module

Processes raw sensory inputs (vision, audio, proprioception, etc.) into abstract, compressed representations suitable for the world model and memory.

Filters irrelevant low-level sensory details
Outputs structured representations, not raw pixels
Feeds both Short-Term Memory and the World Model
Can handle multi-modal inputs simultaneously
🌍

World Model (Central Element)

The heart of the architecture. Predicts future states given current state and proposed actions. Trained via H-JEPA with self-supervised learning on observed data — no labels or rewards needed.

Takes current state (from STM) + proposed action → predicts future state
Fully differentiable — gradients flow from cost module back to actor
Trained with H-JEPA: predicts representations, not pixels
Can roll out multiple future trajectories for planning

Planning to reach a cup on a table: the world model simulates reaching trajectories, predicts collisions, predicts cup grasp success — all without moving the robot arm. The cost module evaluates each predicted trajectory. Gradients flow back to the actor to improve the action sequence.

Key: "The cost function is known and all modules are fully differentiable, so no action needs to be taken in reality to predict the future cost."

💰

Cost Module

Evaluates the desirability of predicted future states. Has two sub-components: a fixed intrinsic cost and a trainable critic.

Intrinsic cost (fixed): energy efficiency, danger proximity, homeostasis, basic needs
Critic (trainable): learned estimator of future cumulative cost — like a value function in RL
Configurator weights the two components based on current task/goal
Drives intrinsic motivation — behavior emerges without external reward
🤖

Actor Module

Selects and outputs action sequences. Can operate via two inference modes: inference-by-optimisation (slow, precise) or amortised inference (fast, approximate).

Inference by optimisation: gradient descent through world model to minimise cost
Amortised inference: trained policy network that directly outputs actions (fast)
Operates at multiple timescales corresponding to H-JEPA hierarchy levels
Replaces RL's trial-and-error with gradient-based planning
🧠

Short-Term Memory (STM)

Stores the current world state estimate. Updated continuously by both the perception module (from real observations) and the world model (from predicted future states during planning).

Maintains coherent state estimate across multiple perception-planning cycles
Integrates real observations with simulated predictions
Enables reasoning about objects outside the current field of view
Analogous to working memory in cognitive neuroscience

Perception–Planning–Action Cycle

1 Perception updates Short-Term Memory

Raw sensory inputs processed by Perception module → abstract representations stored in STM as current world state.

2 Actor proposes candidate action sequences

The Actor generates one or more candidate action trajectories to be evaluated by the World Model.

3 World Model rolls out predicted futures

For each candidate action, the World Model predicts the resulting future state sequence — entirely in simulation.

4 Cost module evaluates predicted outcomes

Intrinsic cost + critic assigns a scalar cost to each predicted future trajectory.

5 Gradients flow back to refine actions

Because all modules are differentiable, gradients of cost w.r.t. action can be computed — the actor refines its proposal without any real-world execution.

6 Best action executed; loop repeats

The lowest-cost action is taken in the real world. Perception updates STM with the new observation. The cycle repeats.

Level 03

Energy-Based Models

03
⚡

EBM Core Concept

EBMs assign a scalar energy F(x,y) to every configuration of variables. Compatible pairs receive low energy; incompatible pairs receive high energy. Inference = minimise energy over y.

F(x,y) → low: compatible (car driving slowly in snow)
F(x,y) → high: incompatible (car teleporting)
Inference: ŷ = argmin_y F_w(x,y)
No partition function needed — avoids intractability
Ellipse EBM (latent variable example): E_{r₁,r₂}(y,z) = (y₁ − r₁·sin z)² + (y₂ − r₂·cos z)² Inference: find z that minimises energy → Finds the closest point on the ellipse to y → Model learns a 1D manifold in 2D space → No explicit probability distribution needed
🎭

Latent Variable EBMs

Extend standard EBMs with hidden variables z whose correct value is never observed. They capture information about y not available from x — enabling structured prediction and uncertainty representation.

z encodes: pose, lighting, segmentation, other drivers' intentions
Minimisation: F_w(x,y) = argmin_z E_w(x,y,z)
Marginalization: F_w = −(1/β) log ∫ exp[−βE] dz (expensive)
Critical: z must have minimised information capacity to prevent collapse
💥

The Collapse Problem

Joint-embedding architectures risk collapse — encoders ignore all inputs and output constant vectors, making all energies equal and the model useless.

Encoder outputs s_x = s_y = constant for all inputs → energy always = 0
Any model capable of multimodal output is vulnerable
Must explicitly prevent via contrastive, regularized, or architectural methods
Latent variable z collapse: z memorizes y instead of learning useful representation
🔄

Contrastive Training Methods

Push down energy of training samples; pull up energy of contrastive (negative) samples. Works but scales badly — potentially exponentially — with data dimensionality.

Pairwise hinge loss: [F(x,y) − F(x,ŷ) + m]⁺
SimCLR, MoCo, InfoNCE use contrastive objectives
Max-likelihood is a special case: Monte Carlo samples as negatives
Fatal flaw: generating good negatives in high dimensions is exponentially hard
📐

Regularised / Architectural Methods

Limit the volume of low-energy space by construction or explicit regularisation. Preferred over contrastive methods — better dimensional scaling.

PCA, k-means, normalising flows: architectural constraint on low-energy volume
VICReg: enforces variance above threshold + minimises off-diagonal covariance
Barlow Twins: drives cross-correlation matrix toward identity
BYOL/SimSiam: momentum encoder + stop-gradient tricks
Sparse/variational autoencoders: minimise z information capacity
📚

Historical EBMs: Hopfield & Boltzmann

Understanding the historical development of EBMs illuminates why the JEPA approach is needed.

Hopfield (1982): recurrent network, associative memory, but suffers from spurious minima
Boltzmann Machine (1983): Hinton & Sejnowski; first hidden units (latent variables)
RBM: restricted to visible–hidden connections; MCMC training is too costly
Denoising AE: contrastive EBM; works for text, fails for continuous domains due to multimodality
Level 04

JEPA & H-JEPA

04
🧩

JEPA — Joint Embedding Predictive Architecture

An EBM that combines embedding modules with latent variables. Learns dependencies between x and y at the level of learned internal representations — not raw signal space.

Encoder E_x: x → s_x (context representation)
Encoder E_y: y → s_y (target representation)
Predictor g: (s_x, z) → ŝ_y ≈ s_y
Energy = distance(ŝ_y, s_y) in representation space
Latent z encodes information about y missing from x
I-JEPA (Image instantiation of JEPA): Context: large region of image (85-100%) Target: 4 small blocks (15-20% each) E_x(context) → s_x E_y(full image) → s_y (EMA encoder) g(s_x, position_tokens) → ŝ_y Loss = ‖ŝ_y − s_y‖² Encoders can ignore texture; they only need to predict semantic content (pose, object identity)
🏛️

H-JEPA — Hierarchical JEPA

Stacks multiple JEPA levels at increasing levels of abstraction and temporal scale, enabling hierarchical planning across multiple time horizons.

JEPA-1 (bottom): short-term predictions, low-level representations
JEPA-2 (middle): medium-term predictions via pooled representations
JEPA-N (top): long-horizon predictions, highest abstraction
CNN + pooling between levels to coarse-grain representations
Enables hierarchical planning: complex task → subtasks at each level
🔍

Why Representation Space Prediction Wins

Predicting in representation space allows encoders to filter out unpredictable details and focus on what matters — solving blurriness and mode-dropping simultaneously.

Encoder discards: texture, exact lighting, background noise
Encoder retains: object identity, pose, spatial relationships
Latent z represents the unpredictable variance — encodes "which mode"
Result: crisp, semantically meaningful representations without blurriness
⚡

Amortised Inference

Rather than solving an optimisation problem at inference time for every input, amortised inference trains a separate encoder to directly predict the optimal representation — trading training time for fast inference.

Inference-by-optimisation: gradient descent through world model (slow, precise)
Amortised: encoder s_x = Enc(x) approximates argmin (fast)
Used in actor module: amortised policy network outputs actions directly
Can be retrained to improve as the world model improves
🔗

JEPA vs. Generative Models

Unlike generative models (VAEs, diffusion, MAE) that reconstruct in pixel space, JEPA predicts in embedding space — avoiding the fundamental limitation of pixel-space prediction.

Generative: must predict all unpredictable pixel details → blurry or mode-dropping
JEPA: encoder abstracts away irrelevant details before prediction
MAE ViT-H/14 linear probe: 77.2% vs I-JEPA 79.3% (fewer epochs)
JEPA also works for multimodal data without redesigning per modality
🌐

JEPA as a World Model

The H-JEPA serves as the world model in LeCun's six-module architecture — predicting future states in abstract representation space to enable safe, efficient model-predictive planning.

Trained via SSL on observed sequences — no labels or rewards needed
Predicts future representations given current state + proposed action
Fully differentiable: cost gradients flow through world model to actor
Multi-timescale: H-JEPA levels enable reasoning at different horizons
Best Practices

Training Strategies & Key Insights

05

✓ Prefer regularised over contrastive methods

Regularised training (VICReg, Barlow Twins) scales better with data dimensionality than contrastive methods, which require exponentially many negatives in high dimensions.

Contrastive: need good negative samples → exponential scaling with dimension ❌ Regularised (VICReg): → variance term: var(z) ≥ threshold → covariance term: off-diagonal near 0 → better dimensional scaling ✅

✓ Minimise latent variable information capacity

If z has too much capacity, it will memorise y instead of learning useful representations — causing a form of collapse where the predictor ignores the context entirely.

Too much z capacity: z simply memorizes target y → context encoder learns nothing ❌ Right z capacity: z encodes only unpredictable variance (which mode) while encoder learns the predictable semantic content ✅

✓ Use EMA for stable targets

The EMA target encoder provides slowly-evolving, stable prediction targets — preventing collapse without requiring negative pairs or contrastive losses.

θ̄ ← τ·θ̄ + (1−τ)·θ τ: 0.996 → 1.0 over training → Target always slightly ahead of encoder → Non-trivial, stable learning signal → No negatives needed ✅

✓ SSL is the most data-efficient paradigm

Self-supervised learning extracts far more information per sample than supervised or RL — making it the right foundation for world model learning.

Bits per sample: Supervised (ImageNet): ~10 bits (class label) Self-supervised (masking): ~10⁵ bits (predicting every masked patch) → SSL is ~10,000× more informationally efficient per training sample ✅

✓ Replace RL with gradient-based planning

Rather than learning by trial-and-error with a scalar reward, LeCun argues for planning through a differentiable world model — enabling safe, efficient planning without real-world execution.

RL: Execute action → observe outcome → update Millions of real/simulated steps needed ❌ Model-based planning: Simulate action through world model Gradient descent to minimise predicted cost No real-world action needed for planning ✅

✓ Intrinsic motivation over external rewards

Built-in cost functions (energy efficiency, danger avoidance, curiosity) drive useful behaviour even without external reward signals — enabling more autonomous operation.

External reward (RL): Needs hand-designed reward function Reward hacking: agent exploits loopholes ❌ Intrinsic cost: Built-in drives: safety, efficiency, goals Critic learns: expected future cost Configurator weights both per task ✅

✓ Hierarchical planning via H-JEPA

Complex long-horizon tasks decompose naturally into subtasks at each level of the H-JEPA hierarchy — each level operating at its appropriate timescale and abstraction.

Task: "Cook dinner" Level 3 (high): dinner → shopping → prep → cook Level 2 (mid): prep → chop onions → boil water Level 1 (low): chop → position knife → apply force Each level regularly updated by new observations → Robust to unexpected events ✅

✓ Maximise likelihood = contrastive method

Negative log-likelihood training is a special case of contrastive training, where Monte Carlo samples from the model serve as the "negative" examples.

NLL loss gradient: ∂L/∂w = ∂F(x,y)/∂w ← lower training point energy − E_{y'~P}[∂F(x,y')/∂w] ← raise energy elsewhere (MC) Conclusion: MLE is contrastive! → Subject to same dimensional scaling issues ❌
