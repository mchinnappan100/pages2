Level 01

Core Concepts & Background

01
🎯

What is I-JEPA?

I-JEPA (Image-based Joint-Embedding Predictive Architecture) is a non-generative self-supervised method for learning semantic image representations without hand-crafted data augmentations.

Predicts representations of target blocks from a context block
Operates entirely in abstract representation space — not pixels
Published at ICCV 2023 by Meta AI / FAIR
Instantiates LeCun's 2022 JEPA framework for images

Given a photo of a dog: the model sees a large context region (most of the image) and must predict the abstract representation of 4 small hidden target patches — e.g., the paw, the ear, part of the background, and the tail.

Key: it predicts what the target encoder would output for those patches, not what the pixels look like. Texture details can vary; pose and object identity cannot.

🏗️

The JEPA Framework (LeCun 2022)

Joint-Embedding Predictive Architecture: a predictor maps the embedding of one view to the embedding of another, conditioned on a latent variable. Loss is computed in embedding space, not input space.

Conceptual origin: "A path towards autonomous machine intelligence"
Latent variable z conditions the prediction (masking positions in I-JEPA)
Loss: L(ŝ_y, s_y) = ‖g_φ(s_x, z) − f_θ̄(y)‖²
No need to reconstruct unpredictable low-level details
⚖️

Three SSL Families

Self-supervised learning methods divide into three families, each with fundamental tradeoffs that I-JEPA is designed to overcome.

Invariance-based: SimCLR, DINO, iBOT — high semantic quality but augmentation bias
Generative / mask-denoising: MAE, BEiT — pixel space, multi-modal friendly, lower off-the-shelf semantics
JEPA / predictive: I-JEPA — representation space, no augmentation bias, high semantics
Invariance-based (DINO): Augment image → two views Maximize similarity of embeddings ✅ High semantic quality ❌ Color jitter biases against counting tasks Generative (MAE): Mask patches → predict pixel values ✅ No augmentation needed ❌ Model wastes capacity on texture I-JEPA: Mask patches → predict patch embeddings ✅ No augmentation needed ✅ Focuses on semantic content only
🧠

Why Representation Space Matters

Predicting in pixel space forces the model to handle unpredictable low-level details — noise, texture, exact lighting. Predicting in representation space allows these to be naturally discarded.

Pixel: must predict exact RGB values — wastes model capacity
Representation: predict semantic content — object, pose, position
Ablation shows: pixel targets → 40.7 Top-1; repr. targets → 66.9 Top-1
RCDM visualizations confirm: model captures pose, not texture
🚫

No Augmentations — Why It Matters

View-invariant methods require hand-crafted augmentations (color jitter, random crops, flips) that encode domain priors — and inadvertently hurt tasks that depend on those very properties.

Color jitter: hurts color-dependent tasks
Random crops: hurts spatial/counting tasks — object count discarded
I-JEPA: no augmentations → no such bias; outperforms DINO on Clevr/Count
Enables broader application: video, audio, multimodal without redesigning augmentations
🔄

Collapse Prevention via EMA

A key challenge in joint-embedding methods is representational collapse — all embeddings becoming identical. I-JEPA prevents this elegantly using an Exponential Moving Average target encoder.

Target encoder θ̄ ← τθ̄ + (1−τ)θ
τ starts at 0.996, slowly increases to 1.0
Target encoder never trained by gradient — provides stable targets
No negative pairs, contrastive losses, or clustering required
Level 02

Architecture & Training

02
🔧

Three Components

I-JEPA consists of a context encoder, a target encoder (EMA copy), and a predictor. All three are Vision Transformers (ViT).

Context encoder f_θ: Processes the visible context block → patch representations s_x
Target encoder f_θ̄: EMA copy; processes full image → target representations s_y
Predictor g_φ: Narrow transformer; maps s_x + mask tokens → predicted ŝ_y for each target block
Context Encoder (ViT-L/16, full depth) Input: image patches in context region Bx Output: s_x = {s_xj}_{j∈Bx} Trained by: gradient descent (AdamW) Target Encoder (ViT-L/16, EMA copy) Input: ALL image patches Output: s_y = {s_y1, ..., s_yN} Trained by: θ̄ ← 0.996·θ̄ + 0.004·θ Predictor (ViT, 12 layers, dim 384) Input: s_x + positional mask tokens {m_j} Output: ŝ_y(i) for each target block B_i Trained by: gradient descent (AdamW)
📐

The Predictor — Bottleneck Design

The predictor is a lightweight ViT with a narrower embedding dimension (384 vs 1024 for ViT-L). This bottleneck is not a limitation — it's a crucial design choice.

Predictor dim 384 (bottleneck) → 70.7 Top-1
Predictor dim 1024 (full) → 68.4 Top-1
Bottleneck prevents encoder from taking prediction shortcuts
Forces encoder to build rich, abstract representations
Depth: 6 layers (ViT-B), 12 layers (ViT-L/H), 16 layers (ViT-G)
📉

Loss Function

Average L2 distance in representation space across all M target blocks. No contrastive term, no reconstruction term, no clustering.

Loss = (1/M) Σᵢ Σ_{j∈Bᵢ} ‖ŝ_yj − s_yj‖²
M=4 independent target block predictions per image
Gradients flow to context encoder θ and predictor φ only
Target encoder θ̄ receives no gradient — only EMA updates
⚙️

Training Hyperparameters

AdamW optimizer with linear LR warmup and cosine decay. Weight decay is annealed from low to high during training.

Optimizer: AdamW, batch size 2048
LR: warm 1e-4→1e-3 (15 epochs) then cosine decay to 1e-6
Weight decay: linearly increased 0.04 → 0.4 during training
EMA momentum: 0.996 → 1.0 over training
No [cls] token — average-pooled patches for evaluation
🔍

Target Masking: Output Not Input

A critical design choice: masking is applied to the OUTPUT of the target encoder, not its input. The target encoder sees the full unmasked image.

Mask output → target encoder processes clean, unmasked input
Produces higher-quality target representations
Ablation: output masking → 67.3; input masking → 56.1 (ViT-H/16)
Contrast with MAE: applies masking before the encoder
🎨

RCDM Visualizations

Predictor outputs are decoded to pixels via a frozen Retrieval-Conditioned Diffusion Model (RCDM) to visualize what the predictor "thinks" about the masked regions.

Multiple samples from same context → vary in texture, consistent in pose
Confirms model captures semantic structure, not pixel noise
Consistent qualities: captured by the model
Variable qualities: discarded as unpredictable low-level detail
Level 03

Masking Strategy

03
🎯

Multi-Block Target Masking

4 independent target blocks are sampled per image, each covering 15–20% of the image area. This creates a rich, multi-faceted prediction task.

M = 4 target blocks per image
Scale: 15–20% of image (0.15, 0.2)
Aspect ratio: 0.75 to 1.5 (near-square)
Blocks are semantically large enough to require abstract reasoning
224×224 image → 196 patches (14×14 grid) 4 target blocks, each ~0.17 × 196 ≈ 33 patches Total masked: ~132/196 patches (67%) But: context block covers 85–100% of image → context is large and informative → prediction is hard but solvable

The combination of large context + semantically-sized targets forces the model to reason about object structure, not just interpolate adjacent pixels.

🔲

Context Block Design

A single context block covers 85–100% of the image (minus the target regions). The large informative context prevents trivial solutions.

Scale: 0.85–1.0 of image area
Unit (square) aspect ratio
Overlapping regions with target blocks removed from context
Ablation: context scale 0.40–1.0 → 31.2; 0.85–1.0 → 54.2 Top-1
📊

Target Scale Is Critical

The scale of target blocks determines whether the task requires semantic reasoning or can be solved with local texture interpolation.

Scale (0.075, 0.2): too small → trivially solvable → 19.2 Top-1
Scale (0.15, 0.2): sweet spot → 54.2 Top-1
Scale (0.2, 0.3): too large → not enough context → 33.6 Top-1
Blocks must be large enough to require understanding object structure
🔢

Number of Target Blocks

More target blocks forces the model to be predictive about more of the image simultaneously, dramatically improving representation quality.

1 target block → 9.0 Top-1
2 target blocks → 22.0 Top-1
3 target blocks → 48.5 Top-1
4 target blocks → 54.2 Top-1 ✓ chosen design
❌

Why Other Masking Strategies Fail

Ablations show that the specific multi-block strategy is crucial — rasterized, random, or single-block approaches produce drastically worse results.

Rasterized (quadrant) targets → 15.5 Top-1 (vs 54.2)
Single large block → 20.2 Top-1
Random masking (60%) → 17.6 Top-1
Small targets are trivially predictable by interpolation
Random masking: too easy; individual patches lack semantic content
🎲

Mask Tokens for Position

Positional embeddings are used as mask tokens {m_j} for the target block positions. The predictor uses these to know which locations to predict.

Each mask token m_j encodes positional embedding for patch j
Predictor input: s_x (context embeddings) + {m_j}_{j∈Bᵢ} (target positions)
Applied independently M=4 times per image
Enables conditional prediction: "predict patch at position j given context"

Compute Efficiency

I-JEPA is dramatically more efficient than alternatives — enabling large-model pretraining that was previously cost-prohibitive.

>10× more efficient than MAE ViT-H/14
2.5× faster than iBOT ViT-S/16 for equivalent or better quality
ViT-H/14 on ImageNet in <72h on 16 A100s
Only ~7% overhead vs MAE per iteration, but ~5× fewer iterations needed
📈

Scalability

I-JEPA scales predictably with larger models and datasets, with no diminishing returns in the tested range.

IN-1K → IN-22K: CIFAR-100 87.5 → 89.5, iNat18 47.6 → 50.5
ViT-H → ViT-G: further gains on semantic tasks
Larger patch size (ViT-G/16) slightly hurts local prediction tasks
No signs of saturation; architecture continues to benefit from scale
Best Practices

Key Ablation Findings

05

✓ Multi-block targets (4 blocks, 15–20% scale)

The masking strategy is the most critical design choice. Multi-block targets dramatically outperform all alternatives.

Multi-block (0.15,0.2) × 4 → 54.2 Top-1 ✅ Rasterized (quadrant) × 3 → 15.5 Top-1 ❌ Single block (0.6) × 1 → 20.2 Top-1 ❌ Random (0.6) × 1 → 17.6 Top-1 ❌

✓ Predict representations, not pixels

Operating in representation space is the core conceptual advantage over generative methods.

Target-encoder output (repr.) → 66.9 Top-1 ✅ Pixel space prediction → 40.7 Top-1 ❌ (ViT-L/16, 500 epochs)

✓ Mask after (not before) the target encoder

The target encoder must see the full, unmasked image to produce high-quality targets.

Mask output of encoder → 67.3 Top-1 ✅ Mask input to encoder → 56.1 Top-1 ❌ (ViT-H/16, 300 epochs)

✓ Use a bottleneck predictor (dim 384)

A narrower predictor forces the encoder to build richer representations by preventing shortcut learning.

Predictor dim 384 (bottleneck) → 70.7 Top-1 ✅ Predictor dim 1024 (full) → 68.4 Top-1 ❌ (ViT-L/16)

✓ Deep predictor (12+ layers for large models)

The predictor needs enough capacity to solve the spatial prediction task without becoming a bottleneck.

Predictor depth 6 → 64.0 Top-1 Predictor depth 12 → 66.9 Top-1 ✅ (ViT-L/16, 500 epochs)

✓ Anneal weight decay (0.04 → 0.4)

Increasing weight decay during training improves linear probe performance, while fixed weight decay slightly favors semi-supervised.

WD 0.04→0.4 (annealed) → 77.8 linear eval ✅ WD fixed 0.05 → 76.4 linear eval WD fixed 0.05 → 70.7 semi-supervised ✅

✓ Large, informative context block (85–100%)

A large context ensures the prediction task is well-conditioned and solvable, not just random guessing.

Context scale (0.40, 1.0) → 31.2 Top-1 ❌ Context scale (0.65, 1.0) → 47.1 Top-1 Context scale (0.85, 1.0) → 54.2 Top-1 ✅

✓ Target scale sweet spot (0.15–0.2)

Too small: trivially predictable by interpolation. Too large: not enough context. The sweet spot forces semantic reasoning.

Scale (0.075, 0.2) → 19.2 Top-1 (too small) Scale (0.15, 0.2) → 54.2 Top-1 ✅ sweet spot Scale (0.2, 0.3) → 33.6 Top-1 (too large)
