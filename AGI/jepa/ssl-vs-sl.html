Level 01

Core Concepts & The Label Problem

01
🏷️

What Is a Label?

A label is a human-assigned annotation that tells a model the correct output for a given input. Labels are the foundation of supervised learning — and their cost is the central bottleneck.

ImageNet: 1.2M images, each hand-labelled by humans
Medical imaging: labels require expert radiologists
Translation data: requires bilingual human annotators
Cost: $0.01–$1 per label × millions of samples = millions of dollars
🧠

How Humans Learn vs. How Models Learn

A critical asymmetry: humans learn rich world models as children mostly through observation — without explicit labelling. Current AI lacks this ability.

A child learns "dog" from ~10 examples across contexts
A supervised model needs thousands of labelled dog images
SSL bridges this: learn from the data's own structure
LeCun: SSL is the "dark matter of intelligence"
📊

The Four Learning Paradigms

ML training splits into four paradigms, each with different data requirements, efficiency, and generality. Understanding where SSL sits is key.

Supervised: labeled (x → y) pairs, task-specific
Unsupervised: no labels, learns structure/density
Reinforcement: reward signal from environment
Self-supervised: labels derived from the data itself
Supervised: (image, "cat") → learn image classifier Unsupervised: {images} → learn clusters or PCA Reinforcement: (state, action) → reward → learn policy Self-supervised: input: "The cat sat on the [MASK]" label: "mat" ← auto-derived from input text → model learns language structure for free

SSL is sometimes called "predictive learning" — the model predicts one part of the input from another, generating its own supervision signal.

💡

The Information Efficiency Gap

The core reason SSL has overtaken supervised pretraining: each training sample yields vastly more learning signal when you predict structure rather than a single label.

Supervised on ImageNet: ~10 bits per image (one of 1000 classes)
SSL masking on same image: ~10⁵ bits (predicting every masked patch)
GPT next-token: ~13 bits per token × thousands of tokens per document
Result: SSL models learn richer representations from the same data
Level 02

Supervised Learning — Deep Dive

02

"The supervised learning approach requires a labeled dataset. The model learns to map inputs to outputs by minimizing a loss over known (x, y) pairs. It excels at narrow, well-defined tasks where labeled data is plentiful."

— Standard ML textbook definition
⚙️

How Supervised Learning Works

A dataset of (input, label) pairs is assembled. A model f_θ is trained to minimise a task-specific loss L(f_θ(x), y) over all pairs.

Collect labeled dataset D = {(x₁,y₁), …, (xₙ,yₙ)}
Choose loss: cross-entropy (classification), MSE (regression)
Minimise: θ* = argmin_θ Σ L(f_θ(xᵢ), yᵢ)
Evaluate: measure accuracy on held-out labeled test set
Image Classification (ResNet on ImageNet): Input: 224×224 RGB image Label: integer in [0, 999] ← 1 of 1000 classes Loss: cross_entropy(logits, label) Grad: backprop through 50 conv layers Result: 78% top-1 accuracy on held-out set Cost: 1.28M labeled images ~$50,000–$500,000 in annotation cost
✅

Strengths of Supervised Learning

Supervised learning is highly effective when labeled data is available and the task is well-defined. It remains the gold standard for targeted, high-stakes tasks.

Highly interpretable: loss directly measures task performance
Precise: directly optimises the metric you care about
Proven: decades of theory, tooling, and benchmarks
Works well even with small models for narrow tasks
⚠️

Weaknesses of Supervised Learning

The label bottleneck and brittleness to distribution shift are fundamental limitations — not engineering problems that can be patched away.

Labels are expensive, slow, and often require domain experts
Brittle: degrades sharply outside the label distribution
Each new task requires a new labeled dataset
Cannot leverage the vast ocean of unlabeled data
Label noise: humans make annotation mistakes at ~3–5%
📈

Supervised Scaling Laws

Performance scales predictably with data and model size — but you hit a wall when labeled data runs out, which happens orders of magnitude before unlabeled data.

ImageNet ceiling: ~90% top-1 — annotation errors are the limit
Labeled internet images: ~billions (hard to scale beyond)
Unlabeled internet images: ~trillions (effectively unlimited)
SSL lets you use the trillion-scale dataset; SL cannot
Level 03

Self-Supervised Learning — Deep Dive

03

"Self-supervised learning obtains supervisory signals from the data itself, often leveraging the underlying structure in the data. The label comes for free — hidden inside the input."

— Yann LeCun, Meta AI Research
🔮

The Core Idea: Predict Parts from Parts

SSL works by withholding some part of the input and training the model to predict it from the rest. The "label" is auto-generated from the data — no human needed.

Text: mask 15% of tokens → predict the masked tokens (BERT)
Text: predict next token given all previous tokens (GPT)
Image: mask 75% of patches → reconstruct masked patches (MAE)
Image: predict representation of one view from another (SimCLR)
Audio: predict spectrogram of future frames (wav2vec)
BERT Masked Language Model (MLM): Input tokens: ["The", "cat", "[MASK]", "on", "the", "mat"] Target label: "sat" ← taken directly from input No human needed — mask is applied automatically GPT Causal Language Model (CLM): Input: "The cat sat on the" Target: "mat" ← next token in same sequence Trains on ALL token positions simultaneously → 1 document = thousands of training pairs

One Wikipedia article generates ~500 self-supervised training examples. The same article yields exactly 1 supervised example if used for document classification.

🏗️

SSL Pretext Tasks

The invented task used to generate self-supervised signal is called a "pretext task." The model learns rich representations as a byproduct of solving it — the pretext task is discarded at the end.

Masked token prediction: BERT, T5, SpanBERT
Next token prediction: GPT, LLaMA, Claude
Masked patch prediction: MAE, BEiT, I-JEPA
Contrastive views: SimCLR, MoCo, CLIP
Rotation prediction: RotNet (predict image rotation angle)
Jigsaw puzzle: predict permutation of shuffled patches
✅

Strengths of Self-Supervised Learning

SSL's superpower is scale: it can leverage virtually unlimited unlabeled data, making it the foundation of every modern large-scale model.

No expensive annotation — uses raw data directly
Scales to trillions of tokens / images
Learns general-purpose representations transferable to many tasks
Often outperforms supervised pretraining on downstream tasks
Enables few-shot and zero-shot generalisation
⚠️

Weaknesses of Self-Supervised Learning

SSL is not a silver bullet. It has real costs and failure modes, particularly around collapse, compute requirements, and misaligned pretext tasks.

Representation collapse: model ignores all inputs (must be prevented)
Pretext–downstream mismatch: wrong pretext → poor transfer
Computationally expensive: GPT-4 pretraining = $100M+
Harder to evaluate: no direct task metric during pretraining
Still needs supervised fine-tuning for specific tasks
🔁

Collapse — The Central Risk

The greatest failure mode of SSL: the encoder learns to output a constant vector for all inputs. Energy is minimised trivially — the model learns nothing.

If E_x and E_y → same constant: all (x,y) pairs have zero energy
Contrastive methods: push up energy of negative pairs (SimCLR, MoCo)
Regularised methods: VICReg, Barlow Twins — enforce variance
Architectural: EMA target encoder (BYOL, I-JEPA)
Collapse prevention — VICReg: Variance term: std(z_i) ≥ γ for each dimension → forces each dimension to be "used" Invariance term: ‖z_A − z_B‖² → representations of same image match Covariance term: off-diagonal of cov(Z) → 0 → each dimension encodes different information EMA (used in BYOL, I-JEPA): θ̄ ← 0.996·θ̄ + 0.004·θ → slowly evolving target prevents shortcut
🌐

Landmark SSL Models

A timeline of the key SSL breakthroughs that reshaped modern AI, from word2vec to GPT-4.

2013: word2vec — predict context words from center word
2018: BERT — bidirectional masked language model
2018: GPT — autoregressive next-token prediction
2020: SimCLR / MoCo — contrastive image SSL
2021: CLIP — image–text contrastive pretraining
2022: MAE, I-JEPA — masked image patch prediction
2023+: GPT-4, Claude, LLaMA — SSL + RLHF at scale
Level 04

The Modern Pretraining Pipeline

04

"SSL does the heavy lifting — learning general representations from massive unlabeled data. Supervised fine-tuning sharpens the output for a specific task. This two-stage pipeline has become the dominant paradigm for large-scale AI."

— Standard practice at OpenAI, Anthropic, Google, Meta

SSL Pretraining → Supervised Fine-Tuning Flow

Stage 1 — SSL Pretraining
Raw internet text / images (trillions of tokens, zero labels). Model learns to predict masked tokens / next tokens. Builds a rich internal world model.
→
Pretrained Base Model
General-purpose encoder / decoder with rich representations. Can solve many tasks zero-shot but not optimised for any specific one.
Stage 2 — Supervised Fine-Tuning
Small labeled dataset (thousands–millions of (x,y) pairs). Task-specific loss fine-tunes the pretrained weights. Much cheaper than training from scratch.
→
Task-Specific Model
Specialist model for sentiment analysis, medical diagnosis, code generation, etc. Outperforms training from scratch with far less labeled data.

Why Fine-Tuning Works So Well

✓ Representations transfer across tasks

The pretrained model already understands syntax, semantics, object shapes, and relations. Fine-tuning only teaches it the output format of the new task.

GPT pretrained on: all of Wikipedia + books Fine-tuned with: 10,000 labeled medical QA pairs → Achieves expert-level medical QA → Would need millions of pairs training from scratch

✓ Few-shot learning emerges from SSL scale

Large enough SSL models can solve new tasks with just a handful of labeled examples in the prompt — or even zero, through instruction following.

Zero-shot: "Translate to French: Hello" → No French training pairs seen during fine-tuning Few-shot (3 examples in context): English: "dog" → French: "chien" English: "cat" → French: "chat" English: "house" → ? → "maison" ✅

✓ Parameter-efficient fine-tuning (PEFT)

Because pretrained representations are so rich, you often need to fine-tune only a tiny fraction of weights (LoRA, prefix tuning) — reducing cost dramatically.

Full fine-tune GPT-4: ~$10M–$100M LoRA fine-tune (rank 16): Only 0.1% of parameters updated Cost: $100–$1,000 Performance: 95% of full fine-tune ✅

✓ Supervised head on top of SSL features

A linear classifier trained on top of frozen SSL features often rivals fully supervised models — showing how rich the learned representations are.

I-JEPA linear probe on ImageNet: Frozen SSL features + linear classifier → 79.3% top-1 accuracy vs MAE (pixel-space SSL): 77.2% vs Supervised ResNet-50: 76.1%

Level 06

When to Use Which Paradigm

06
🏥

Use Supervised When…

Supervised learning remains the right choice for narrow, high-stakes tasks where labeled data is available and domain-specific precision is paramount.

You have a well-defined output space (finite classes, bounded regression)
Labeled data is plentiful or annotation is affordable
The task is narrow and you don't need multi-task generalisation
You need a directly interpretable training signal
Real-time inference: small supervised models are cheaper to serve
Good fits for pure supervised learning: ✅ Fraud detection (tabular features, clear labels) ✅ Tumour classification from specific scan type ✅ Quality control in manufacturing (defect/no defect) ✅ Demand forecasting (continuous target, rich history) ✅ Optical character recognition (characters are finite) When NOT to use pure supervised: ❌ Open-domain question answering ❌ General-purpose code generation ❌ Zero-shot translation to unseen language pairs
🌍

Use SSL (Pretraining) When…

SSL is essential when you need general-purpose representations, cannot afford large annotation budgets, or want a foundation model to power multiple downstream tasks.

Data is abundant but unlabeled (text, images, audio, code)
You need to serve many different downstream tasks from one model
Annotation is expensive or requires scarce domain expertise
You need few-shot or zero-shot generalisation
Building a foundation model used across an organization
Good fits for SSL pretraining: ✅ LLM for code (pretrain on GitHub → fine-tune on tasks) ✅ Medical imaging foundation (pretrain on 10M scans) ✅ Multilingual NLP (pretrain on 100 languages) ✅ Audio understanding (pretrain on raw waveforms) ✅ Scientific paper understanding (pretrain on arXiv) Typical pipeline: SSL pretrain (no labels, massive data) ↓ Supervised fine-tune (small labeled set) ↓ Deploy to multiple tasks via prompting / adapters
⚖️

The Combined Pipeline

Modern AI rarely uses either paradigm in isolation. The dominant practice is SSL pretraining followed by supervised fine-tuning — getting the best of both worlds.

SSL provides: scale, generalisation, rich representations
SL provides: task alignment, measurable performance, controllability
RLHF adds: human preference alignment on top of both
Used by: GPT-4, Claude, Gemini, LLaMA, CLIP, BERT

Predicts in representation space — avoids blurry pixel prediction

Decision Guide

→ Abundant unlabeled data + multiple tasks

Use SSL pretraining. Pretrain once on the raw data corpus, then fine-tune lightweight heads per task. The investment in pretraining amortises across all tasks.

→ Small labeled dataset, narrow task

Start from a publicly available SSL-pretrained model (BERT, LLaMA, CLIP) and fine-tune on your labeled data. Never train from scratch unless you have a very strong reason.

→ Domain-specific data, no pretrained model

Pretrain with SSL on your domain data (e.g., medical records, legal filings, genomics), then fine-tune supervised. This beats both pure SSL general models and supervised from scratch.

→ Tabular / structured data

Supervised learning (gradient boosting, deep tabular nets) still dominates here. SSL has fewer natural pretext tasks for tabular data though research is active.

→ Zero / few labeled examples

Use a large SSL-pretrained model with in-context learning (prompting). Modern LLMs can solve new tasks from 0–5 examples — no fine-tuning required.

→ Real-time, resource-constrained

Distil from a large SSL model into a small supervised one. The large model provides soft labels; the small model learns to match them at a fraction of the inference cost.
