The Problem We Cannot Solve

01

"Modern LLMs work remarkably well, but we don't have a complete theory explaining why they reason, generalise, or hallucinate the way they do."

— Central thesis of this paper
🧠

What an LLM Is (Mathematically)

A transformer language model computes a conditional distribution over the next token given context. The entire system is a composition of attention and feedforward layers.

Output: $P_\theta(y_t \mid y_{<t})$ — a distribution over vocabulary
Training: minimise $\mathcal{L} = -\sum_t \log P_\theta(y_t \mid y_{<t})$
No explicit encoding of objects, causality, or goals
Knowledge is distributed — not localisable to single parameters
Transformer Forward Pass
$$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$ $$h_\ell = \text{LayerNorm}(h_{\ell-1} + \text{Attn}(h_{\ell-1}))$$ $$h_\ell' = \text{LayerNorm}(h_\ell + \text{FFN}(h_\ell))$$

The model has no loss term that rewards "understanding physics" — any such knowledge must emerge from next-token prediction alone.

📦

The Distributed Representation Problem

Knowledge is not stored in one place. The concept "dog" may activate dozens of neurons across many layers — no single location encodes it.

Layer 6: neurons {130, 441, 987} partially activate
Layer 17: attention head 12 shows relevant pattern
Layer 22: MLP neuron 3810 fires strongly
No single parameter encodes "Newton's laws"
🌀

Polysemanticity

Many neurons respond to multiple unrelated concepts simultaneously — a phenomenon called polysemanticity — making clean interpretation almost impossible.

One neuron: "Golden Gate Bridge" AND "computer graphics" AND "violin"
Superposition hypothesis: $n$ features encoded in $d \ll n$ dimensions [1]
Sparse coding: $\mathbf{h} = \text{ReLU}(W\mathbf{x})$ with $\|\mathbf{h}\|_0 \ll d$
Makes probing and ablation studies unreliable
⚠️

The Hallucination Problem

LLMs generate fluent, confident text that is factually wrong. This is not a bug — it follows directly from the training objective, which rewards fluency, not truth.

Training loss: $\mathcal{L}_{\text{LM}}$ — no truth-grounding term
Model cannot distinguish "I know this" from "I pattern-complete this"
RLHF [2] reduces but does not eliminate hallucination
No principled solution exists — an open question
Open Question 1

Mechanistic Interpretability

02

"Can we reverse-engineer neural networks the way electrical engineers reverse-engineer circuits?"

— Olah et al., Zoom In: An Introduction to Circuits [3]
🔬

Attention Head Circuits

Individual attention heads learn identifiable algorithms. Mechanistic interpretability reverse-engineers these circuits to understand what computation each head performs.

Induction heads: copy [A][B]...[A] → [B] patterns
Previous-token heads: $\text{attn}(i, i-1)$ — copy prior token
Bracket-matching heads: track syntactic structure
Name-mover heads: resolve "John ... he" pronouns [4]
import torch import torch.nn.functional as F def attention(Q, K, V, d_k): # Scaled dot-product attention # Q, K, V: (batch, heads, seq, d_k) scores = torch.matmul(Q, K.transpose(-2, -1)) / d_k**0.5 weights = F.softmax(scores, dim=-1) # (B, H, seq, seq) return torch.matmul(weights, V), weights # To probe an induction head: check if # weights[b, h, i, j] is high when # token[j] == token[i-1] (offset-1 copy pattern)

Induction heads implement in-context learning — they are a key discovery of mechanistic interpretability [3].

🧩

Sparse Autoencoders for Feature Recovery

Sparse autoencoders (SAEs) decompose polysemantic neuron activations into monosemantic features — a leading technique for recovering interpretable concepts from LLM internals.

Encode: $\mathbf{z} = \text{ReLU}(W_e \mathbf{h} + \mathbf{b}_e)$
Decode: $\hat{\mathbf{h}} = W_d \mathbf{z} + \mathbf{b}_d$
Loss: $\|\mathbf{h} - \hat{\mathbf{h}}\|_2^2 + \lambda \|\mathbf{z}\|_1$
Dictionary size $d_{\text{SAE}} \gg d_{\text{model}}$ — overcomplete basis
import torch, torch.nn as nn class SparseAutoencoder(nn.Module): def __init__(self, d_model, d_sae): super().__init__() self.W_enc = nn.Linear(d_model, d_sae) self.W_dec = nn.Linear(d_sae, d_model, bias=False) self.b_enc = nn.Parameter(torch.zeros(d_sae)) def forward(self, h): z = torch.relu(self.W_enc(h) + self.b_enc) h_hat = self.W_dec(z) return h_hat, z def sae_loss(h, h_hat, z, lam=1e-3): recon = (h - h_hat).pow(2).mean() sparsity = lam * z.abs().mean() return recon + sparsity

Each column of $W_d$ is a "feature direction" — directions in activation space that correspond to interpretable concepts [5].

🎯

Linear Representation Hypothesis

Many features appear to be encoded as linear directions in activation space. The concept $c$ corresponds to a vector $\mathbf{v}_c$ such that the model "has" the concept when $\mathbf{h} \cdot \mathbf{v}_c > \theta$.

Word2Vec analogy: $\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}} + \mathbf{v}_{\text{woman}} \approx \mathbf{v}_{\text{queen}}$
Probing: train linear classifier $\hat{c} = \text{sign}(W_p \mathbf{h})$ to test if concept $c$ is encoded
Causal intervention: add $\alpha \mathbf{v}_c$ to $\mathbf{h}$ and observe effect on output
Open question: is the linearity real, or an artifact of probing?
❓

Why Interpretability Is Still Unsolved

Even the best mechanistic tools cover only a fraction of model behaviour. The field faces fundamental scaling challenges.

GPT-4 has $\sim 10^{12}$ parameters — manually annotating circuits is infeasible
Circuits interact non-linearly — no compositional guarantee
Evaluation: no ground truth for "correct" interpretation
Representation geometry may be fundamentally non-linear at scale
Open Question 2

World Models & Causality

03
🌍

What Is a World Model?

A world model is a function that predicts future states from current states and actions, enabling planning without real-world execution.

Formal definition: $\hat{s}_{t+1} = f_\theta(s_t, a_t)$
Goal: $\min_\theta \mathbb{E}[\|s_{t+1} - \hat{s}_{t+1}\|^2]$
LLMs can simulate some world state (via text) but lack explicit dynamics
Open question: Do LLMs contain implicit world models? [6]
import torch, torch.nn as nn class SimpleWorldModel(nn.Module): """Predicts next latent state from current state + action.""" def __init__(self, d_state, d_action, d_hidden): super().__init__() self.net = nn.Sequential( nn.Linear(d_state + d_action, d_hidden), nn.SiLU(), nn.Linear(d_hidden, d_hidden), nn.SiLU(), nn.Linear(d_hidden, d_state), ) def forward(self, s, a): # s: (B, d_state), a: (B, d_action) return self.net(torch.cat([s, a], dim=-1)) # Loss: mean squared error in latent space def world_model_loss(pred_s, true_s): return (pred_s - true_s).pow(2).mean()

In JEPA, both $s_t$ and $\hat{s}_{t+1}$ live in a learned latent space — not pixel space — preventing blurry predictions.

⚡

The Blurry Prediction Problem

Predicting directly in raw signal space (pixels, tokens) with MSE loss produces blurry averages when the target is multimodal — a fundamental failure mode of pixel-space world models.

If ball can go left or right: $\hat{y} = \mathbb{E}[y]$ — ball teleports to center
Mathematically: $\arg\min_{\hat{y}} \mathbb{E}[\|y - \hat{y}\|^2] = \mathbb{E}[y]$
Solution A: generative models (VAE, GAN, Diffusion) — expensive
Solution B: JEPA — predict in representation space, not signal space
🔗

Causal vs. Statistical Learning

LLMs learn statistical correlations, not causal relationships. Causal reasoning — "if I intervene on $X$, what happens to $Y$?" — requires a different inductive bias.

Observational: $P(Y \mid X=x)$ — what co-occurs
Interventional: $P(Y \mid \text{do}(X=x))$ — causal effect [7]
LLMs conflate correlation with causation in reasoning tasks
Open question: Can gradient descent learn causal structure from observational data alone?
Pearl's Do-Calculus
$$P(Y \mid \text{do}(X)) \neq P(Y \mid X)$$ $$P(Y \mid \text{do}(X)) = \sum_z P(Y \mid X, Z=z) P(Z=z)$$

Current LLMs cannot distinguish these without explicit causal structure — a fundamental limitation for scientific reasoning and planning.

🧪

Emergent World Models in LLMs

Probing studies suggest LLMs develop implicit spatial and entity models [6]. The open question is whether these are genuine causal models or sophisticated correlation patterns.

Othello-GPT [8]: trained on Othello moves, probing recovers board state
Linear probes: $\hat{c} = W_p h_\ell + b_p$ — classification accuracy above 90%
Causal intervention: patching $h_\ell$ changes predicted board correctly
Open question: Does this scale? Is it robust to distribution shift?
Open Question 3

Reasoning, Planning & Generalisation

04
🧮

Is LLM "Reasoning" Real?

Chain-of-thought prompting dramatically improves performance on multi-step problems. Whether this constitutes genuine reasoning or sophisticated retrieval is actively debated.

CoT [9]: "Let's think step by step" → structured scratchpad
Scratchpad tokens allow model to revisit intermediate computations
Fails on novel out-of-distribution arithmetic with small perturbations
Open question: What is the computational complexity class of transformer reasoning?
🌀

Transformers and Computational Depth

A transformer with $L$ layers computes a function of depth $O(L)$. This imposes a fundamental limit on problems requiring $\omega(L)$ sequential reasoning steps.

A single forward pass has fixed depth — no iteration
Result: transformers cannot solve problems in $\text{TC}^0$ efficiently [10]
CoT helps by externalising depth into output tokens
Open question: Can recurrent or iterative architectures close this gap?
Depth-Complexity Bound
$$f_\theta = f_L \circ f_{L-1} \circ \cdots \circ f_1$$ $$\text{depth}(f_\theta) = O(L) \text{ — fixed at inference time}$$

Generating $T$ output tokens gives $O(L \cdot T)$ effective depth — explaining why longer CoT sequences solve harder problems.

🎯

Generalisation: In-Distribution vs. OOD

LLMs generalise extremely well in-distribution but can fail catastrophically on distribution shift. Understanding the geometry of the learned hypothesis class is an open problem.

In-distribution: cross-entropy generalisation gap is well-studied
OOD: systematic failure on compositional and structural shifts
Double descent: test loss rises then falls again with model size [11]
Open question: What implicit regularisation does SGD impose on transformers?
📐

Scaling Laws & Their Limits

Neural scaling laws [12] show predictable power-law improvement with compute. Whether scaling alone reaches human-level general intelligence is deeply contested.

$\mathcal{L}(N) \approx \left(\frac{N_c}{N}\right)^{\alpha_N}$ — power law in parameters $N$
$\mathcal{L}(C) \approx \left(\frac{C_c}{C}\right)^{\alpha_C}$ — power law in compute $C$
Emergent abilities: qualitative jumps at certain scale thresholds [13]
Open question: Are emergent abilities real phase transitions or measurement artifacts?
Open Question 4

JEPA: A Different Inductive Bias

05

"The question is not whether JEPA solves interpretability. It does not. The question is whether its inductive bias — predicting latent representations rather than raw tokens — produces more structured, more plannable world models."

— This paper
🧩

JEPA Objective

JEPA replaces next-token prediction with prediction in latent representation space. Given context $x$ and target $y$, encoders map both to representations; a predictor bridges the gap.

$s_x = E_x(x)$ — context representation
$s_y = E_y(y)$ — target representation (EMA encoder)
$\hat{s}_y = g(s_x, z)$ — predicted target representation
$\mathcal{L}_{\text{JEPA}} = \|g(s_x, z) - \text{sg}(s_y)\|_2^2$
import torch, torch.nn as nn, copy class JEPA(nn.Module): def __init__(self, encoder, predictor, ema_tau=0.996): super().__init__() self.enc_x = encoder # online encoder (context) self.enc_y = copy.deepcopy(encoder) # EMA target encoder for p in self.enc_y.parameters(): p.requires_grad_(False) self.predictor = predictor self.tau = ema_tau @torch.no_grad() def update_ema(self): # θ̄ ← τ·θ̄ + (1−τ)·θ for p_online, p_ema in zip( self.enc_x.parameters(), self.enc_y.parameters()): p_ema.data.mul_(self.tau).add_( p_online.data, alpha=1 - self.tau) def forward(self, x, y, z=None): s_x = self.enc_x(x) with torch.no_grad(): s_y = self.enc_y(y) # stop-gradient target if z is not None: s_hat = self.predictor(s_x, z) else: s_hat = self.predictor(s_x) loss = (s_hat - s_y.detach()).pow(2).mean() return loss

EMA update prevents collapse without requiring negative pairs. $\tau$ anneals from 0.996 → 1.0 over training.

🔍

Why Representation Space Prediction Wins

Predicting $s_y$ instead of $y$ lets the encoder discard unpredictable details (texture, exact pixel values) and retain semantic information (object identity, pose).

Discards: texture, lighting, background noise, camera jitter
Retains: object identity, spatial relationships, motion direction
No blurry predictions: $\hat{s}_y$ is deterministic given $s_x$
I-JEPA: 79.3% linear probe vs MAE 77.2% with fewer epochs [14]
🏛️

H-JEPA: Hierarchical World Models

Hierarchical JEPA stacks multiple JEPA levels at increasing temporal scales — low-level fine-grained predictions at the bottom, high-level abstract predictions at the top.

Level 1: $s^{(1)}_t \to s^{(1)}_{t+1}$ (short-horizon, fine-grained)
Level $k$: $s^{(k)}_t = \text{Pool}(s^{(k-1)}_t)$ — coarser, longer horizon
Top level plans the goal; bottom levels execute it
Enables planning complex tasks across multiple timescales
❓

What JEPA Does Not Solve

JEPA is a significant architectural advance but does not solve interpretability, hallucination, or causal reasoning — the latent space remains learned and distributed.

Latent vector $[0.34, -1.82, 0.77, \ldots]$ is still uninterpretable
Dimension 418 ≠ "gravity" — no symbolic grounding
Superposition still exists within the latent encoder
Open question: Can structured latent spaces be enforced by design?

Modular AI & The Path Forward

06
🏗️

The Modular Hypothesis

Future AI systems may consist of specialised modules — perception, memory, world model, planner, reasoner, language — rather than a single monolithic network.

Each module trains for a specific objective — interpretable interfaces
Intermediate representations are inspectable between modules
Planning and reasoning become explicit, not emergent
LeCun's six-module architecture [15] is a concrete proposal
🔮

Open Research Agenda

The field is transitioning from "make larger LLMs" to fundamental architectural questions about reasoning and grounding.

Can gradient descent learn causal structure from observational data?
Can latent spaces be structured by design to be interpretable?
Can memory, reasoning, and perception be coordinated modular components?
Can scaling laws continue, or do new architectures become necessary?

1 Measuring Polysemanticity

Use activation histograms to detect neurons that fire for unrelated inputs — a quantitative proxy for polysemanticity.

import torch def polysemanticity_score(activations, labels, n_classes): """ activations: (N, d) neuron activations labels: (N,) class labels Returns: (d,) per-neuron entropy over classes """ d = activations.shape[1] scores = torch.zeros(d) for c in range(n_classes): mask = (labels == c) p = activations[mask].mean(0) / ( activations.mean(0) + 1e-8) scores -= p * (p + 1e-8).log() return scores # high score → polysemantic

2 Linear Probe for Concept Detection

Train a linear classifier on frozen LLM activations to test whether a concept is linearly encoded at a given layer.

import torch.nn as nn class LinearProbe(nn.Module): def __init__(self, d_model, n_concepts): super().__init__() self.probe = nn.Linear(d_model, n_concepts) def forward(self, h): # h: (B, d_model) — frozen LLM activations return self.probe(h) # Train with cross-entropy; high accuracy means # concept is linearly encoded at this layer. # Causal: patch h += alpha * probe.weight[c] # and check if output changes correctly.

3 Gradient-Based Planning through World Model

Optimise action sequence $a$ by backpropagating through a differentiable world model to minimise predicted cost.

def plan(world_model, cost_fn, s0, horizon=10, n_steps=100, lr=0.01): """ world_model: s_t, a_t -> s_{t+1} cost_fn: s -> scalar cost """ # actions initialised to zero actions = torch.zeros(horizon, d_action, requires_grad=True) opt = torch.optim.Adam([actions], lr=lr) for _ in range(n_steps): s = s0 total_cost = 0.0 for t in range(horizon): s = world_model(s, actions[t]) total_cost = total_cost + cost_fn(s) opt.zero_grad() total_cost.backward() # grads flow back! opt.step() return actions.detach()

4 VICReg: Collapse-Free Self-Supervised Learning

VICReg prevents representational collapse through three explicit regularisation terms — a key training component for JEPA-style models.

def vicreg_loss(z1, z2, lam=25, mu=25, nu=1): """ z1, z2: (B, d) projected embeddings lam: invariance weight mu: variance weight nu: covariance weight """ # Invariance inv = (z1 - z2).pow(2).mean() # Variance: each dim > gamma=1 std1 = torch.sqrt(z1.var(0) + 1e-4) std2 = torch.sqrt(z2.var(0) + 1e-4) var = (F.relu(1 - std1).mean() + F.relu(1 - std2).mean()) # Covariance: off-diagonal near zero z1 = z1 - z1.mean(0) z2 = z2 - z2.mean(0) N, d = z1.shape C1 = (z1.T @ z1) / (N - 1) C2 = (z2.T @ z2) / (N - 1) cov = (off_diagonal(C1).pow(2).sum() + off_diagonal(C2).pow(2).sum()) / d return lam*inv + mu*var + nu*cov
