Chapter 01

The Problem Worth Solving

Daily friction, the existing landscape, and a clear design philosophy

1.1 The Daily Friction

Every developer, every day, faces a set of small but cumulative frictions with Git. None of these problems are hard — but they are slow, and they compound.

✍️

Commit Messages

  • Staring at a diff for 30 seconds
  • Writing vague "fix stuff" messages
  • 20 commits/week × 2 min = 40 hrs/year
🔍

History Archaeology

  • Reading a month-old commit
  • Wondering why a change was made
  • Tracing impact across files
⚔️

Merge Conflicts

  • Neither side of the diff is obvious
  • Need to understand both intents
  • Risk of silent data loss
📋

Release Notes

  • Needed at the last minute
  • Require reading all commits
  • Hard to write well under pressure
🗺️

New Codebases

  • A day spent just reading files
  • No map of the architecture
  • Can't ask the code questions
🎓

Git Concepts

  • "What does rebase actually do?"
  • Man pages are dense
  • Tutorials lack context

A local-first, Git-integrated, configurable AI assistant that works offline, protects privacy, and follows Git's own extensibility conventions. This is what git-ai fills.

1.3 Design Philosophy

1
Local-firstNo cloud dependency by default. Your code never leaves your machine unless you explicitly configure an external provider.
2
Advisory onlyAI output is always a suggestion. git-ai never commits, never pushes, never rewrites history.
3
Git is the source of truthgit-ai reads from Git; it never writes to it.
4
Follow Git conventionsExternal Git commands (git-ai on PATH) are Git's own extensibility model.
5
PluggableProviders, output formats, and capabilities are all extensible without changing core code.

Chapter 02

Design Decisions

Every architectural choice made deliberately, with explicit trade-offs

2.1 How to Integrate with Git

Three approaches exist. The choice determines the entire user experience.

OptionMechanismProsCons
A: Git Hooks prepare-commit-msg hook Automatic on every commit Per-repo, runs without consent, slows commit path
B: Git Alias alias.ai = !git-ai Works immediately after config Not discoverable, manual per-user setup required
C: External Command ✅ git-ai binary on PATH Zero config, discoverable, follows Git model Binary must be on PATH
Decision
Option C — External Git Command

Place a binary named git-ai on PATH. Git automatically maps git ai <args> to git-ai <args>. This is exactly how git lfs, git flow, and git hub work. Zero configuration from the user's perspective after install.

2.2 The Provider Decision

ApproachPrivacyCostOfflineLatency
OpenAI API Code sent to cloud $$$ No ~1–3s
Anthropic Claude Code sent to cloud $$$ No ~1–3s
Ollama (local) ✅ Never leaves machine Free Yes ~2–10s
llama.cpp (local) Never leaves machine Free Yes ~2–10s
Decision
Ollama as default, pluggable provider interface for all others

The Provider interface means adding OpenAI, Anthropic, or Gemini is a single new package implementing five methods. Existing commands need no changes.

2.3 The Configuration Decision

Git already has a configuration system: ~/.gitconfig and .git/config. Every Git user already knows how to use git config.

Decision
Use Git's own config system — no new file formats

Users configure git-ai exactly like they configure Git itself. No new config file location, no new documentation burden.

2.4 The Output Decision

Developers use git-ai in two distinct contexts: interactive terminals and scripts/CI pipelines.

Decision
--format flag on every command: plain (default), markdown, json

JSON mode suppresses all decorative output and emits a single JSON object. Every command becomes scriptable with zero post-processing.

4.2 The Universal Data Flow

The data flow for every command is identical. This consistency was a deliberate design decision — it makes commands easy to write, test, and reason about.

1
Parse CLI flagscobra parses --format, --style, --severity, --amend, etc.
2
Load configinternal/config reads git-config, applies defaults, returns Config struct.
3
Check git repointernal/git — git rev-parse --show-toplevel. Exit 2 if not a repo.
4
Gather contextinternal/git — StagedDiff(), CommitShow(), LogRange(), Metadata().
5
Build promptinternal/prompt — template + context + token budget truncation.
6
Call providerinternal/provider — POST /api/generate (or equivalent). Provider selected from config.
7
Format outputinternal/output — plain, markdown, or JSON based on --format flag.
8
Print to stdoutfmt.Println(result). The response is printed and the program exits.
9
Exit 0The repository is never touched. This is the invariant that makes git-ai safe.

4.4 The Registry Pattern

internal/provider/registry.go — the factory functiongo
// New is the provider factory. Commands call provider.New(cfg) and receive
// a Provider interface. They never import ollama, openai, or llamacpp directly.
func New(cfg *config.Config) (Provider, error) {
    switch cfg.Provider {
    case "ollama", "":
        return ollama.New(cfg.Ollama.URL, cfg.Model), nil
    case "openai":
        return openai.New(cfg.OpenAI.BaseURL, cfg.OpenAI.APIKey, cfg.Model), nil
    case "llamacpp":
        return llamacpp.New(cfg.LlamaCpp.URL, cfg.Model), nil
    default:
        return nil, fmt.Errorf("unknown provider %q", cfg.Provider)
    }
}

5.2 Why internal/?

Go's internal/ convention makes packages private to the module. External code cannot import github.com/mchinnappan/git-ai/internal/.... This is intentional:

  • The provider interface, prompt templates, and config schema are implementation details.
  • Future breaking changes to internal packages cannot break external importers.
  • Only commands/ and cmd/git-ai/main.go form the public surface.

5.3 Why commands/ is Separate from cmd/git-ai/

cmd/git-ai/main.go is deliberately thin: register commands, execute cobra, handle exit codes. All logic lives in commands/. This separation means commands are testable without building the binary.

internal/config/config.gogo
package config

import (
    "os"       // os.Getenv — reads OPENAI_API_KEY
    "os/exec"  // exec.Command — runs "git config --get <key>"
    "strconv"  // strconv.ParseFloat, Atoi — parses numeric values
    "strings"  // strings.TrimSpace — cleans git output
)

// Config is the fully-resolved configuration for one git-ai invocation.
// Immutable after Load() returns.
type Config struct {
    Provider    string
    Model       string
    Temperature float64
    MaxTokens   int
    Ollama      OllamaConfig
    OpenAI      OpenAIConfig
    LlamaCpp    LlamaCppConfig
    Review      ReviewConfig
    Commit      CommitConfig
}

func Load() (*Config, error) {
    // Start with safe, sensible defaults.
    cfg := &Config{
        Provider:    "ollama",
        Model:       "llama3.2:latest",
        Temperature: 0.2,   // low = focused, predictable output
        MaxTokens:   4096,
        Ollama:      OllamaConfig{URL: "http://localhost:11434"},
        OpenAI:      OpenAIConfig{BaseURL: "https://api.openai.com"},
        LlamaCpp:    LlamaCppConfig{URL: "http://localhost:8080"},
        Review:      ReviewConfig{Severity: "medium"},
        Commit:      CommitConfig{Style: "conventional"},
    }

    // set overwrites *dest only when git config has a value.
    // Preserves the default when the key is absent.
    set := func(dest *string, key string) {
        if v := gitConfig(key); v != "" { *dest = v }
    }

    set(&cfg.Provider, "ai.provider")
    set(&cfg.Model,    "ai.model")

    // Numeric values need explicit parsing.
    if v := gitConfig("ai.temperature"); v != "" {
        if f, err := strconv.ParseFloat(v, 64); err == nil { cfg.Temperature = f }
    }
    if v := gitConfig("ai.maxTokens"); v != "" {
        if n, err := strconv.Atoi(v); err == nil { cfg.MaxTokens = n }
    }

    set(&cfg.Ollama.URL,    "ai.ollama.url")
    set(&cfg.OpenAI.BaseURL,"ai.openai.url")
    set(&cfg.LlamaCpp.URL,  "ai.llamacpp.url")
    set(&cfg.Review.Severity,"ai.review.severity")
    set(&cfg.Commit.Style,  "ai.commit.style")

    // API key: git-config first (per-repo), then environment variable.
    if v := gitConfig("ai.openai.apikey"); v != "" {
        cfg.OpenAI.APIKey = v
    } else if v := os.Getenv("OPENAI_API_KEY"); v != "" {
        cfg.OpenAI.APIKey = v
    }
    return cfg, nil
}

// gitConfig runs "git config --get <key>" and returns trimmed output.
// Returns "" if the key is absent — git exits 1 for missing keys, which is normal.
func gitConfig(key string) string {
    out, err := exec.Command("git", "config", "--get", key).Output()
    if err != nil { return "" }
    return strings.TrimSpace(string(out))
}

Chapter 07

The Git Layer

A thin, named wrapper around exec.Command — readable as documentation

7.1 Design Rationale

The git layer provides named functions for every operation git-ai needs. Commands never construct raw exec.Command calls. git.StagedDiff() is unambiguous; exec.Command("git","diff","--cached") is not.

7.2 Full Annotated Implementation

internal/git/git.gogo
package git

import (
    "errors"   // errors.As — unwraps ExitError to get stderr text
    "os/exec"  // exec.Command — spawns git subprocesses
    "strings"  // TrimSpace, Split
)

// Run executes "git <args...>" and returns stdout as a trimmed string.
// On failure, returns stderr so callers see the same message a developer would.
func Run(args ...string) (string, error) {
    out, err := exec.Command("git", args...).Output()
    if err != nil {
        var exitErr *exec.ExitError
        if errors.As(err, &exitErr) {
            return "", errors.New(strings.TrimSpace(string(exitErr.Stderr)))
        }
        return "", err
    }
    return strings.TrimSpace(string(out)), nil
}

// RepoRoot verifies we are inside a git repo and returns the root path.
// Commands call this first — failure → exit code 2.
func RepoRoot() (string, error) {
    return Run("rev-parse", "--show-toplevel")
}

// StagedDiff returns changes in the index (what git commit would record).
// This is the primary input for `git ai commit` and `git ai review`.
func StagedDiff() (string, error) {
    return Run("diff", "--cached")
}

// CommitShow returns the full diff + metadata for a ref.
// ref can be "HEAD", "HEAD~3", a SHA, or a tag name.
func CommitShow(ref string) (string, error) {
    return Run("show", ref)
}

// LogRange returns one-line log for from..to (from exclusive, to inclusive).
func LogRange(from, to string) (string, error) {
    return Run("log", "--oneline", from+".."+to)
}

// ConflictedFiles returns files with unresolved merge conflicts.
// --diff-filter=U selects only "Unmerged" entries.
func ConflictedFiles() ([]string, error) {
    out, err := Run("diff", "--name-only", "--diff-filter=U")
    if err != nil || out == "" { return nil, err }
    return strings.Split(out, "\n"), nil
}

// Metadata gathers a snapshot of repo state used in every prompt template.
// Returns a map because templates reference keys by name, not position.
func Metadata() (map[string]string, error) {
    meta := map[string]string{}
    root, err := RepoRoot()
    if err != nil { return nil, err }
    meta["root"] = root
    branch, _ := Run("rev-parse", "--abbrev-ref", "HEAD")
    meta["branch"] = branch
    head, _ := Run("rev-parse", "--short", "HEAD")
    meta["head"] = head
    status, _ := Run("status", "--short")
    meta["status"] = status
    return meta, nil
}

The Provider System

8.1 The Ollama Provider

internal/provider/ollama/ollama.go — key methodsgo
type Provider struct {
    URL    string
    Model  string
    client *http.Client  // reused across requests — not created per call
}

// New creates a Provider with a 120-second timeout.
// 120s accommodates large diffs on slow hardware (e.g. CPU-only inference).
func New(url, model string) *Provider {
    return &Provider{
        URL: url, Model: model,
        client: &http.Client{Timeout: 120 * time.Second},
    }
}

// Health pings the Ollama root — returns 200 OK with "Ollama is running".
func (p *Provider) Health() error {
    resp, err := p.client.Get(p.URL)
    if err != nil {
        return fmt.Errorf("ollama not reachable at %s: %w", p.URL, err)
    }
    resp.Body.Close()
    return nil
}

// Generate sends prompt to POST /api/generate with stream:false.
// stream:false = single HTTP round-trip, full response in one JSON object.
func (p *Provider) Generate(ctx context.Context, prompt string) (string, error) {
    body, _ := json.Marshal(generateRequest{
        Model: p.Model, Prompt: prompt, Stream: false,
    })
    req, _ := http.NewRequestWithContext(ctx, "POST",
        p.URL+"/api/generate", bytes.NewReader(body))
    req.Header.Set("Content-Type", "application/json")

    resp, err := p.client.Do(req)
    if err != nil { return "", err }
    defer resp.Body.Close()

    raw, _ := io.ReadAll(resp.Body)
    var gr generateResponse
    json.Unmarshal(raw, &gr)
    if gr.Error != "" { return "", fmt.Errorf("ollama: %s", gr.Error) }
    return gr.Response, nil
}

Chapter 09

The Prompt Engine

Deterministic templates, bounded context, explicit system instructions

9.1 Three Design Principles

🎯

Deterministic

  • Same inputs → identical prompt
  • Enables regression testing
  • Output is reproducible across runs
  • No random template selection
📏

Bounded

  • Every prompt respects maxTokens
  • Large diffs are truncated visibly
  • Never silently cuts context
  • 1 token ≈ 4 chars approximation
📋

Explicit System Instructions

  • Every prompt begins with 4 rules
  • Prevents hallucination
  • Demands brevity and action
  • "Output ONLY…" eliminates preamble

9.2 The System Preamble

9.3 The Build Function — Dispatch Table

internal/prompt/prompt.go — Build()go
// Build assembles a complete, token-budgeted prompt.
//   kind      — template name ("commit", "review", "ask", etc.)
//   meta      — key-value repo metadata (branch, head, existing_message, ...)
//   diff      — raw content: diff text, log, conflict text, or repo context
//   maxTokens — configured token budget
func Build(kind string, meta map[string]string, diff string, maxTokens int) string {
    diff = truncateDiff(diff, maxTokens)  // always truncate first

    switch kind {
    case "commit":          return commitPrompt(meta, diff)
    case "commit_amend":    return commitAmendPrompt(meta, diff)
    case "review":          return reviewPrompt(meta, diff)
    case "explain_commit":  return explainCommitPrompt(meta, diff)
    case "explain_concept": return explainConceptPrompt(meta["concept"])
    case "diff":            return diffPrompt(meta, diff)
    case "changelog":       return changelogPrompt(meta, diff)
    case "release":         return releasePrompt(meta, diff)
    case "conflicts":       return conflictsPrompt(meta, diff)
    case "repo":            return repoPrompt(diff)
    case "ask":             return askPrompt(meta["question"], diff)
    default:
        return fmt.Sprintf("%s\n\n%s", systemPrompt, diff)
    }
}

9.4 Token Budget Enforcement

internal/prompt/prompt.go — truncateDiff()go
// truncateDiff trims diff content to fit within the token budget.
// Approximation: 1 token ≈ 4 characters (standard for English/code text).
// We reserve 512 tokens for the prompt template, leaving the rest for diff.
// Lines are preserved whole — we never cut in the middle of a diff hunk.
func truncateDiff(diff string, maxTokens int) string {
    budget := (maxTokens - 512) * 4  // chars available for diff
    if budget <= 0 { budget = 4096 }  // safety floor
    if len(diff) <= budget { return diff }  // fits — return unchanged

    lines := strings.Split(diff, "\n")
    var b strings.Builder
    for _, line := range lines {
        if b.Len()+len(line)+1 > budget {
            // Visible marker — developer knows context was cut
            b.WriteString("\n[... diff truncated to fit token budget ...]")
            break
        }
        b.WriteString(line)
        b.WriteByte('\n')
    }
    return b.String()
}

func commitPrompt(meta map[string]string, diff string) string {
    return fmt.Sprintf(`%s

Repository context:
  Branch: %s
  HEAD: %s

Staged diff:
%s

Task: Write a concise commit message for these staged changes.
- Use the Conventional Commits format (type(scope): description).
- First line max 72 characters.
- Add a blank line then a short body if the change needs explanation.
- Output ONLY the commit message, nothing else.`,
        systemPrompt,       // the four-rule system preamble
        meta["branch"],   // e.g. "main" or "feature/auth"
        meta["head"],     // e.g. "a3f9b12"
        diff)               // truncated staged diff
}
Why "Output ONLY the commit message"?
Without this instruction, models prepend "Here is a commit message for your changes:" and append "Let me know if you'd like me to adjust it." That prose is noise in a terminal. The explicit constraint removes it entirely.

10.2 Language Detection Implementation

internal/prompt/context.go — detectLanguages()go
// detectLanguages walks two directory levels and counts file extensions.
// Two levels is a deliberate trade-off: deep enough to detect real usage,
// fast enough to not block the user waiting for a full recursive walk.
func detectLanguages(root string) []string {
    counts := map[string]int{}
    extLang := map[string]string{
        ".go": "Go", ".ts": "TypeScript", ".tsx": "TypeScript",
        ".js": "JavaScript", ".py": "Python", ".rs": "Rust",
        ".java": "Java", ".rb": "Ruby", ".cs": "C#",
        ".sh": "Shell", ".toml": "TOML", ".yaml": "YAML",
    }
    entries, _ := os.ReadDir(root)
    for _, e := range entries {
        if strings.HasPrefix(e.Name(), ".") { continue }  // skip .git, .github
        path := filepath.Join(root, e.Name())
        if e.IsDir() {
            sub, _ := os.ReadDir(path)
            for _, s := range sub {
                if lang, ok := extLang[filepath.Ext(s.Name())]; ok { counts[lang]++ }
            }
        } else {
            if lang, ok := extLang[filepath.Ext(e.Name())]; ok { counts[lang]++ }
        }
    }
    // Sort by count descending, return names only.
    // (insertion sort is fine — slice is at most ~20 entries)
    return sortedLangs(counts)
}

10.3 The Ask Prompt

internal/prompt/prompt.go — askPrompt()go
func askPrompt(question, repoContext string) string {
    return fmt.Sprintf(`%s

Repository context:
%s

Question: %s

Task: Answer the question using only the repository context supplied above.
If the answer cannot be determined from the context, say so explicitly.
Be concise and cite specific files or commits where relevant.`,
        systemPrompt, repoContext, question)
}
"Say so explicitly"
Without this instruction, models answer questions the context doesn't support — confidently and incorrectly. The explicit constraint forces honesty about knowledge boundaries.
Key packages used by the context builder: os.ReadDir (directory walking), os.ReadFile (README and manifests), path/filepath (extension detection), strings.Builder (efficient string construction). No third-party dependencies.
