DKIM Verification White Paper

Abstract

DomainKeys Identified Mail (DKIM) is a critical email authentication mechanism that enables receiving systems to verify the integrity and authenticity of email messages. This paper explains how modern email clients such as Gmail validate DKIM signatures using DNS-based public key cryptography.

1. Introduction

Email remains one of the most widely used communication systems. However, it is also vulnerable to spoofing and tampering. DKIM provides a cryptographic mechanism to ensure that messages are sent by authorized domains and remain unchanged during transit.

2. DKIM Overview

  • Uses public/private key cryptography
  • Signing happens at the sending mail server
  • Verification happens at the receiving mail server
  • Public keys are published via DNS

3. DKIM Architecture

Sender Mail Server
    ↓ (DKIM Signature added)
Internet Transit
    ↓
Receiver (e.g., Gmail)
    ↓
DNS Lookup → Public Key
    ↓
Signature Verification

4. DKIM Verification Process

4.1 Signature Extraction

Gmail extracts the DKIM-Signature header from the email.

DKIM-Signature: v=1; a=rsa-sha256; d=example.com; s=selector1;

4.2 DNS Lookup

Using the selector and domain, Gmail queries DNS:

selector1._domainkey.example.com

4.3 Public Key Retrieval

v=DKIM1; k=rsa; p=PUBLIC_KEY

4.4 Hash Recalculation

Gmail recomputes the email body hash and canonicalized headers as defined in the DKIM signature.

4.5 Cryptographic Verification

The public key is used to verify the signature. If the computed values match the signed values, the email is considered authentic.

5. Verification Outcomes

  • Pass: Signature valid, email trusted
  • Fail: Signature invalid or message altered
  • Neutral: No signature or unable to verify

6. Integration with SPF and DMARC

DKIM works alongside SPF and DMARC to provide a comprehensive email authentication framework. While DKIM ensures message integrity, SPF validates sending servers and DMARC enforces policy.

7. Conclusion

DKIM is a foundational technology for securing email communication. By leveraging DNS-based public key distribution and cryptographic validation, it ensures trust in modern email systems.
