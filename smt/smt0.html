Salesforce Migration Tool Documentation

Overview

The Salesforce Migration Tool is a desktop application designed to simplify data migration between Salesforce organizations. It supports complex data migrations, including single and two-step processes, using JSON load plans to define objects, queries, and field mappings. The tool leverages the Salesforce CLI and Bulk API 2.0 for secure and efficient data operations.

Features

  • Authentication: Securely authenticate with source and target Salesforce orgs using usernames or aliases via Salesforce CLI.
  • JSON Load Plans: Define migration plans in JSON format, supporting single-step and two-step migrations with composite keys and field mappings.
  • Monaco Editor Integration: Edit and validate JSON load plans with real-time syntax checking.
  • Data Migration: Execute migrations with support for inserts and updates, using Bulk API 2.0 for high-performance data loading.
  • CSV Generation: Generate CSV files for source and target data, with options to export data without loading.
  • Data Export: Export data from both source and target orgs as CSV files, packaged into ZIP archives with a summary report.
  • Progress Tracking: Monitor migration progress with a progress bar and detailed logs.
  • Pause and Resume: Pause migrations after each object for review, with options to continue or stop.
  • Theme Support: Toggle between light and dark themes for better user experience.
  • Drag-and-Drop Upload: Upload JSON load plans via drag-and-drop or file selection.

Setup Requirements

To use the Salesforce Migration Tool, ensure the following prerequisites are met:

  • Node.js: Install Node.js (v14 or higher) to support Salesforce CLI installation.
  • Salesforce CLI: Install globally using: 
    npm install -g @salesforce/cli
    Verify installation with: 
    sf --version
  • Authenticated Salesforce Orgs: Authenticate your source and target orgs using: 
    sf org login web -a <alias>
    Check authenticated orgs with: 
    sf force org list

How to Use

Follow these steps to migrate data using the Salesforce Migration Tool:

  1. Authenticate Orgs:
    • Navigate to the Connection tab.
    • Enter the username or alias for the source and target orgs.
    • Click Authenticate for each org. The status will update to "Connected" upon successful authentication.
  2. Load a Plan:
    • Go to the Load Plan tab.
    • Upload a JSON load plan by clicking the upload area or dragging and dropping a .json file.
    • Alternatively, click Load Sample Plan to use a predefined plan.
    • Select objects to include in the migration from the modal that appears.
    • Edit the JSON in the Monaco Editor if needed, then click Validate JSON to ensure correctness.
    • Download the edited plan using Download Plan if desired.
  3. Execute Migration:
    • Navigate to the Migration tab.
    • Select objects to migrate using the pill box interface.
    • Click Validate Migration to verify org connectivity and object configurations.
    • Click Start Migration to begin data transfer. The migration pauses after each object, allowing you to continue or stop.
    • Alternatively, click Generate CSV Only to export data without loading it to the target org.
    • Use Export Data to download CSV files from both orgs, packaged as all.zip with a summary.
    • Monitor progress in the progress bar and logs. Download logs using Download Log.

Load Plan Structure

The load plan is a JSON file that defines the objects, queries, and field mappings for migration. Below is an example structure: 


[
  {
    "object": "AttributeCategory",
    "compositeKeys": ["Code"],
    "query": "SELECT Name, Description, Code FROM AttributeCategory WHERE Code != null",
    "fieldMappings": {
      "Name": "Name",
      "Code": "Code",
      "Description": "Description"
    }
  },
  {
    "object": "AttributePicklist",
    "compositeKeys": ["Name"],
    "query": "SELECT Name, Description, DataType, Status FROM AttributePicklist WHERE Status = 'Active'  AND Name != null",
    "fieldMappings": {
      "Name": "Name",
      "Description": "Description",
      "Status": "Status",
      "DataType": "DataType"
    }
  },
  {
    "object": "AttributeCategoryAttribute",
    "compositeKeys": ["AttributeCategory.Code", "AttributeDefinition.Code"],
    "query": "SELECT  AttributeCategory.Code, AttributeDefinition.Code FROM AttributeCategoryAttribute",
    "fieldMappings": {
      "AttributeCategoryId": {
        "lookup": {
          "object": "AttributeCategory",
          "key": "Code",
          "field": "AttributeCategory.Code"
        }
      },
      "AttributeDefinitionId": {
        "lookup": {
          "object": "AttributeDefinition",
          "key": "Code",
          "field": "AttributeDefinition.Code"
        }
      }
    }
  }
  ]

Key components:

  • object: The Salesforce object to migrate (e.g., Product2).
  • compositeKeys: Array of fields used to identify records uniquely.
  • query: SOQL query to retrieve records from the source org.
  • fieldMappings: Defines how source fields map to target fields, supporting direct mappings, lookups, and composite lookups.
  • steps: Optional array for two-step migrations, each with its own query and field mappings.

Troubleshooting

  • Salesforce CLI Not Found: Ensure Salesforce CLI is installed and accessible. Run sf --version to verify. Install with npm install -g @salesforce/cli.
  • Authentication Failure: Verify usernames/aliases and ensure orgs are authenticated using sf force auth web login.
  • Invalid JSON: Check the load plan for syntax errors in the Editor. Use Validate JSON to identify issues.
  • Migration Errors: Review logs in the Migration tab. Common issues include missing fields, invalid queries, or insufficient permissions.
  • Bulk API Issues: Ensure the target org supports Bulk API 2.0 and that the user has appropriate permissions.
