Step 2: Deploy Apex Controllers

Create the following Apex classes:

JsonReportController.cls

public with sharing class JsonReportController {
    @AuraEnabled
    public static List<SObject> getReportData(
        String objectApiName, 
        List<String> fields, 
        String filter, 
        Integer recordLimit
    ) {
        // Build dynamic SOQL query
        String query = 'SELECT ' + String.join(fields, ', ');
        query += ' FROM ' + objectApiName;
        
        if (String.isNotBlank(filter)) {
            query += ' WHERE ' + filter;
        }
        
        if (recordLimit != null && recordLimit > 0) {
            query += ' LIMIT ' + recordLimit;
        }
        
        return Database.query(query);
    }
}

CsvExportController.cls

public with sharing class CsvExportController {
    @AuraEnabled
    public static String generateCSVFile(String csvData, String fileName) {
        // Decode Base64 CSV data
        Blob csvBlob = EncodingUtil.base64Decode(csvData);
        
        // Create ContentVersion
        ContentVersion cv = new ContentVersion();
        cv.Title = fileName;
        cv.PathOnClient = fileName;
        cv.VersionData = csvBlob;
        cv.FirstPublishLocationId = UserInfo.getUserId();
        insert cv;
        
        // Get download URL
        cv = [SELECT ContentDocumentId FROM ContentVersion WHERE Id = :cv.Id];
        return '/sfc/servlet.shepherd/document/download/' + cv.ContentDocumentId;
    }
}

üìñ Usage Guide

üîÑ Basic Workflow

  1. Prepare JSON Configuration - Create a JSON file with report and dashboard specifications
  2. Upload File - Use the file upload input to select your JSON file
  3. Review Preview - Check the floating balloon to verify JSON structure
  4. Interact with Data - Search, sort, and paginate through results
  5. Visualize - View rendered charts based on your configuration
  6. Export - Download data as CSV if needed

üìù JSON Configuration Structure

Your JSON file must follow this structure:

{
  "report": {
    "object": "Account",
    "fields": ["Name", "Industry", "AnnualRevenue"],
    "filter": "Industry != null",
    "recordLimit": 100,
    "allowExport": true
  },
  "dashboard": {
    "charts": [
      {
        "title": "Revenue by Industry",
        "type": "bar",
        "xAxis": "Industry",
        "yAxis": "AnnualRevenue"
      }
    ]
  }
}

üéõÔ∏è Configuration Parameters

Parameter Type Required Description
object String ‚úÖ Yes Salesforce object API name (e.g., "Account", "Opportunity")
fields Array ‚úÖ Yes List of field API names to retrieve
filter String ‚ùå No SOQL WHERE clause (without "WHERE" keyword)
recordLimit Integer ‚ùå No Maximum number of records to retrieve
allowExport Boolean ‚ùå No Enable/disable CSV export button
charts Array ‚úÖ Yes Array of chart configuration objects
title String ‚úÖ Yes Chart title/label
type String ‚úÖ Yes Chart type: "bar", "line", "pie", "doughnut"
xAxis String ‚úÖ Yes Field name for X-axis data
yAxis String ‚úÖ Yes Field name for Y-axis data (numeric)

Processes uploaded JSON file and initializes report data.

Parameters: event (File input change event)

Parameters: object, fields, filter, recordLimit

handleSearch(event)

Filters data based on search input.

Parameters: event (Search input change event)

handleSort(event)

Sorts datatable by selected column.

Parameters: event (Datatable sort event)

renderCharts()

Renders Chart.js visualizations based on configuration.

Parameters: None

exportToCSV()

Generates and downloads CSV file of current data.

Parameters: None

üîç Troubleshooting

‚ùå Charts Not Rendering

Solution:

  • Verify Chart.js static resource is uploaded correctly
  • Check browser console for JavaScript errors
  • Ensure yAxis field contains numeric data
  • Confirm canvas elements are present in DOM

‚ùå No Data Displayed

Solution:

  • Check user has read access to specified object and fields
  • Verify SOQL filter syntax is correct
  • Review Apex debug logs for query errors
  • Ensure recordLimit allows sufficient records

‚ùå CSV Export Not Working

Solution:

  • Confirm allowExport is set to true in JSON
  • Check user has create permission on ContentVersion
  • Review CsvExportController for errors
  • Verify Base64 encoding is correct

‚ùå JSON Parsing Error

Solution:

  • Validate JSON syntax using online validator
  • Ensure all required fields are present
  • Check for trailing commas and proper quotes
  • Verify file encoding is UTF-8

‚ú® Best Practices

‚úÖ Do's

  • Use meaningful chart titles and labels
  • Limit recordLimit to improve performance
  • Test JSON configurations in sandbox first
  • Use indexed fields in filter conditions
  • Include Id field for better datatable performance
  • Validate field-level security before deployment

‚ùå Don'ts

  • Don't query too many records without pagination
  • Don't use complex SOQL subqueries in filter
  • Don't expose sensitive data in reports
  • Don't create charts with non-numeric yAxis
  • Don't skip error handling in Apex
  • Don't forget to test with different user profiles
