Step 1: Design Your Report

Business users and admins create reports using the standard Salesforce interface:

  • Navigate to Reports tab in Salesforce
  • Click "New Report" and select object type
  • Add fields, filters, and groupings using drag-and-drop
  • Configure charts and dashboard elements
  • Save and run the report to verify results

Step 2: Export as JSON

Export the report data in JSON format:

  • Open your saved report in Salesforce
  • Click the "Export" button
  • Select "JSON" as the export format
  • Download the JSON file to your local machine
  • Optionally validate the JSON structure

Step 3: Upload to LWC Viewer

Access the LWC viewer component in your LWR site:

  • Navigate to your LWR site page containing the viewer component
  • Click the file upload button
  • Select the JSON file you downloaded
  • Review the JSON preview balloon to verify contents
  • Component automatically processes the data

Step 4: Automatic Rendering

The LWC component intelligently processes and displays your data:

  • Parses JSON structure (factMap for matrix reports)
  • Generates dynamic table columns based on fields
  • Renders interactive charts using Chart.js
  • Applies SLDS styling for consistent look and feel
  • Implements pagination for large datasets

Step 5: User Interaction

End users can interact with the data in multiple ways:

  • Search across all fields in real-time
  • Sort columns by clicking headers
  • Navigate through pages using pagination controls
  • View data visualizations and charts
  • Export filtered/sorted data to CSV for offline analysis

‚ùì Frequently Asked Questions

Do I need coding skills to use these components?

No! Business users and admins can create reports using the standard Salesforce interface without any coding. Developers only need to deploy the LWC components once.

Can I update reports after deployment?

Yes! Simply modify your report in Salesforce, re-export as JSON, and upload the new file. Changes are reflected immediately.

Does this work with custom objects?

Absolutely! These components work with any Salesforce object‚Äîstandard or custom‚Äîas long as users have proper field-level security and sharing access.

What about data security?

Security is maintained through Salesforce's standard mechanisms. Field-level security, object permissions, and sharing rules are all enforced automatically.

Can I customize the look and feel?

Yes! Components use SLDS styling which can be customized. You can also modify the CSS in the component files to match your branding.

How large can my reports be?

Components include pagination to handle large datasets efficiently. For very large reports (thousands of records), consider using filters in Salesforce before exporting.

Can external users access these reports?

Yes! When deployed to Experience Cloud sites, external users (partners, customers) can access reports based on their community profile permissions.

Is real-time data supported?

Reports show data from the time of JSON export. For real-time updates, consider automating the export process or implement API-based data fetching in a future enhancement.
