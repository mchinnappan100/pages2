Full Copy Sandbox Refresh

Important information before refreshing your sandbox from Production

Critical Warning

A sandbox refresh is a destructive operation. All existing data in your sandbox will be permanently overwritten.

What Gets Copied

  • ✓ All metadata: Custom objects, fields, workflows, validation rules, and configurations
  • ✓ All production data: Records from all standard and custom objects
  • ✓ Apex code & triggers: All classes, triggers, and test classes
  • ✓ Page layouts & apps: All UI configurations and Lightning apps

What Doesn't Get Copied

  • ✗ Connected Apps with Consumer Keys: OAuth integrations must be reconfigured
  • ✗ Named Credentials: External system credentials need to be re-entered
  • ✗ Email deliverability settings: Sandbox email sending is restricted by default
  • ✗ Custom domain URLs: Sandbox URL structure differs from production
  • ✗ Certificate & Key pairs: Security certificates must be regenerated

Critical Reminders

Data Will Be Overwritten

Any data, customizations, or testing work in your current sandbox will be permanently lost and replaced with production data.

User Passwords Reset

All sandbox user passwords will be reset. Users will need to reset their passwords after the refresh.

Integration Testing Required

All external integrations, APIs, and connected apps will need to be reconfigured and tested.

Chatter Data Excluded

Chatter posts, feeds, and file attachments are not copied during a sandbox refresh.

Pre-Refresh Checklist

Always verify the refresh status and test critical functionality before resuming normal operations.
